# Snort_Built_In_Variables_Mental_Model

| Concept | Operational Meaning |
|---|---|
| Snort variable | A named placeholder defined in snort.conf that substitutes for an IP address, port, or range in rule headers |
| $ prefix | All variables in Snort rules are prefixed with $ to distinguish them from literal IP values |
| $HOME_NET | Represents the internal network; matches traffic originating from or destined to internal addresses |
| $EXTERNAL_NET | Represents external addresses; typically defined as everything that is not $HOME_NET |
| $HTTP_PORTS | Represents the ports Snort treats as HTTP traffic; defaults to port 80 but can include others |
| any keyword | Not a variable; a literal wildcard that matches all IPs or all ports in that position |
| snort.conf | The central configuration file where all built-in and custom variables are defined and assigned values |
| Custom variable | A user-defined variable set in snort.conf using the var keyword; referenced in rules with the $ prefix |
| Variable scope | Variables apply globally across all rules that reference them; changing a variable in snort.conf changes behavior for every rule using it |
| $HOME_NET vs any | $HOME_NET targets only the defined internal range; any matches everything including external addresses |
| $EXTERNAL\_NET vs !$HOME_NET | $EXTERNAL\_NET is typically defined as !$HOME_NET in snort.conf; they are functionally equivalent when configured that way |
| Variable advantage | Using variables makes rules portable; updating the IP range in snort.conf updates all rules that reference that variable at once |

# Snort_Built_In_Variables_Configuration_Checklist

| Step | Task | Device | Command | Expected Result |
|---:|---|---|---|---|
| 1 | Open snort.conf to view or edit variable definitions | Ubuntu Workstation | sudo nano /etc/snort/snort.conf | Configuration file opens; variable definitions visible near the top |
| 2 | Locate the $HOME_NET definition | Ubuntu Workstation | Search for HOME_NET in snort.conf | Line reads: var HOME_NET <ip_range_or_CIDR> |
| 3 | Set $HOME_NET to the internal subnet | Ubuntu Workstation | var HOME_NET 192.168.1.0/24 | $HOME_NET now resolves to the defined internal range in all rules |
| 4 | Locate the $EXTERNAL\_NET definition | Ubuntu Workstation | Search for EXTERNAL\_NET in snort.conf | Line typically reads: var EXTERNAL\_NET !$HOME_NET |
| 5 | Locate the $HTTP_PORTS definition | Ubuntu Workstation | Search for HTTP_PORTS in snort.conf | Line reads: portvar HTTP_PORTS [80,8080] or similar |
| 6 | Define a custom variable if needed | Ubuntu Workstation | var MY_SERVERS 10.0.0.0/8 | Custom variable is available for use in rules |
| 7 | Save snort.conf after any variable change | Ubuntu Workstation | Ctrl+O, Enter, Ctrl+X | File saved with updated variable definitions |
| 8 | Open the local rules file | Ubuntu Workstation | sudo nano /etc/snort/rules/local.rules | Rules file opens for editing |
| 9 | Write a rule using $HOME_NET as the source | Ubuntu Workstation | alert icmp $HOME_NET any -> any any (msg:"ICMP FROM INTERNAL"; sid:1000008;) | Rule targets traffic originating from internal addresses only |
| 10 | Write a rule using $EXTERNAL_NET as the source | Ubuntu Workstation | alert tcp $EXTERNAL_NET any -> $HOME_NET any (msg:"INBOUND TCP FROM EXTERNAL"; sid:1000009;) | Rule targets inbound traffic from external sources to internal addresses |
| 11 | Write a rule using $HTTP_PORTS as the destination port | Ubuntu Workstation | alert tcp any any -> any $HTTP_PORTS (msg:"HTTP PORT ACCESS"; sid:1000010;) | Rule targets traffic destined for defined HTTP ports |
| 12 | Save the rules file | Ubuntu Workstation | Ctrl+O, Enter, Ctrl+X | File saved with variable-based rules |
| 13 | Test Snort configuration syntax | Ubuntu Workstation | sudo snort -T -c /etc/snort/snort.conf | Snort resolves all variables and reports configuration OK |
| 14 | Run Snort and generate ICMP traffic from an internal host | Win_Station | ping <target_ip> | Alert fires because Win_Station is within $HOME_NET |
| 15 | Verify ICMP from an external source does not trigger a $HOME_NET rule | Ubuntu Workstation | Simulate traffic from outside $HOME_NET range | No alert for the $HOME_NET source rule; confirms variable is scoped correctly |

# Snort_Built_In_Variables_Skeleton

Rule using $HOME_NET as source
alert <protocol> $HOME_NET <src_port> -> any any (msg:"<LABEL>"; sid:<SID>;)

Rule using $EXTERNAL_NET as source targeting $HOME_NET
alert <protocol> $EXTERNAL_NET any -> $HOME_NET any (msg:"<LABEL>"; sid:<SID>;)

Rule using $HTTP_PORTS as destination port
alert tcp any any -> any $HTTP_PORTS (msg:"<LABEL>"; sid:<SID>;)

Rule using a custom variable
alert <protocol> $MY_SERVERS any -> any any (msg:"<LABEL>"; sid:<SID>;)

ICMP internal source lab example
alert icmp $HOME_NET any -> any any (msg:"ICMP PACKET DETECTED"; sid:1000011;)

Custom variable definition in snort.conf
var HOME_NET 192.168.1.0/24
var EXTERNAL_NET !$HOME_NET
var MY_SERVERS 10.0.0.0/8
portvar HTTP_PORTS [80,8080]

# Snort_Built_In_Variables_Failure_Checks

| Symptom | Command | What Usually Broke |
|---|---|---|
| Snort reports undefined variable error on startup | sudo snort -T -c /etc/snort/snort.conf | Variable referenced in a rule is not defined in snort.conf; add the var definition |
| Rule using $HOME_NET fires on external traffic | cat /var/log/snort/alert | $HOME_NET is defined too broadly in snort.conf; narrow the CIDR to the actual internal range |
| Rule using $EXTERNAL\_NET never fires | sudo snort -T -c /etc/snort/snort.conf | $EXTERNAL\_NET is set to any instead of !$HOME_NET; correct the definition in snort.conf |
| Custom variable has no effect in rules | Review rule and snort.conf | Variable name in the rule does not exactly match the var name defined in snort.conf; names are case-sensitive |
| $HTTP_PORTS rule misses traffic on port 8080 | cat /var/log/snort/alert | $HTTP_PORTS is defined as only port 80; update portvar HTTP_PORTS in snort.conf to include 8080 |
| Variable change in snort.conf has no effect | sudo systemctl status snort | Snort was not restarted after editing snort.conf; restart to apply changes |
