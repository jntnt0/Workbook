# ASAv_Access_Rules_Service_Definition_HTTP_Mental_Model

| Concept | Operational Meaning |
|---|---|
| Service Definition | The protocol and port that a firewall rule matches against — without it, the rule has no basis to permit or deny specific traffic types |
| Four Required Rule Components | Source, Destination, Service, Action — all four must be defined; a rule with any missing component will not function as intended |
| HTTP (TCP/80) | Hypertext Transfer Protocol — the service used by web browsers to communicate with web servers; TCP-based, destination port 80 |
| Rule with Missing Service | An access rule scoped to specific objects but lacking a defined service still hits the implicit deny for unmatched traffic — connection times out |
| ASDM Service Field | The Service column in an access rule accepts built-in service objects (HTTP, HTTPS, FTP, etc.) or custom service objects — "ip" means all protocols and should be replaced when restricting to a specific service |
| Stateful TCP Inspection | Once HTTP is permitted outbound, the ASAv tracks the TCP session and automatically allows the return traffic — no explicit inbound rule needed for the reply |

# ASAv_Access_Rules_Service_Definition_HTTP_Skeleton

## Completed Rule State After This Task

| Field | Value |
|---|---|
| Interface | Workstations |
| Action | Permit |
| Source | All_Workstations |
| Destination | All_Servers |
| Service | HTTP (TCP/80) |

## Equivalent CLI (for reference)

```
object network All_Workstations
 subnet 10.233.<Win_Station_octet>.0 255.255.254.0

object network All_Servers
 subnet 10.233.<IIS_Server_octet>.0 255.255.254.0

access-list Workstations_access_in extended permit tcp object All_Workstations object All_Servers eq 80
access-group Workstations_access_in in interface Workstations
```

# ASAv_Access_Rules_Service_Definition_HTTP_Failure_Checks

| Symptom | Likely Cause | Resolution |
|---|---|---|
| Still ERR_CONNECTION_TIMED_OUT after adding HTTP | Apply not clicked after editing the rule | Click Apply and retry |
| Page loads but shows IIS default page instead of "cyberbit" | Correct server reached but wrong virtual host or app not deployed | Confirm URL is http://IIS_Server exactly — no trailing slash variations |
| Service picker shows HTTP was added but rule still shows "ip" | "ip" was not removed and is overriding the specific service entry | Remove the "ip" entry from the service field so only HTTP remains |
| HTTP works but other services (RDP, SMB) still fail | Expected — only HTTP is permitted at this stage | Additional services must be added to the rule in subsequent tasks |
| Changes lost after navigating away | ASDM staged changes not applied before leaving the panel | Always click Apply before navigating away from Access Rules |
