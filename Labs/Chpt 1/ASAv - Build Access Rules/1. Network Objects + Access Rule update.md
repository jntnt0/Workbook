# ASAv_Access_Rules_Network_Objects_Mental_Model

| Concept | Operational Meaning |
|---|---|
| Network Object | A named IP address, subnet, or range used as a reusable reference inside access rules, NAT policies, and other ASDM constructs |
| Access Rule | A firewall policy entry that evaluates traffic against Source, Destination, Service, and Action — all four must be defined for the rule to function |
| Implicit Deny | The ASAv denies all traffic not matched by an explicit permit rule — no rule match means the packet is dropped silently |
| /23 Subnet (255.255.254.0) | Covers two adjacent /24 blocks; the Cyberbit lab environment uses this mask, not the default /24 — the third octet of the network address must be even |
| Source / Destination Scope | Changing a rule from "any/any" to specific network objects restricts what traffic the rule matches — if the object range is wrong or the service is missing, traffic hits the implicit deny |
| ASDM Apply Button | Changes staged in the ASDM console are not pushed to the ASAv until Apply is clicked — the running config is not updated until then |
| Running Config vs Startup Config | Applied changes go to RAM (running config) only — a reboot without saving wipes them; write memory / Ctrl+S commits to flash |

# ASAv_Access_Rules_Network_Objects_Skeleton

## Network Object Parameters Reference

| Object Name | Type | IP Address | Netmask |
|---|---|---|---|
| All_Workstations | Network | 10.233.\<Win_Station_octet\>.0 | 255.255.254.0 |
| All_Servers | Network | 10.233.\<IIS_Server_octet\>.0 | 255.255.254.0 |

## Equivalent CLI (for reference)

```
object network All_Workstations
 subnet 10.233.<Win_Station_octet>.0 255.255.254.0

object network All_Servers
 subnet 10.233.<IIS_Server_octet>.0 255.255.254.0

access-list Workstations_access_in extended permit ip object All_Workstations object All_Servers
access-group Workstations_access_in in interface Workstations
```

# ASAv_Access_Rules_Network_Objects_Failure_Checks

| Symptom                                               | Likely Cause                                                                       | Resolution                                                                           |
| ----------------------------------------------------- | ---------------------------------------------------------------------------------- | ------------------------------------------------------------------------------------ |
| ERR_CONNECTION_TIMED_OUT after rule update            | Service/protocol not defined in the rule — all four components required            | Add HTTP (TCP/80) to the rule's service field (covered in next task)                 |
| Objects not visible in rule Source/Destination picker | Objects not yet applied — only staged in ASDM                                      | Click Apply before editing the rule                                                  |
| Wrong subnet in object                                | Third octet copied from wrong host or /24 assumed instead of /23                   | Re-ping both hostnames, use the actual third octet, confirm netmask is 255.255.254.0 |
| Rule change has no effect after Apply                 | Changes not saved to flash — device previously rebooted without write memory       | Verify running config reflects changes; save with Ctrl+S after confirming correct    |
| IIS_Server ping resolves but HTTP times out           | Rule missing service definition or destination object range excludes IIS_Server IP | Verify IIS_Server IP falls within the All_Servers 10.233.XX.0/23 range               |
