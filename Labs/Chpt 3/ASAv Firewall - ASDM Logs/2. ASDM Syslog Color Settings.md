# ASDM_Syslog_Color_Settings_Mental_Model
| Concept | Operational Meaning |
|---|---|
| Severity levels 0-7 | The ASA uses eight severity levels; lower number = higher severity |
| Foreground color | The text color applied to a log entry row in the ASDM syslog panel |
| Background color | The row fill color applied behind the text for a given severity level |
| Emergency (0) default foreground | Dark red; the most visually urgent color, reserved for the highest severity |
| Color settings scope | Colors are a local ASDM display preference only; they do not affect what the ASA logs or transmits to external syslog servers |
| Right-click access | Color settings are not in the main ASDM menu; they are only accessible by right-clicking an existing log entry in the panel |
| Severity communicated visually | Both foreground and background color work together to make severity immediately recognizable without reading the severity number column |

# ASDM_Syslog_Color_Settings_Severity_Reference
| Level | Name | Default Foreground |
|---|---|---|
| 0 | Emergency | Dark red |
| 1 | Alert | Red |
| 2 | Critical | Orange |
| 3 | Error | Yellow |
| 4 | Warning | Standard/Black |
| 5 | Notification | Standard/Black |
| 6 | Informational | Standard/Black |
| 7 | Debugging | Standard/Black |

# ASDM_Syslog_Color_Settings_Configuration_Checklist
| Step | Task | Expected Result |
|---|---|---|
| 1 | Ensure ASDM logging is enabled and at least one log entry exists in the panel | Syslog panel on Home screen is populated |
| 2 | Right-click any log entry in the Latest ASDM Syslog Messages panel | Context menu appears |
| 3 | Click "Color Settings" from the context menu | Color Settings dialog opens |
| 4 | Locate the top row of the severity list (Emergency, level 0) | Emergency row is visible at the top |
| 5 | Read the foreground color column for Emergency (0) | Foreground color is Dark red |

# ASDM_Syslog_Color_Settings_Access_Color_Settings_Skeleton
ASDM GUI Path:
  Home screen
  -> Latest ASDM Syslog Messages panel
  -> Right-click any log entry
  -> Select: "Color Settings"

Color Settings Dialog Layout:
  Column 1: Severity level number (0-7)
  Column 2: Severity name (Emergency, Alert, Critical, etc.)
  Column 3: Foreground color
  Column 4: Background color

Target Row:
  Level 0 - Emergency
  Foreground: Dark red

Note: At least one log entry must exist in the panel before right-click
      options become available. If the panel is empty, generate traffic
      or re-enable logging first.

# ASDM_Syslog_Color_Settings_Verification_Skeleton
ASDM GUI Path:
  Home screen
  -> Latest ASDM Syslog Messages panel
  -> Right-click any log entry
  -> "Color Settings"
  -> Confirm Emergency (level 0) foreground = Dark red

Expected color assignments for top severity levels:
  0 - Emergency:  Foreground = Dark red
  1 - Alert:      Foreground = Red
  2 - Critical:   Foreground = Orange
  3 - Error:      Foreground = Yellow

Note: Color settings are display-only and local to the ASDM session.
      No CLI verification command exists for this; it is GUI-only.

# ASDM_Syslog_Color_Settings_Verification_Commands
| Task | Method | Expected Result |
|---|---|---|
| Access color settings | Right-click log entry -> "Color Settings" | Color Settings dialog opens |
| Confirm Emergency foreground | Read level 0 row foreground column | Dark red |
| Confirm Alert foreground | Read level 1 row foreground column | Red |
| Confirm Critical foreground | Read level 2 row foreground column | Orange |
| Confirm Error foreground | Read level 3 row foreground column | Yellow |
| Confirm settings are display-only | Check that no CLI command reflects color changes | No `show` command exposes ASDM color preferences |

# ASDM_Syslog_Color_Settings_Rollback
| Step | Task | Method | Expected Result |
|---|---|---|---|
| 1 | Open Color Settings | Right-click log entry -> "Color Settings" | Dialog opens |
| 2 | Identify any modified color entries | Compare current colors against default reference table | Deviations from default are visible |
| 3 | Reset modified entries to default | Select the modified row and restore the original color value | Row reflects default color |
| 4 | Click Ok to apply | Click Ok in the dialog | Dialog closes with restored settings |
| 5 | Confirm panel reflects restored colors | Observe log entry row colors in the panel | Colors match the default severity color scheme |

# ASDM_Syslog_Color_Settings_Failure_Checks
| Symptom | What to Check | What Usually Broke |
|---|---|---|
| Right-click produces no context menu | Confirm at least one log entry exists in the panel | Panel is empty; no entries to right-click; enable logging and generate traffic first |
| "Color Settings" option is missing from context menu | Confirm you are right-clicking a log entry row, not the panel header or empty space | Click target is wrong; right-click must land on an actual log entry |
| Color Settings dialog is empty | ASDM version or display rendering issue | Rare; close and reopen ASDM |
| Emergency row foreground does not show Dark red | Color was previously modified by another user or session | Reset the Emergency row to its default dark red foreground value |
| Color changes do not persist after ASDM restart | Color settings are session-local in some ASDM versions | Expected behavior in some environments; document the defaults rather than relying on persistence |
