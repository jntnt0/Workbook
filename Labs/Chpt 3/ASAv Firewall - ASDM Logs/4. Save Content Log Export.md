# ASDM_Save_Content_Log_Export_Mental_Model
| Concept | Operational Meaning |
|---|---|
| "Save Content" scope | Exports the entire contents of the Latest ASDM Syslog Messages panel, not just the entry that was right-clicked |
| Panel as a rolling buffer | The ASDM syslog panel holds a finite number of the most recent log entries; older entries scroll out as new ones arrive |
| Export is a point-in-time snapshot | The text file captures exactly what is in the panel at the moment Save Content is clicked; no new entries are added after export |
| Export does not include historical logs | Logs that have already scrolled out of the panel before export are not included in the file |
| Text file format | Exported file is plain text; each row in the panel becomes a line in the file including severity, timestamp, syslog ID, and description |
| Right-click target does not affect export scope | The entry you right-click is irrelevant to the export; the action always saves the full panel contents |
| Use case | Save Content is useful for capturing a session snapshot for offline review, documentation, or sharing without access to a dedicated syslog server |

# ASDM_Save_Content_Log_Export_Configuration_Checklist
| Step | Task | Expected Result |
|---|---|---|
| 1 | Ensure the ASDM syslog panel is populated with log entries | Latest ASDM Syslog Messages panel has visible entries |
| 2 | Right-click any log entry in the panel | Context menu appears |
| 3 | Click "Save Content" from the context menu | Save dialog opens |
| 4 | Name the file `logs.txt` | Filename field shows `logs.txt` |
| 5 | Navigate to the Desktop as the save location | Desktop is selected as the destination |
| 6 | Click Save | File is written to the Desktop |
| 7 | Navigate to the Desktop and double-click `logs.txt` | File opens in Notepad or default text editor |
| 8 | Examine the file contents | All entries from the ASDM syslog panel are present; not just the right-clicked entry |

# ASDM_Save_Content_Log_Export_Export_Skeleton
ASDM GUI Path:
  Home screen
  -> Latest ASDM Syslog Messages panel
  -> Right-click any log entry
  -> Select: "Save Content"
  -> Save As dialog:
       Filename:  logs.txt
       Location:  Desktop
  -> Click: Save

Open exported file:
  Desktop -> logs.txt -> double-click

Expected file contents:
  All log entries currently visible in the ASDM syslog panel
  Each entry includes: severity, timestamp, syslog ID, source, destination, description
  Scope: panel buffer only; no entries beyond what was visible at time of export

# ASDM_Save_Content_Log_Export_Verification_Skeleton
Verification steps:
  1. Open logs.txt from the Desktop
  2. Count or scroll through entries
  3. Confirm the file contains more than one entry
  4. Confirm the file matches what was visible in the ASDM panel at export time
  5. Confirm no entries exist beyond what was in the panel

Expected result:
  File contains the same "latest" logs that appeared on the Home screen
  File is NOT limited to the single right-clicked entry
  File does NOT contain historical logs beyond the panel buffer

CLI cross-reference (optional):
  show logging asdm
  Compares the active ASDM log buffer against the exported file contents

# ASDM_Save_Content_Log_Export_Verification_Commands
| Task | Method | Expected Result |
|---|---|---|
| Confirm file was created | Navigate to Desktop | `logs.txt` is present |
| Confirm file scope | Open `logs.txt` and count entries | Multiple entries matching the full panel contents |
| Confirm right-click entry is not the only entry | Compare file line count to panel entry count | Counts match; file contains all panel entries |
| Confirm no out-of-panel entries | Compare file contents to current panel | No additional entries appear in the file beyond what the panel showed |
| Cross-reference via CLI | SSH to ASAv -> `show logging asdm` | Active ASDM log buffer reflects same events as exported file |

# ASDM_Save_Content_Log_Export_Failure_Checks
| Symptom | What to Check | What Usually Broke |
|---|---|---|
| "Save Content" option is missing from context menu | Confirm you are right-clicking a log entry row, not the panel header or empty area | Click target is wrong; must right-click an actual log entry |
| Save dialog does not appear | Confirm ASDM has write permissions to the filesystem | Java security settings or permissions are blocking the file dialog |
| `logs.txt` does not appear on the Desktop | Confirm the save location was set to Desktop before clicking Save | File was saved to a different location; search for `logs.txt` on the local filesystem |
| File opens but is empty | Panel was cleared before Save Content was clicked | No entries were in the panel at time of export; re-enable logging, generate traffic, then export again |
| File contains only one entry | Expected behavior is misunderstood | Save Content always exports the full panel; if only one entry exists in the panel then only one entry will appear in the file |
| File contents do not match current panel | Export was taken at an earlier point in time | Expected; export is a snapshot; new entries generated after the export will not appear in the file |
| File is unreadable or garbled | Wrong application opened the file | Open with Notepad or any plain text editor |
