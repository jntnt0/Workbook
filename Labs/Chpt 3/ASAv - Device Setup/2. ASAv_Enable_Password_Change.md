ASAv_Enable_Password_Change_Mental_Model

| Concept | Operational Meaning |
|---|---|
| Enable password | Controls access to privileged EXEC mode in the CLI; entered after the `enable` command at the `>` prompt |
| User login credentials | Username and password used to authenticate into ASDM or SSH; entirely separate from the enable password |
| Privileged EXEC mode | The elevated CLI mode (`#` prompt) required to make configuration changes; gated by the enable password |
| Enable password vs user password | Changing the enable password has no effect on any user account passwords; ASDM login is unaffected |
| Default credentials | Cisco devices ship with known default credentials; best practice requires changing them on first configuration |
| ASDM Device Name/Password | The ASDM panel at Configuration > Device Setup > Device Name/Password manages both the hostname and the enable password |
| Change the privileged mode password checkbox | Must be checked before the enable password fields become active in ASDM; unchecked means no change is applied |

ASAv_Enable_Password_Change_Configuration_Checklist

| Step | Task | Device | Action | Expected Result |
|---:|---|---|---|---|
| 1 | Open ASDM | Win_Station | Launch ASDM from desktop shortcut; connect to `mgmt.asav.internal` as `admin` / `P@ssw0rd` | ASDM loads and displays the device dashboard |
| 2 | Navigate to Device Name/Password | ASDM | Configuration > Device Setup > Device Name/Password | Device Name/Password panel opens |
| 3 | Locate the Enable Password section | ASDM | Find the Enable Password section in the middle of the panel | Section is visible with a greyed-out checkbox |
| 4 | Enable the password change fields | ASDM | Check the box next to "Change the privileged mode password" | Password fields become active |
| 5 | Enter the new enable password | ASDM | Type `10203040` in the Password field | Field is populated |
| 6 | Confirm the new enable password | ASDM | Type `10203040` in the Confirm Password field | Both fields match |
| 7 | Apply the change | ASDM | Click Apply | Enable password is updated on the ASAv |
| 8 | Close ASDM | Win_Station | Close the ASDM window | ASDM session ends |
| 9 | Reopen ASDM | Win_Station | Launch ASDM from desktop shortcut | ASDM login prompt appears |
| 10 | Log in with original user credentials | Win_Station | Device: `mgmt.asav.internal` / Username: `admin` / Password: `P@ssw0rd` | Login succeeds; ASDM loads normally |

ASAv_Enable_Password_Change_Skeleton

    ASDM: Configuration > Device Setup > Device Name/Password
    Check: "Change the privileged mode password"
    Password:         10203040
    Confirm Password: 10203040
    Click Apply

    CLI equivalent:
    enable password 10203040

ASAv_Enable_Password_Change_Verification_Commands

    ASDM: Configuration > Device Setup > Device Name/Password
    Confirm enable password section shows the checkbox was applied

    CLI verification (from SSH session):
    > enable
    Password: 10203040
    Prompt changes to #

    User login verification:
    Reopen ASDM using admin / P@ssw0rd
    Successful login confirms user credentials are unchanged

ASAv_Enable_Password_Change_Rollback

| Step | Task | Device | Action | Expected Result |
|---:|---|---|---|---|
| 1 | Open ASDM | Win_Station | Launch ASDM; connect to `mgmt.asav.internal` as `admin` / `P@ssw0rd` | ASDM loads |
| 2 | Navigate to Device Name/Password | ASDM | Configuration > Device Setup > Device Name/Password | Panel opens |
| 3 | Enable the password change fields | ASDM | Check "Change the privileged mode password" | Password fields become active |
| 4 | Enter the previous enable password | ASDM | Type the original enable password in both fields | Fields are populated |
| 5 | Apply | ASDM | Click Apply | Enable password reverts to previous value |
| 6 | Verify rollback | Win_Station | Open SSH session; run `enable`; enter original password | Privileged mode is accessible with the original enable password |

ASAv_Enable_Password_Change_Failure_Checks

| Symptom | Likely Cause | Device | Command | Fix |
|---|---|---|---|---|
| Password fields remain greyed out | "Change the privileged mode password" checkbox not checked | ASDM | Device Name/Password panel | Check the checkbox before entering the password |
| ASDM login fails after enable password change | User login credentials confused with enable password | Win_Station | ASDM login prompt | Use `admin` / `P@ssw0rd` for ASDM login; enable password is CLI only |
| CLI `enable` command rejects new password | Change was not applied or Apply was not clicked | ASDM | Re-navigate to Device Name/Password | Reapply the change and confirm Apply completes without error |
| Apply returns an error | Password does not meet complexity requirements | ASDM | Error banner in ASDM | Use a password that meets the ASA minimum requirements |
| Enable password change lost after reload | Change was applied to running config only | ASAv CLI | `show running-config` vs `show startup-config` | Run `write memory` to persist the change to flash |
