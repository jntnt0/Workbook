ASAv_AAA_Server_Group_Configuration_Mental_Model

| Concept | Operational Meaning |
|---|---|
| AAA | Authentication, Authorization, and Accounting; the framework for controlling who can access the device, what they can do, and logging what they did |
| Authentication | Verifies the identity of a user attempting to access the device |
| Authorization | Defines what an authenticated user is permitted to do once logged in |
| Accounting | Records user activity for audit and compliance purposes |
| AAA Server Group | A named container that groups one or more AAA servers using the same protocol; the ASA sends authentication requests to the group |
| AAA access tree | The full configuration chain: create a server group, add servers to the group, assign the group to an AAA access policy |
| TACACS+ | Terminal Access Controller Access Control System Plus; Cisco-preferred protocol; separates authentication, authorization, and accounting; uses TCP port 49 |
| RADIUS | Remote Authentication Dial-In User Service; combines authentication and authorization; uses UDP ports 1812 and 1813; most widely deployed AAA protocol |
| LDAP | Lightweight Directory Access Protocol; integrates with directory services such as Microsoft Active Directory for user authentication |
| HTTP Form | Used for clientless SSL VPN web-based authentication; presents a web login form to the user |
| OAUTH | Open Authorization; a token-based delegated access standard; not supported as a Cisco ASA AAA Server Group protocol |
| SAML | Security Assertion Markup Language; federated identity and SSO standard; not available as an ASA AAA Server Group protocol option |
| Local vs external AAA | Local uses accounts stored on the ASA itself; external AAA offloads authentication to a dedicated server group |

ASAv_AAA_Server_Group_Configuration_Checklist

| Step | Task | Device | Action | Expected Result |
|---:|---|---|---|---|
| 1 | Open ASDM | Win_Station | Launch ASDM from desktop shortcut; connect to `mgmt.asav.internal` as `admin` / `P@ssw0rd` | ASDM loads and displays the device dashboard |
| 2 | Navigate to AAA Server Groups | ASDM | Configuration > Device Management > Users/AAA > AAA Server Groups | AAA Server Groups panel opens; existing groups are listed |
| 3 | Open new server group dialog | ASDM | Click Add in the AAA Server Groups section | New AAA Server Group dialog opens |
| 4 | Assign a group name | ASDM | Enter a descriptive name for the server group | Name field is populated |
| 5 | Examine available protocols | ASDM | Click the Protocol dropdown | Available options: TACACS+, RADIUS, LDAP, HTTP Form |
| 6 | Note unavailable protocols | ASDM | Confirm OAUTH and SAML are not present in the list | OAUTH and SAML are not supported as ASA AAA Server Group protocols |
| 7 | Select the required protocol | ASDM | Choose the appropriate protocol for the environment | Protocol is selected; server group is configured for that protocol |
| 8 | Confirm and close dialog | ASDM | Click OK | New server group appears in the AAA Server Groups list |
| 9 | Add a server to the group | ASDM | With the new group selected, click Add in the Servers in Selected Group section | Server entry dialog opens |
| 10 | Configure the server entry | ASDM | Enter the server IP address and any required parameters for the selected protocol | Server is associated with the group |
| 11 | Apply the configuration | ASDM | Click OK, then click Apply | AAA server group and server entry are saved to the ASAv |
| 12 | Assign the group to an AAA access policy | ASDM | Navigate to the relevant AAA access policy and reference the new server group | Server group is active in the AAA access tree |

ASAv_AAA_Server_Group_Configuration_Skeleton

    ASDM: Configuration > Device Management > Users/AAA > AAA Server Groups
    Click: Add
      Group Name: <group_name>
      Protocol:   <TACACS+ | RADIUS | LDAP | HTTP Form>
    Click: OK

    With group selected, in Servers in Selected Group section:
    Click: Add
      Interface Name: <interface>
      Server Name or IP Address: <server_ip>
    Click: OK
    Click: Apply

    CLI equivalent:
    aaa-server <group_name> protocol <tacacs+ | radius | ldap>
    aaa-server <group_name> (<interface>) host <server_ip>
    write memory

ASAv_AAA_Server_Group_Configuration_Verification_Commands

    ASDM: Configuration > Device Management > Users/AAA > AAA Server Groups
    Confirm group appears with correct protocol and server entries

    CLI verification (from SSH session):
    show running-config aaa-server
    Confirm group name, protocol, interface, and server IP are present

    Test AAA server reachability:
    test aaa-server authentication <group_name> host <server_ip> username <user> password <pass>
    Successful result confirms the ASA can reach and query the AAA server

ASAv_AAA_Server_Group_Configuration_Rollback

| Step | Task | Device | Action | Expected Result |
|---:|---|---|---|---|
| 1 | Open ASDM | Win_Station | Launch ASDM; connect to `mgmt.asav.internal` | ASDM loads |
| 2 | Navigate to AAA Server Groups | ASDM | Configuration > Device Management > Users/AAA > AAA Server Groups | AAA Server Groups panel opens |
| 3 | Remove any AAA access policy references | ASDM | Navigate to the relevant AAA access policy; remove the reference to the server group | Group is no longer referenced by any policy |
| 4 | Select the server group to remove | ASDM | Click the server group entry in the list | Group is highlighted |
| 5 | Delete the server group | ASDM | Click Delete | Server group and all associated server entries are removed |
| 6 | Apply | ASDM | Click Apply | Configuration is pushed to the ASAv |
| 7 | Verify rollback | ASDM | Confirm the group no longer appears in the AAA Server Groups panel | AAA server group is removed |

ASAv_AAA_Server_Group_Configuration_Failure_Checks

| Symptom                                           | Likely Cause                                                    | Device   | Command                                                        | Fix                                                             |
| ------------------------------------------------- | --------------------------------------------------------------- | -------- | -------------------------------------------------------------- | --------------------------------------------------------------- |
| OAUTH or SAML not found in protocol list          | Neither protocol is supported as an ASA AAA Server Group option | ASDM     | Protocol dropdown in Add Server Group dialog                   | Use TACACS+, RADIUS, LDAP, or HTTP Form as appropriate          |
| AAA server group created but authentication fails | Server IP unreachable or shared secret mismatch                 | ASAv CLI | `test aaa-server authentication <group_name> host <server_ip>` | Verify server IP, interface, and shared secret are correct      |
| Server group exists but is not being used         | Group not assigned to an AAA access policy                      | ASDM     | Review AAA access policy configuration                         | Reference the server group in the appropriate AAA access policy |
| Cannot delete server group                        | Group is still referenced by an active AAA access policy        | ASDM     | Review AAA access policies for references to the group         | Remove all policy references before deleting the group          |
| Protocol mismatch between ASA and server          | Wrong protocol selected for the server type                     | ASAv CLI | `show running-config aaa-server`                               | Delete the group and recreate with the correct protocol         |
| AAA configuration lost after reload               | Change applied to running config only                           | ASAv CLI | `show running-config aaa-server`                               | Run `write memory` to persist to flash                          |
