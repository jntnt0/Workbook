# Lab 5 — Identifying the Attacker's C&C IP Address

## Tab 1

Once malware establishes itself on a host, it needs to phone home — communicating outbound to the attacker's Command and Control server to receive instructions and exfiltrate data. This outbound connection is logged as a network event with a `DestinationIp` field. Pivoting on the malicious process and inspecting its outbound connections reveals the attacker's infrastructure directly.

The goal of this task is to identify the IP address of the attacker's C&C server by examining outbound network connections initiated by `GoogleUpdate.exe`.

---

### Step-by-Step Thought Process

**Search for the malicious Image.**  
Anchor the search to the full Image path of the confirmed malware. Network connection events tied to this process will appear in the results alongside process execution events.

**Locate the DestinationIp field.**  
In the results set, find the `DestinationIp` field. This field captures the remote IP address the malicious process attempted to reach. That destination is the attacker's C&C server.

**Eliminate the wrong answers by analysis.**  
Not every IP in the results is attacker infrastructure. Evaluate each candidate:

- **239.255.255.250** — This is a reserved multicast address used by SSDP and UPnP for local network discovery. It is not routable to the internet and cannot be a C&C server.
- **172.16.100.4** — This falls in the DMZ segment (172.16.100.X) visible in the network diagram. It is an internal address — not attacker-controlled external infrastructure.
- **199.203.100.175** — Internet segment address but not the destination recorded in the outbound connection log.
- **199.203.100.79** — Internet segment (199.203.100.X), externally routable, and confirmed as the `DestinationIp` value in the GoogleUpdate.exe network connection event. This is the C&C server.

---

### Expected Splunk Query

```
source=/home/splunk/data.csv Image="C:\\Users\\User067\\Downloads\\GoogleUpdate.exe"
```
Then locate and inspect the `DestinationIp` field in the results.

---

### Final Query (Clean)

```
source=/home/splunk/data.csv Image="C:\\Users\\User067\\Downloads\\GoogleUpdate.exe"
| table Image DestinationIp CommandLine
```

---

### Findings

**IP Elimination Table:**

| IP Address | Segment | Verdict |
|---|---|---|
| 239.255.255.250 | Multicast (SSDP/UPnP) | Not attacker — reserved local multicast, not internet routable |
| 172.16.100.4 | DMZ — 172.16.100.X | Not attacker — internal network address |
| 199.203.100.175 | Internet — 199.203.100.X | Not attacker — not present in DestinationIp field |
| **199.203.100.79** | **Internet — 199.203.100.X** | **Confirmed C&C server — recorded DestinationIp of GoogleUpdate.exe outbound connection** |

**Answer:** 199.203.100.79. This is the attacker's C&C server, sitting in the Internet segment and confirmed by the outbound network connection logged against the malicious GoogleUpdate.exe Image path.
