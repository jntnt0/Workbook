# Lab 6 — Why Attackers Target the Windows Temp Folder

## Tab 1

Malware needs somewhere to land. When a payload is delivered via a user-level process — like a file downloaded through a browser — it executes with the permissions of the logged-in user, not an administrator. That means it cannot write to protected directories like `C:\Program Files\` or `C:\Windows\System32\` without triggering a UAC prompt. The Windows temp folder solves that problem entirely. It is writable by any logged-in user with no elevation required.

The goal of this task is to understand why `GoogleUpdate.exe` was stored and executed from a user-accessible directory, and why the Windows temp folder is a standard staging ground for malicious payloads.

---

### Step-by-Step Thought Process

**Understand the permission model.**  
The Windows temp folder (`C:\Users\<Username>\AppData\Local\Temp`) inherits the read and write permissions of the currently logged-in user. No admin rights, no UAC prompt, no elevation required. A payload delivered through any user-level process — browser, email client, document macro — can write directly to this location and execute from it without any additional privilege negotiation.

**Evaluate the "OS deletes temp files" claim.**  
This is the most common misconception about the temp folder. Windows does not automatically or reliably purge temp directory contents. Cleanup requires deliberate user action or a scheduled task — neither of which is guaranteed. Attackers cannot count on the OS covering their tracks, and this is not why they choose the temp folder.

**Evaluate the RAMDISK claim.**  
The Windows temp folder resides on the standard system disk — typically `C:\`. It is not mounted on a RAMDISK. Write speed is identical to any other directory on the same volume. This option is simply false.

**Connect to the investigation.**  
In this attack, the payload landed in `C:\Users\User067\Downloads\` — a user-accessible directory operating on the same principle as the temp folder. No admin rights were needed. Chrome downloaded the file, the user's session permissions allowed it to be written and executed, and the attack proceeded without a single UAC prompt.

**Why the temp folder specifically in other attacks.**  
Downloads folders are visible and reviewed. The temp folder is buried in AppData — most users never navigate there. Combined with the no-elevation-required write access, it is the preferred staging location for malware that needs to operate quietly under a standard user context.
