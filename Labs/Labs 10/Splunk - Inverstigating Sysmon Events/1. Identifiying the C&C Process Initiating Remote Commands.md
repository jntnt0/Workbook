# Lab 1 — Identifying the C&C Process Initiating Remote Commands

## Tab 1

In a Command and Control (C&C) attack, an attacker uses a compromised host to execute commands remotely — often by spawning CMD.exe as a child process from a malicious parent. The malware itself is the parent; CMD.exe is just the tool it uses. Standard endpoint security may not flag this because the individual commands (like `net users /domain`) are legitimate Windows utilities. Threat hunting requires you to look one level up — at *who* called the command, not just what the command was.

The goal of this task is to identify the malicious parent process responsible for initiating the `net users /domain` command on the compromised host.

---

### Step-by-Step Thought Process

**Set the data source.**  
All log data for this investigation lives in a CSV file on the Splunk host. You must scope the search to that source before anything else, or you'll be searching across unrelated indexes.

**Search for the suspicious command.**  
The attacker used `net users /domain` to enumerate domain accounts. Searching for this string in the search pane surfaces all events where this command appears in any field.

**Filter down to the exact execution pattern.**  
The malware didn't call `net users /domain` directly — it called it through CMD.exe using the `/c` flag. Filtering the `CommandLine` field to `cmd.exe /c  net users /domain` isolates only the relevant process executions and eliminates noise.

**Trace to the parent.**  
Once filtered, expand any result and inspect the `ParentCommandLine` field. This field reveals the process that spawned CMD.exe — that process is the malware acting as the C&C agent.

---

### Expected Splunk Query

```
source=/home/splunk/data.csv "net users /domain"
```
Then filter `CommandLine` field by:
```
cmd.exe /c  net users /domain
```
Then inspect the `ParentCommandLine` field of any result.

---

### Final Query (Clean)

```
source=/home/splunk/data.csv "net users /domain"
| search CommandLine="cmd.exe /c  net users /domain"
| table ParentCommandLine CommandLine
```

---

### Findings

**Answer:** `GoogleUpdate.exe`  
The `ParentCommandLine` field reveals `C:\Users\User067\Downloads\GoogleUpdate.exe` as the process that spawned CMD.exe to run `net users /domain`. This is the attacker's C&C malware, disguised as a legitimate Google updater binary to avoid detection.
