# Splunk_Suspicious_Traffic_Destination_Port_Mental_Model

| Concept | Operational Meaning |
|---|---|
| `dest_port` field | The port on the destination host that traffic was directed to — identifies the service or protocol in use |
| `stats count by dest_port` | Aggregates log events by port, showing which port carried the most connections for the filtered pair |
| `sort -count` | Surfaces the dominant port first — the port with the most connections is the primary communication channel |
| Port 80 (HTTP) | Standard unencrypted web traffic — commonly abused for exfiltration because it blends with normal web traffic and is rarely blocked outbound |
| Port 80 vs Port 53 distinction | Port 53 (DNS) tunneling is a known exfiltration technique but produces a different traffic signature; port 80 exfiltration leverages HTTP to move data in plain sight |
| High-volume port 80 to unknown external IP | A large sustained data transfer over HTTP to an IP with no legitimate network map presence is a strong exfiltration indicator |

# Splunk_Suspicious_Traffic_Destination_Port_Investigation_Checklist

| Step | Action | Notes |
|---|---|---|
| 1 | Confirm time range is set to **All Time** | Consistent with prior tasks |
| 2 | Run targeted query filtering on confirmed suspicious pair | Scope query to `src_ip=172.16.100.21 dest_ip=199.203.100.86` |
| 3 | Add `dest_port` to stats aggregation | Use `stats count by dest_port` to enumerate all ports used |
| 4 | Sort descending by count | The dominant port surfaces first |
| 5 | Record the top `dest_port` value | This is the primary communication channel between the pair |
| 6 | Contextualize the port | Identify the associated protocol and assess whether it is consistent with the exfiltration pattern |
| 7 | Confirm port 80 (HTTP) | No encryption, blends with normal web traffic — consistent with covert outbound data transfer |

# Splunk_Suspicious_Traffic_Destination_Port_Verification_Steps

| Check | Expected Result |
|---|---|
| Query returns populated table | `dest_port` and `count` columns present |
| Top result is the dominant port | Port **80** surfaces with the highest connection count |
| Port 80 contextualized | HTTP — unencrypted, commonly allowed outbound, consistent with covert data exfiltration |
| No encryption indicator | Port 443 (HTTPS) not the dominant port — traffic is plaintext HTTP |

# Splunk_Suspicious_Traffic_Destination_Port_Failure_Checks

| Symptom | Likely Cause | Fix |
|---|---|---|
| No results returned | Time range not All Time, or src/dest IP typo | Reset time range; verify IP values match exactly from Task 1 output |
| Multiple ports returned with similar counts | Pair uses several ports — check top result only for dominant channel | Focus on highest count entry |
| Query syntax error after paste | Encoding issue with quotes or equals signs | Manually retype affected characters in Splunk search bar |
| Port 53 assumed without querying | Do not infer the port from attack type alone | Always run the query — DNS tunneling and HTTP exfiltration produce similar byte volumes but different port signatures |
