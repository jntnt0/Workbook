# Splunk_Malicious_Activity_Classification_Mental_Model

| Concept | Operational Meaning |
|---|---|
| Data Exfiltration | Unauthorized transfer of data from inside a network to an external destination — characterized by high outbound volume, sustained connections, and an internal source targeting an external unknown IP |
| Network Sniffing | Passive interception of traffic — does not generate outbound log entries or high byte counts; a listener produces no initiating connection logs |
| Man-in-the-Middle (MitM) | Traffic interception via proxy positioning — does not itself generate high-volume outbound logs to an external IP; evidence would be ARP/certificate anomalies, not sustained HTTP transfers |
| SQL Injection | Application-layer attack targeting a database — generates few connection log entries; high data volume is not a primary characteristic; destination would be an internal DB, not an external IP |
| Internal-to-external direction | Traffic originating from inside the network destined for an unknown external IP is the foundational exfiltration pattern |
| Log volume as a classifier | High log entry counts combined with high byte volumes rule out passive attacks and point toward active, sustained data movement |
| Port 80 as exfiltration vehicle | HTTP is used to blend exfiltration into normal web traffic — commonly allowed through firewalls outbound, making it a preferred covert channel |

# Splunk_Malicious_Activity_Classification_Investigation_Checklist

| Step | Action | Notes |
|---|---|---|
| 1 | Aggregate all evidence from prior tasks | Connection count, byte volume, directional skew, destination port, IP pair identity |
| 2 | Confirm traffic direction | Source is internal (`172.16.100.21` — DMZ segment); destination is external (`199.203.100.86`) |
| 3 | Eliminate Network Sniffing | Passive attack — produces no outbound logs or byte transfer to external IPs |
| 4 | Eliminate MitM | Requires positioning between two hosts — does not produce sustained high-volume outbound HTTP to unknown external IP |
| 5 | Eliminate SQL Injection | Targets internal database services — generates few log entries; destination would be internal, not external |
| 6 | Confirm Data Exfiltration | All indicators align: internal source, external unknown destination, high connection count, ~85MB outbound, port 80 HTTP transport |
| 7 | Document the full evidence chain | Connection count + byte volume + port + traffic direction = Data Exfiltration classification |

# Splunk_Malicious_Activity_Classification_Attack_Elimination_Skeleton

Reference table for elimination-based classification at any investigation:

| Attack Type | Log Volume | Byte Volume | Traffic Direction | Typical Destination | Eliminated By |
|---|---|---|---|---|---|
| Network Sniffing | None / minimal | None | N/A — passive | Local segment | No outbound logs generated |
| MitM | Low — proxy artifacts | Low-moderate | Lateral / in-path | Between two internal hosts | No sustained external outbound transfer |
| SQL Injection | Low | Low-moderate | Inbound to DB | Internal database server | Few log entries; internal destination |
| **Data Exfiltration** | **High** | **High — outbound skewed** | **Internal → External** | **Unknown external IP** | **Not eliminated — all indicators match** |

# Splunk_Malicious_Activity_Classification_Failure_Checks

| Symptom | Likely Cause | Fix |
|---|---|---|
| Misclassified as SQL Injection | Focusing on log count alone without considering destination IP type | Confirm destination is external — SQL Injection targets internal DB servers |
| Misclassified as MitM | Assuming any interception-style attack fits high traffic | MitM does not produce high-volume outbound logs to unknown external IPs |
| Misclassified as Network Sniffing | Overlooking that sniffing is passive | Sniffing generates no initiating connection logs or outbound byte transfers |
| Port 53 assumed as DNS exfiltration | Inferring attack type from port assumption rather than query output | Always confirm port from Task 3 query before classifying; port 80 HTTP exfiltration is equally common |
