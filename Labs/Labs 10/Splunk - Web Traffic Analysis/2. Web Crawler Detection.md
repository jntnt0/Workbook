# Splunk SIEM – Web Crawler Detection

## Overview

Web crawlers systematically browse websites by making large numbers of requests across
many different pages. While legitimate crawlers (search engine bots) are common, malicious
crawlers are used for reconnaissance, content scraping, and vulnerability mapping.

A key challenge in detecting crawlers via logs is that the `URI` field often includes query
parameters (e.g., `/page?id=5&user=admin`). Counting raw URI values would treat the same
page with different parameters as different pages — inflating the unique page count and
producing false negatives. The resource path must be isolated before counting.

---

## Step-by-Step Lab

### 1. Understand the Detection Criteria

A **web crawl** is defined here as:
- **100 or more requests** made to **distinct pages** (not just distinct URIs)
- By a **single User-Agent**
- That is **not Mozilla-based** (excluded because Mozilla is shared across millions of
  legitimate users, making per-agent attribution unreliable)
