# ASDM_Logging_And_Syslog_Severity_Analysis_Mental_Model

| Concept | Operational Meaning |
|---|---|
| ASDM logging | Real-time syslog display within the ASDM interface; must be explicitly enabled per session |
| Syslog severity levels | Numbered 0 to 7; lower number = higher severity; 0 is emergency, 7 is debugging |
| Severity 4 (Warning) | Indicates a condition requiring attention; ACL deny events are logged at this level |
| Severity 6 (Informational) | Routine operational events; established and torn-down connections are logged here |
| %ASA-4-106023 | The specific syslog message ID for a packet denied by an ACL; severity 4 |
| %ASA-6-302013 | TCP connection built; severity 6 informational |
| %ASA-6-302014 | TCP connection torn down; severity 6 informational |
| TCP 1433 | Default SQL Server port; no access rule permits this from the Workstations network in this lab |
| Implicit deny behavior | The ASA drops and logs any traffic not explicitly permitted; generates a 106023 at severity 4 |
| Test-NetConnection | PowerShell cmdlet that tests TCP reachability; TcpTestSucceeded: False confirms the port is blocked |
| Severity column in ASDM | The first column in the ASDM syslog viewer; displays the numeric severity level of each message |
| Why not severity 6 | Severity 6 requires a connection to be established; a denied packet never reaches that state |

# ASDM_Logging_And_Syslog_Severity_Analysis_Configuration_Checklist

| Step | Task | Where | Action | Expected Result |
|---:|---|---|---|---|
| 1 | Enable ASDM logging | ASDM Home screen | Scroll to bottom → click "Enable ASDM Logging" | Syslog viewer begins populating with live messages |
| 2 | Open PowerShell | Win_Station desktop | Open Windows PowerShell | PowerShell prompt displayed |
| 3 | Test TCP 1433 connectivity | PowerShell | Run Test-NetConnection -ComputerName iis_server -Port 1433 | TcpTestSucceeded: False returned |
| 4 | Examine syslog messages | ASDM syslog viewer | Scroll through log entries; filter or search for 1433 | 106023 deny message for TCP 1433 visible |
| 5 | Note severity level | Severity column (first column) | Read severity value on the 1433-related log entry | Severity 4 (Warning) confirmed |
| 6 | Note syslog message ID | Message column | Read the message ID | %ASA-4-106023 confirmed |

# ASDM_Logging_And_Syslog_Severity_Analysis_Skeleton

```
! Enable logging to ASDM buffer via CLI (equivalent to ASDM button):
logging enable
logging asdm informational

! To view live syslog from CLI:
show logging

! To filter log for specific port activity:
show logging | include 1433

! Expected syslog output for a denied TCP 1433 connection:
! %ASA-4-106023: Deny tcp src Workstations:x.x.x.x/xxxxx
!   dst Servers:x.x.x.x/1433 by access-group "Workstations_access_in"
```

# ASDM_Logging_And_Syslog_Severity_Analysis_Failure_Checks

| Symptom | Likely Cause | Check | Fix |
|---|---|---|---|
| No syslog messages appear | ASDM logging not enabled | Check ASDM Home screen logging status | Click Enable ASDM Logging |
| No 1433 entry in syslog | Test-NetConnection not yet run | Confirm PowerShell command was executed | Run Test-NetConnection -ComputerName iis_server -Port 1433 |
| Syslog shows severity 6 for 1433 | A permit rule for TCP 1433 exists on the Workstations interface | Check Access Rules for TCP 1433 entry | Remove unintended rule if present |
| TcpTestSucceeded: True | An access rule permits TCP 1433 | Check Access Rules list | Remove unintended TCP 1433 permit rule |
| Cannot find 1433 entry in log | Log buffer too large; entry scrolled out of view | Use ASDM log filter or search for 1433 | Apply port or message ID filter in syslog viewer |
| 106023 message ID not recognized | Unfamiliar with ASA syslog message IDs | Cross-reference severity column value | Severity 4 + deny language in message body confirms 106023 |
