# ASDM_Interface_Subnet_Mask_Inspection_Mental_Model

| Concept | Operational Meaning |
|---|---|
| Interface settings | The ASA assigns each interface an IP address and subnet mask defining its network segment |
| Subnet mask | Determines which portion of an IP address identifies the network vs the host |
| /23 (255.255.254.0) | A subnet spanning two consecutive Class C blocks; provides 512 addresses per segment |
| /24 (255.255.255.0) | The most common lab default; only 256 addresses per segment |
| Why /23 matters | The Cyberbit lab environment uses /23, not /24; assuming /24 will produce incorrect network calculations |
| Interface segments in this lab | Workstations network, Servers network, and management interface all use /23 masks |
| ASDM interface view | Displays interface name, IP address, subnet mask, enabled state, and security level in a single table |
| Do not assume defaults | Always inspect the actual interface configuration; lab environments frequently deviate from textbook defaults |

# ASDM_Interface_Subnet_Mask_Inspection_Configuration_Checklist

| Step | Task | Where | Action | Expected Result |
|---:|---|---|---|---|
| 1 | Open interface settings | Configuration → Device Setup → Interface Settings → Interfaces | Navigate to path | Interface list displayed showing all ASAv interfaces |
| 2 | Locate subnet mask column | Interface table | Read the mask value for each interface | All interfaces show 255.255.254.0 (/23) |
| 3 | Note interface names | Interface table | Record Name, IP Address, and Subnet Mask for each interface | Workstations, Servers, and management interfaces confirmed |

# ASDM_Interface_Subnet_Mask_Inspection_Verification

| Check | Method | Expected Result |
|---|---|---|
| Subnet mask via ASDM | Configuration → Device Setup → Interface Settings → Interfaces | All interfaces display 255.255.254.0 |
| Subnet mask via CLI | show interface ip brief | All interface masks confirm /23 |
| Prefix length equivalent | Manual conversion | 255.255.254.0 = /23; 22 bits network, 9 bits host, 512 total addresses |
| Usable host range per /23 | Manual calculation | 510 usable host addresses per segment |

# ASDM_Interface_Subnet_Mask_Inspection_Failure_Checks

| Symptom | Likely Cause | Check | Fix |
|---|---|---|---|
| ACL or NAT rule misses expected hosts | Subnet mask assumed as /24 instead of actual /23 | Inspect interface settings and confirm mask | Recalculate network range using 255.255.254.0 |
| Interface not visible in ASDM list | Interface disabled or not configured | Check enabled column in interface table | Enable the interface and assign IP/mask |
| Mask displays differently than expected | Interface configured with prefix notation vs dotted decimal | Cross-check show running-config interface | Both notations represent the same mask; /23 = 255.255.254.0 |
| Wrong subnet used in access rule source/destination | Default /24 assumption applied during rule creation | Review network objects used in rules | Update network objects to reflect /23 boundary |
