# ASDM_SSH_Access_Via_Startup_Wizard_Mental_Model

| Concept | Operational Meaning |
|---|---|
| Startup Wizard | A guided multi-step configuration tool in ASDM covering core ASAv settings in sequence |
| Administrative Access step | Step 8 of the Startup Wizard; defines which hosts and protocols can access the ASA management plane |
| ASDM (HTTPS) access | The existing entry in step 8 at the start of this task; permits HTTPS-based GUI access to management |
| SSH access | CLI-based encrypted remote access; must be explicitly permitted as a separate entry alongside ASDM |
| 0.0.0.0/0.0.0.0 | Permits access from any IP address; mirrors the scope of the existing ASDM entry |
| Management interface | The dedicated out-of-band interface used for administrative access; separate from Workstations and Servers |
| Mirroring the existing entry | The new SSH entry uses the same interface and IP range as the existing ASDM entry; only the Access Type changes |
| SSH vs ASDM access | ASDM uses HTTPS (TCP 443); SSH uses TCP 22; both must be independently permitted |
| Effect of this change | After applying, the ASA accepts SSH connections on the management interface from any source address |

# ASDM_SSH_Access_Via_Startup_Wizard_Configuration_Checklist

| Step | Task | Where | Action | Expected Result |
|---:|---|---|---|---|
| 1 | Launch Startup Wizard | Configuration → Device Setup → Startup Wizard | Click Launch Startup Wizard | Wizard opens at step 1 |
| 2 | Advance to Administrative Access | Wizard navigation | Click Next through steps 1 to 7 | Step 8 (Administrative Access) displayed |
| 3 | Review existing entry | Administrative Access table | Note existing entry: ASDM/HTTPS, management interface, 0.0.0.0/0.0.0.0 | Existing ASDM entry confirmed |
| 4 | Add new access entry | Administrative Access table | Click Add | New access entry dialog opens |
| 5 | Set Access Type | Access Type dropdown | Select "SSH" | Access Type set to SSH |
| 6 | Set interface | Interface dropdown | Select "management" | Interface set to management |
| 7 | Set IP address | IP Address field | Enter "0.0.0.0" | IP field shows 0.0.0.0 |
| 8 | Set subnet mask | Mask field | Enter "0.0.0.0" | Mask field shows 0.0.0.0 (any source) |
| 9 | Save entry | Dialog | Click Ok | New SSH entry appears in Administrative Access table alongside existing ASDM entry |
| 10 | Complete wizard | Wizard navigation | Click Next through remaining steps → Finish | Configuration applied to ASA |
| 11 | Test SSH access | Win_Station → SSH client or terminal | Connect to mgmt.asav.internal via SSH as admin | SSH session established; ASA CLI prompt displayed |

# ASDM_SSH_Access_Via_Startup_Wizard_Verification

| Check | Method | Expected Result |
|---|---|---|
| SSH entry in Startup Wizard | Step 8 Administrative Access table | SSH entry listed alongside ASDM entry; management interface, 0.0.0.0/0.0.0.0 |
| SSH entry via CLI | show running-config ssh | ssh 0.0.0.0 0.0.0.0 management present in output |
| SSH connection succeeds | Connect to mgmt.asav.internal via SSH | ASA CLI prompt returned; login with admin / P@ssw0rd |
| Privilege escalation path | At CLI prompt | > then enable (password: P@ssw0rd) → # then configure terminal → (config)# |

# ASDM_SSH_Access_Via_Startup_Wizard_Rollback

| Step | Action |
|---|---|
| Remove SSH entry via wizard | Return to Startup Wizard → step 8 → select SSH entry → Delete → complete wizard |
| Remove SSH entry via CLI | no ssh 0.0.0.0 0.0.0.0 management |
| Confirm ASDM entry preserved | Verify HTTPS entry still present in step 8 after removing SSH entry |

# ASDM_SSH_Access_Via_Startup_Wizard_Failure_Checks

| Symptom | Likely Cause | Check | Fix |
|---|---|---|---|
| SSH connection refused | SSH entry not added or not applied | show running-config ssh | Add entry via wizard or CLI and apply |
| SSH connection refused | Wrong interface specified | Confirm interface is "management" not "Workstations" or "Servers" | Edit entry; set interface to management |
| SSH connects but login fails | Wrong credentials | Confirm username and password | Use admin / P@ssw0rd |
| SSH connects but cannot enter enable mode | Enable password not set or different | At > prompt, type enable and supply password | Use P@ssw0rd as enable password |
| ASDM access lost after wizard | HTTPS entry accidentally deleted in step 8 | Check http entry in running config | Re-add ASDM entry: http 0.0.0.0 0.0.0.0 management |
| Wizard step 8 not reached | Clicked Finish early or skipped steps | Restart wizard and click Next sequentially | Navigate through all steps to reach step 8 |
