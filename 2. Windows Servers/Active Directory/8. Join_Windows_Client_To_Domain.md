# Join_Windows_Client_To_Domain_Mental_Model
| Concept | Operational Meaning |
|---|---|
| Domain join | Adds the Windows computer to Active Directory and creates or uses a computer account |
| Computer account | AD object representing the joined machine, usually named `<computer-name>$` internally |
| AD DNS | DNS service that exposes the domain and domain controller locator records |
| DC locator | Process where a client finds a domain controller using DNS SRV records |
| LDAP SRV records | DNS records clients use to locate domain controllers for directory operations |
| Kerberos SRV records | DNS records clients use to locate Kerberos authentication services |
| Secure channel | Trust relationship between the domain-joined computer and the domain |
| OUPath | Distinguished name of the OU where the computer account should be created |
| Join credential | Domain account authorized to join computers to the domain |
| Local credential | Local admin credential used when joining remote machines |
| Restart | Required for the computer to fully apply domain membership |
| Workgroup rollback | Removing the client from the domain by joining it back to a workgroup |
| First rule | If the client cannot resolve `_ldap._tcp.dc._msdcs.<domain>`, do not attempt the join yet |

# Join_Windows_Client_To_Domain_PreJoin_DNS_Skeleton
```powershell
# Run in elevated PowerShell on the Windows client before attempting domain join.

$DomainFqdn = "corp.local"
$NetbiosName = "CORP"
$DcName = "DC1"
$DcFqdn = "$DcName.$DomainFqdn"
$DcIp = "10.10.10.10"
$InterfaceAlias = "Ethernet"
$OutputPath = "C:\ADPrep\domain-join"

New-Item -ItemType Directory -Force -Path $OutputPath

# Capture current client state.
hostname
Get-ComputerInfo | Select-Object CsName, CsDomain, CsPartOfDomain |
  Tee-Object "$OutputPath\client-membership-before.txt"

Get-NetIPConfiguration |
  Tee-Object "$OutputPath\network-before.txt"

Get-DnsClientServerAddress -AddressFamily IPv4 |
  Tee-Object "$OutputPath\dns-client-before.txt"

# Point client to AD DNS.
Set-DnsClientServerAddress `
  -InterfaceAlias $InterfaceAlias `
  -ServerAddresses $DcIp

Clear-DnsClientCache

# Verify DNS client configuration.
Get-DnsClientServerAddress `
  -InterfaceAlias $InterfaceAlias `
  -AddressFamily IPv4

# Basic reachability.
Test-NetConnection $DcIp
Test-NetConnection $DcIp -Port 53

# AD DNS validation.
Resolve-DnsName $DomainFqdn -Server $DcIp
Resolve-DnsName $DcFqdn -Server $DcIp
Resolve-DnsName "_ldap._tcp.dc._msdcs.$DomainFqdn" -Type SRV -Server $DcIp
Resolve-DnsName "_kerberos._tcp.$DomainFqdn" -Type SRV -Server $DcIp

# DC locator validation.
nltest /dsgetdc:$DomainFqdn
```

# Join_Windows_Client_To_Domain_Local_Join_Skeleton
```powershell
# Run in elevated PowerShell on the Windows client.
# Use this after DNS and DC locator checks pass.

$DomainFqdn = "corp.local"
$NetbiosName = "CORP"

# Prompt for a domain account allowed to join computers.
$JoinCred = Get-Credential "$NetbiosName\Administrator"

# Join domain and restart.
Add-Computer `
  -DomainName $DomainFqdn `
  -Credential $JoinCred `
  -Restart `
  -PassThru `
  -Verbose

# Computer restarts automatically.
```

# Join_Windows_Client_To_Domain_Rename_And_Join_Skeleton
```powershell
# Use this when the client must be renamed and joined cleanly.
# This avoids joining with a throwaway name.

$NewComputerName = "WIN11-01"
$DomainFqdn = "corp.local"
$NetbiosName = "CORP"
$DcFqdn = "DC1.corp.local"
$TargetOU = "OU=Workstations,OU=Corp,DC=corp,DC=local"

$JoinCred = Get-Credential "$NetbiosName\Administrator"

# Rename without rebooting first.
Rename-Computer `
  -NewName $NewComputerName `
  -Force

# Join using the new name.
Add-Computer `
  -DomainName $DomainFqdn `
  -Server $DcFqdn `
  -OUPath $TargetOU `
  -Credential $JoinCred `
  -Options JoinWithNewName,AccountCreate `
  -Restart `
  -PassThru `
  -Verbose

# Computer restarts automatically.
```

# Join_Windows_Client_To_Domain_PostJoin_Verification_Skeleton
```powershell
# Run on the client after reboot and domain sign-in.

$DomainFqdn = "corp.local"
$DcIp = "10.10.10.10"
$DcFqdn = "DC1.corp.local"

# Confirm current identity.
whoami
whoami /groups

# Confirm domain membership.
Get-ComputerInfo |
  Select-Object CsName, CsDomain, CsPartOfDomain

# Confirm DNS still points to AD DNS.
Get-DnsClientServerAddress -AddressFamily IPv4
Get-NetIPConfiguration

# Confirm AD DNS resolution after join.
Resolve-DnsName $DomainFqdn -Server $DcIp
Resolve-DnsName $DcFqdn -Server $DcIp
Resolve-DnsName "_ldap._tcp.dc._msdcs.$DomainFqdn" -Type SRV -Server $DcIp
Resolve-DnsName "_kerberos._tcp.$DomainFqdn" -Type SRV -Server $DcIp

# Confirm DC locator.
nltest /dsgetdc:$DomainFqdn

# Confirm secure channel.
Test-ComputerSecureChannel -Verbose

# Optional: refresh policy and view applied state.
gpupdate /force
gpresult /r
```

# Join_Windows_Client_To_Domain_AD_Computer_Object_Verification_Skeleton
```powershell
# Run on the domain controller or admin workstation with RSAT.

Import-Module ActiveDirectory

$ClientName = "WIN11-01"
$TargetOU = "OU=Workstations,OU=Corp,DC=corp,DC=local"

# Find computer object.
Get-ADComputer `
  -Identity $ClientName `
  -Properties DNSHostName,DistinguishedName,Enabled,LastLogonDate,PasswordLastSet |
  Select-Object Name,DNSHostName,DistinguishedName,Enabled,LastLogonDate,PasswordLastSet

# Search target OU.
Get-ADComputer `
  -Filter * `
  -SearchBase $TargetOU `
  -Properties DNSHostName,LastLogonDate |
  Select-Object Name,DNSHostName,LastLogonDate

# Move object if it landed in CN=Computers or wrong OU.
$Computer = Get-ADComputer -Identity $ClientName

if ($Computer.DistinguishedName -notlike "*$TargetOU") {
    Move-ADObject `
      -Identity $Computer.DistinguishedName `
      -TargetPath $TargetOU
}

# Verify final location.
Get-ADComputer `
  -Identity $ClientName `
  -Properties DistinguishedName |
  Select-Object Name,DistinguishedName
```

# Join_Windows_Client_To_Domain_Secure_Channel_Skeleton
```powershell
# Run on the domain-joined client.

$DomainFqdn = "corp.local"

# Test secure channel.
Test-ComputerSecureChannel -Verbose

# If secure channel is broken, repair with credential.
# Use only after DNS and DC locator are healthy.
$RepairCred = Get-Credential "CORP\Administrator"

Test-ComputerSecureChannel `
  -Repair `
  -Credential $RepairCred `
  -Verbose

# Re-test.
Test-ComputerSecureChannel -Verbose

# Confirm DC locator after repair.
nltest /dsgetdc:$DomainFqdn
```

# Join_Windows_Client_To_Domain_Failure_Checks
| Symptom | Likely Cause | Check | Fix |
|---|---|---|---|
| `Add-Computer` not found | Wrong shell or PowerShell environment issue | `Get-Command Add-Computer` | Use Windows PowerShell elevated |
| Access denied during join | Join credential lacks rights or bad password | Confirm credential and permissions | Use valid delegated join account or Domain Admin in lab |
| Domain not found | Client DNS not pointing to AD DNS | `Get-DnsClientServerAddress` | Set DNS to DC IP |
| DC locator fails | Missing SRV records or wrong DNS server | `Resolve-DnsName _ldap._tcp.dc._msdcs.<domain> -Type SRV` | Fix AD DNS/SRV records first |
| DNS resolves public names only | Client uses public DNS | `ipconfig /all`; `Get-DnsClientServerAddress` | Point client to internal AD DNS |
| `Resolve-DnsName <dc-fqdn>` fails | DC A record missing or wrong zone | Query DNS server directly | Fix DC A record or DNS registration |
| Join fails with wrong OU | Bad OU DN syntax or OU does not exist | Verify `Get-ADOrganizationalUnit -Identity "<target-ou-dn>"` | Correct OUPath |
| Computer lands in `CN=Computers` | OUPath omitted | Check AD computer DistinguishedName | Move object or rejoin with OUPath |
| Computer account already exists | Duplicate/stale computer object | `Get-ADComputer <client-name>` | Reset/remove stale object or use correct credentials |
| Computer name too long | NetBIOS computer name exceeds limits | `hostname`; name length | Rename client to valid name |
| Join succeeds but no domain logon | Reboot not completed or DNS broken after join | Check membership and DNS | Reboot and confirm DNS |
| Join succeeds but secure channel false | Machine password/trust issue | `Test-ComputerSecureChannel` | Repair secure channel or rejoin |
| Secure channel repair fails | DNS/DC locator still broken | `nltest /dsgetdc:<domain>` | Fix DNS and DC locator first |
| Kerberos errors after join | Time skew | `w32tm /query /status`; `Get-Date` | Sync time with domain |
| `nltest /dsgetdc` fails from client | DNS or firewall path issue | SRV lookup and port checks | Fix DNS client, firewall, routing |
| `Test-NetConnection -Port 53` passes but DNS still fails | UDP 53 blocked or DNS record issue | Packet/firewall review; Resolve-DnsName output | Permit UDP 53 and fix records |
| Domain join takes long time | First DNS server unreachable | `Get-DnsClientServerAddress` | Remove dead DNS server from list |
| Join with `-Server` fails | DC FQDN wrong or hardening requires FQDN | Resolve DC FQDN | Use DC FQDN such as `DC1.corp.local` |
| Local admin required | User not elevated on client | `whoami /groups` | Run elevated PowerShell |
| Remote join fails | WinRM/local credential/firewall issue | `Test-WSMan`; local admin check | Fix remoting or join locally |
| Reboot does not happen | `-Restart` omitted | Check membership state | Run `Restart-Computer` |
| GPO not applying after join | Computer in wrong OU or no GPO linked | `gpresult /r`; AD computer DN | Move computer to correct OU and run `gpupdate /force` |
| User cannot log on after join | User account issue, group issue, DNS, or DC locator | Try known-good domain account | Validate user, DNS, and DC locator |
| Rollback leaves stale AD object | Computer removed locally but AD object remains | `Get-ADComputer <client-name>` | Disable/remove/move stale object |
