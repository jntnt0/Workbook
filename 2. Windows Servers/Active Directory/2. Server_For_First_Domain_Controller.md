# Prepare_Windows_Server_For_First_Domain_Controller_Mental_Model
| Concept | Operational Meaning |
|---|---|
| First domain controller | The first server promoted into a new AD forest and root domain |
| Pre-promotion state | The server is still a standalone Windows Server, not yet a DC |
| Server identity | Hostname must be final before promotion because renaming a domain controller later is unnecessary risk |
| Static IP | Domain controllers must not rely on DHCP for their main address |
| DNS client setting | Before promotion, the server must resolve what it needs; after promotion, it usually points to itself |
| AD DS role binaries | Installing the role stages the files and tools; it does not create the domain by itself |
| Promotion | The later step that creates or joins AD DS as a domain controller |
| DSRM password | Recovery password needed during AD DS forest installation |
| Snapshot or checkpoint | Safe revert point before irreversible promotion work in a lab |
| Time sync | Kerberos and domain operations depend on sane time |
| Updates | Patch before promotion so the new DC starts from a healthy baseline |
| Local admin access | Required before promotion because domain accounts do not exist yet in a new forest |
| Firewall readiness | DNS, Kerberos, LDAP, SMB, RPC, and management paths matter after promotion |
| Precheck gate | Do not run `Install-ADDSForest` until hostname, IP, DNS, time, updates, disk, and role checks are clean |
| First rule | Prepare the box like it will become infrastructure, not like a disposable workstation |

# Prepare_Windows_Server_For_First_Domain_Controller_Base_Server_Skeleton
```powershell
# Run in elevated PowerShell on the future first domain controller.

$ServerName = "DC1"
$PrepPath = "C:\ADPrep"

# Confirm current identity.
hostname
Get-ComputerInfo | Select-Object CsName, CsDomain, CsPartOfDomain, WindowsProductName, WindowsVersion

# Create prep folder for baseline exports.
New-Item -ItemType Directory -Force -Path $PrepPath

# Export baseline before changes.
Get-ComputerInfo | Out-File "$PrepPath\computer-baseline-before.txt"
Get-NetIPConfiguration | Out-File "$PrepPath\network-baseline-before.txt"
Get-NetAdapter | Out-File "$PrepPath\netadapter-baseline-before.txt"

# Rename only if current hostname is not final.
if ($env:COMPUTERNAME -ne $ServerName) {
    Rename-Computer -NewName $ServerName -Restart
}

# After reboot, rerun:
hostname
Get-ComputerInfo | Select-Object CsName, CsDomain, CsPartOfDomain, WindowsProductName, WindowsVersion
```

# Prepare_Windows_Server_For_First_Domain_Controller_Static_IP_Skeleton
```powershell
# Run in elevated PowerShell after hostname is final.

$InterfaceAlias = "Ethernet"
$DcIp = "10.10.10.10"
$PrefixLength = 24
$GatewayIp = "10.10.10.1"

# Inspect current interface state.
Get-NetAdapter
Get-NetIPConfiguration -InterfaceAlias $InterfaceAlias

# Configure static IPv4 only if not already correct.
New-NetIPAddress `
  -InterfaceAlias $InterfaceAlias `
  -IPAddress $DcIp `
  -PrefixLength $PrefixLength `
  -DefaultGateway $GatewayIp

# Verify static IP.
Get-NetIPConfiguration -InterfaceAlias $InterfaceAlias

# Test default gateway.
Test-NetConnection $GatewayIp
```

# Prepare_Windows_Server_For_First_Domain_Controller_DNS_Client_Skeleton
```powershell
# Run before AD DS promotion.

$InterfaceAlias = "Ethernet"
$InitialDns = "10.10.10.10"
$GatewayIp = "10.10.10.1"

# For isolated lab where this server will host DNS, use its own future DC IP.
# For production or staged build, use the existing internal DNS server until promotion plan says otherwise.

Set-DnsClientServerAddress `
  -InterfaceAlias $InterfaceAlias `
  -ServerAddresses $InitialDns

# Verify resolver settings.
Get-DnsClientServerAddress -InterfaceAlias $InterfaceAlias -AddressFamily IPv4
Get-NetIPConfiguration -InterfaceAlias $InterfaceAlias

# Basic resolution test.
Resolve-DnsName microsoft.com -Server $InitialDns -ErrorAction SilentlyContinue

# Clear client cache before final precheck.
Clear-DnsClientCache
```

# Prepare_Windows_Server_For_First_Domain_Controller_ADDS_Role_Precheck_Skeleton
```powershell
# Run before forest creation. This stages AD DS role bits but does not promote the server.

$PrepPath = "C:\ADPrep"

# Install AD DS role and management tools.
Install-WindowsFeature `
  -Name AD-Domain-Services `
  -IncludeManagementTools

# Verify role state.
Get-WindowsFeature AD-Domain-Services | Tee-Object "$PrepPath\adds-role-state.txt"

# Import deployment module.
Import-Module ADDSDeployment

# Confirm command availability.
Get-Command Test-ADDSForestInstallation
Get-Command Install-ADDSForest

# Export installed feature state.
Get-WindowsFeature | Where-Object Installed | Out-File "$PrepPath\installed-features.txt"
```

# Prepare_Windows_Server_For_First_Domain_Controller_PrePromotion_Gate_Skeleton
```powershell
# This is the final gate before running Install-ADDSForest in the next workbook.

$DomainFqdn = "corp.local"
$NetbiosName = "CORP"
$InterfaceAlias = "Ethernet"
$DcIp = "10.10.10.10"
$GatewayIp = "10.10.10.1"
$PrepPath = "C:\ADPrep"

# Confirm standalone state.
Get-ComputerInfo | Select-Object CsName, CsDomain, CsPartOfDomain

# Confirm final hostname.
hostname

# Confirm static IP and DNS.
Get-NetIPConfiguration -InterfaceAlias $InterfaceAlias
Get-DnsClientServerAddress -InterfaceAlias $InterfaceAlias -AddressFamily IPv4

# Confirm gateway.
Test-NetConnection $GatewayIp

# Confirm AD DS role staged.
Get-WindowsFeature AD-Domain-Services

# Confirm disk space.
Get-PSDrive -PSProvider FileSystem

# Confirm time.
Get-Date
w32tm /query /status

# Prepare DSRM password.
$DSRM = Read-Host "Enter DSRM password for precheck" -AsSecureString

# Run forest install precheck only.
Test-ADDSForestInstallation `
  -DomainName $DomainFqdn `
  -DomainNetbiosName $NetbiosName `
  -InstallDns `
  -SafeModeAdministratorPassword $DSRM `
  | Tee-Object "$PrepPath\adds-forest-precheck.txt"

# Stop here.
# If precheck passes, create VM snapshot/checkpoint before running Install-ADDSForest.
```

# Prepare_Windows_Server_For_First_Domain_Controller_Failure_Checks
| Symptom | Likely Cause | Check | Fix |
|---|---|---|---|
| `Rename-Computer` fails | Not running elevated or name invalid | Confirm admin shell and target name | Use elevated PowerShell and valid hostname |
| Server loses network after IP change | Wrong IP, prefix, gateway, or interface alias | `Get-NetIPConfiguration`; console access | Correct IP settings from console |
| `New-NetIPAddress` fails because address exists | Existing IP already assigned | `Get-NetIPAddress -InterfaceAlias "<interface>"` | Skip or remove incorrect existing IP first |
| Gateway test fails | Wrong gateway, VLAN, virtual switch, firewall, or subnet | `Test-NetConnection <gateway>` | Fix subnet/gateway/vSwitch/VLAN |
| DNS lookup fails before promotion | Wrong initial DNS server or isolated lab without upstream DNS | `Get-DnsClientServerAddress`; `Resolve-DnsName` | Point to working upstream DNS or accept isolated lab limitation |
| Server still on DHCP | Static IP not configured or wrong adapter changed | `Get-NetIPConfiguration` | Configure static IP on correct adapter |
| Time is wrong | Bad CMOS/VM time sync/time zone | `Get-Date`; `Get-TimeZone`; `w32tm /query /status` | Correct time zone and time source |
| Updates require reboot | Pending reboot after patching or feature install | Check pending reboot registry marker | Reboot before AD promotion |
| AD DS role install fails | Not elevated, missing source files, servicing issue | `Get-WindowsFeature AD-Domain-Services` | Run elevated, repair source, check Windows Update/servicing |
| Management tools missing | `-IncludeManagementTools` omitted | Check AD module/cmdlets | Re-run install with `-IncludeManagementTools` |
| `Import-Module ADDSDeployment` fails | AD DS tools not installed | `Get-WindowsFeature AD-Domain-Services` | Install role and tools |
| Forest precheck rejects NetBIOS name | Too long or invalid characters | Review precheck output | Use short valid NetBIOS name |
| Forest precheck rejects domain name | Invalid DNS name or conflict | Review precheck output | Choose clean AD DNS name |
| Forest precheck complains about DNS | Resolver or DNS plan inconsistent | `Get-DnsClientServerAddress`; `Resolve-DnsName` | Fix pre-promotion DNS client setting |
| Forest precheck complains about password | DSRM password complexity issue | Review prompt/precheck | Use strong DSRM password |
| Disk space too low | OS disk not sized for DC role/logs/SYSVOL | `Get-PSDrive` | Expand disk before promotion |
| Snapshot missing | Promotion would be harder to roll back in lab | Check hypervisor snapshots | Create checkpoint before promotion |
| Server is already domain joined | Wrong build starting point for first forest DC | `(Get-ComputerInfo).CsPartOfDomain` | Use clean standalone server for first forest DC |
| Multiple NICs confuse DNS/AD | Extra adapters register DNS or wrong interface used | `Get-NetAdapter`; `Get-DnsClientServerAddress` | Disable unused NICs or prevent registration |
| IPv6 disabled badly | Broken Windows networking assumptions | Check adapter bindings and DNS behavior | Do not randomly disable IPv6 unless design requires it |
| Local admin password lost | No domain exists yet to recover access | Confirm password before promotion | Store local admin and DSRM secrets securely |
| Server Core management blocked | Firewall/WinRM not ready | `Test-WSMan`; `Get-NetFirewallRule` | Enable intended remote management path |
