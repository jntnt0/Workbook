# Validate_AD_Integrated_DNS_And_SRV_Records_Mental_Model
| Concept | Operational Meaning |
|---|---|
| AD-integrated DNS | DNS zones stored in Active Directory instead of only local zone files |
| Domain zone | Main AD DNS zone, such as `corp.local` |
| `_msdcs` zone | AD locator zone containing forest-wide domain controller discovery records |
| SRV record | DNS record that advertises a service, protocol, port, and target host |
| DC locator | Client process that uses DNS SRV records to find a domain controller |
| LDAP SRV | Records clients use to locate LDAP services on domain controllers |
| Kerberos SRV | Records clients use to locate Kerberos authentication services |
| Global Catalog SRV | Records clients use to locate Global Catalog servers |
| A record | Maps DC hostname to IPv4 address |
| PTR record | Maps DC IP address back to hostname |
| Netlogon registration | Domain controllers register AD DNS records through Netlogon |
| SYSVOL/NETLOGON shares | Required shares that indicate the DC is serving domain policy and logon resources |
| Authoritative answer | DNS server owns the zone and should answer directly for names in that zone |
| First rule | If `_ldap._tcp.dc._msdcs.<domain>` does not resolve, domain join and domain logon will fail or behave badly |

# Validate_AD_Integrated_DNS_And_SRV_Records_Zone_Baseline_Skeleton
```powershell
# Run on the domain controller.

$DomainFqdn = "corp.local"
$DcName = "DC1"
$DcFqdn = "$DcName.$DomainFqdn"
$DcIp = "10.10.10.10"
$DnsServer = "127.0.0.1"
$ValidationPath = "C:\ADPrep\dns-validation"

New-Item -ItemType Directory -Force -Path $ValidationPath

Import-Module DnsServer
Import-Module ActiveDirectory

# Confirm services.
Get-Service DNS,Netlogon,NTDS,ADWS

# Confirm AD identity.
Get-ADDomain
Get-ADForest
Get-ADDomainController -Identity $env:COMPUTERNAME

# List all DNS zones.
Get-DnsServerZone

# Save zone inventory.
Get-DnsServerZone |
  Export-Csv "$ValidationPath\dns-zones.csv" -NoTypeInformation

# Confirm domain zone.
Get-DnsServerZone -Name $DomainFqdn

# Check for _msdcs zone.
Get-DnsServerZone |
  Where-Object {$_.ZoneName -like "*_msdcs*"}
```

# Validate_AD_Integrated_DNS_And_SRV_Records_Core_SRV_Query_Skeleton
```powershell
# Run on the domain controller.

$DomainFqdn = "corp.local"
$DcName = "DC1"
$DcFqdn = "$DcName.$DomainFqdn"
$DcIp = "10.10.10.10"
$DnsServer = "127.0.0.1"

# DC host A record.
Resolve-DnsName $DcFqdn -Type A -Server $DnsServer

# LDAP DC locator records.
Resolve-DnsName "_ldap._tcp.dc._msdcs.$DomainFqdn" -Type SRV -Server $DnsServer

# LDAP domain records.
Resolve-DnsName "_ldap._tcp.$DomainFqdn" -Type SRV -Server $DnsServer

# Kerberos records.
Resolve-DnsName "_kerberos._tcp.$DomainFqdn" -Type SRV -Server $DnsServer
Resolve-DnsName "_kerberos._udp.$DomainFqdn" -Type SRV -Server $DnsServer

# Password change records.
Resolve-DnsName "_kpasswd._tcp.$DomainFqdn" -Type SRV -Server $DnsServer
Resolve-DnsName "_kpasswd._udp.$DomainFqdn" -Type SRV -Server $DnsServer

# Global Catalog records.
Resolve-DnsName "_gc._tcp.$DomainFqdn" -Type SRV -Server $DnsServer

# Optional reverse lookup.
Resolve-DnsName $DcIp -Type PTR -Server $DnsServer -ErrorAction SilentlyContinue
```

# Validate_AD_Integrated_DNS_And_SRV_Records_Client_Query_Skeleton
```powershell
# Run on a Windows client or member server.
# This proves clients can use AD DNS for DC discovery.

$DomainFqdn = "corp.local"
$DcIp = "10.10.10.10"
$DcName = "DC1"
$DcFqdn = "$DcName.$DomainFqdn"

# Confirm client resolver state.
Get-NetIPConfiguration
Get-DnsClientServerAddress -AddressFamily IPv4

# Query DC DNS directly.
Resolve-DnsName $DcFqdn -Type A -Server $DcIp
Resolve-DnsName "_ldap._tcp.dc._msdcs.$DomainFqdn" -Type SRV -Server $DcIp
Resolve-DnsName "_kerberos._tcp.$DomainFqdn" -Type SRV -Server $DcIp
Resolve-DnsName "_gc._tcp.$DomainFqdn" -Type SRV -Server $DcIp

# Query through the client default DNS path.
Resolve-DnsName $DcFqdn -Type A
Resolve-DnsName "_ldap._tcp.dc._msdcs.$DomainFqdn" -Type SRV

# DC locator.
nltest /dsgetdc:$DomainFqdn
```

# Validate_AD_Integrated_DNS_And_SRV_Records_Netlogon_Reregistration_Skeleton
```powershell
# Use only if expected AD SRV records are missing and DNS service/zones are otherwise healthy.
# Run on the affected domain controller.

$DomainFqdn = "corp.local"
$DnsServer = "127.0.0.1"

# Confirm required services first.
Get-Service DNS,Netlogon,NTDS

# Confirm domain zone exists.
Get-DnsServerZone -Name $DomainFqdn

# Restart Netlogon to trigger DC DNS record registration.
Restart-Service Netlogon

# Wait briefly for registration.
Start-Sleep -Seconds 15

# Retest required SRV records.
Resolve-DnsName "_ldap._tcp.dc._msdcs.$DomainFqdn" -Type SRV -Server $DnsServer
Resolve-DnsName "_kerberos._tcp.$DomainFqdn" -Type SRV -Server $DnsServer
Resolve-DnsName "_gc._tcp.$DomainFqdn" -Type SRV -Server $DnsServer

# Check Netlogon and DNS logs if still broken.
Get-WinEvent -LogName System -MaxEvents 100 |
  Where-Object {$_.ProviderName -like "*Netlogon*" -or $_.Message -like "*Netlogon*"}
```

# Validate_AD_Integrated_DNS_And_SRV_Records_Failure_Checks
| Symptom | Likely Cause | Check | Fix |
|---|---|---|---|
| Domain zone missing | DNS not installed, promotion failed, or zone not created | `Get-DnsServerZone` | Verify DNS role and AD promotion state |
| `_msdcs` zone missing | Forest locator zone not created separately or records are nested in domain zone | `Get-DnsServerZone`; inspect domain zone records | Confirm expected design before changing anything |
| DC A record missing | Netlogon registration failed or DNS dynamic update issue | `Resolve-DnsName <dc-fqdn>` | Restart Netlogon after DNS zone health is confirmed |
| DC A record wrong | DC IP changed after promotion or stale record exists | `Get-DnsServerResourceRecord -ZoneName <domain> -Name <dc>` | Correct stale A record and verify static IP |
| LDAP SRV missing | Netlogon did not register DC locator records | SRV lookup and System log | Restart Netlogon, then review Netlogon events |
| Kerberos SRV missing | Netlogon/DNS registration issue | `_kerberos._tcp.<domain>` lookup | Restart Netlogon after DNS service is healthy |
| GC SRV missing | DC is not GC or registration failed | `Get-ADDomainController -Filter *` | Confirm GC state and Netlogon registration |
| Client cannot find DC | Client points to wrong DNS server | `Get-DnsClientServerAddress` on client | Point client to AD DNS server |
| DC resolves records locally but client fails | Firewall, routing, or client resolver issue | `Resolve-DnsName ... -Server <dc-ip>` from client | Fix client path to DNS server |
| `nltest /dsgetdc` fails | SRV records missing or client DNS wrong | SRV lookup from same machine | Fix DNS server list or SRV registration |
| SYSVOL missing | DFSR/SYSVOL initialization issue | `net share`; Directory Service/System logs | Wait briefly after promotion, then troubleshoot DFSR |
| NETLOGON missing | Netlogon service issue or SYSVOL not ready | `Get-Service Netlogon`; `net share` | Start/restart Netlogon after AD health is verified |
| Reverse lookup fails | Reverse zone/PTR missing | `Resolve-DnsName <dc-ip> -Type PTR` | Create reverse zone/PTR if required |
| Duplicate DC A records | Old IP plus new IP registered | `Get-DnsServerResourceRecord -ZoneName <domain> -Name <dc> -RRType A` | Remove stale A record and fix NIC registration |
| Records vanish after reboot | Dynamic update or zone replication problem | Event logs and zone type | Confirm AD-integrated zone and dynamic update health |
| DNS events show load failure | Zone issue, AD DS issue, or DNS service issue | DNS Server log | Fix underlying zone/service/AD problem |
| Directory Service events show replication issues | AD replication problem impacts DNS replication | Directory Service log | Troubleshoot AD replication before DNS changes |
| `Get-DnsServerResourceRecord` fails | Wrong zone name or module/session issue | `Get-DnsServerZone` | Use exact zone name and import DnsServer module |
| Query works with `127.0.0.1` but fails with DC IP | Listener/firewall/binding issue | Check DNS service and port 53 path | Fix DNS listener/firewall/interface |
| Query works on DC but not from client default path | Client not using DC DNS | `Get-DnsClientServerAddress` | Set client DNS server to DC IP |
| Public DNS is used by clients | DHCP/VPN/router gives public DNS | `ipconfig /all` and PowerShell DNS client check | Correct DHCP/VPN DNS options |
| Site-specific SRV missing | AD Sites and Services not configured or site has no DC | Query non-site SRV first | Configure sites/subnets only if needed |
| Global Catalog discovery fails | First DC not advertising GC or registration issue | `Get-ADForest`; `Get-ADDomainController` | Confirm GC status and Netlogon registration |
| Authentication slow | DNS timeout or wrong first DNS server | Check client DNS server order | Remove unreachable DNS server from client list |
| Domain join fails after forest creation | Missing SRV records or wrong client DNS | `_ldap._tcp.dc._msdcs` lookup from client | Fix SRV records and client DNS |
| Admin deletes zone during repair | Destructive AD DNS change | Review change history | Restore from backup/snapshot and escalate |
