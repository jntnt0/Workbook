# Create_New_AD_Forest_And_Root_Domain_Mental_Model
| Concept | Operational Meaning |
|---|---|
| New forest | Creates the top-level AD security and replication boundary |
| Forest root domain | First domain in the forest; all later domains trust upward through this structure |
| First domain controller | The first DC holds the initial domain, DNS, SYSVOL, Global Catalog, and FSMO roles |
| Install-ADDSForest | PowerShell cmdlet that performs the actual new forest creation |
| Test-ADDSForestInstallation | PowerShell cmdlet that checks whether the forest install can succeed before making the change |
| DSRM password | Directory Services Restore Mode password used for offline AD recovery |
| Domain FQDN | Full DNS name of the AD domain, such as `corp.local` or `ad.contoso.com` |
| NetBIOS name | Short legacy domain name, such as `CORP`; must be 15 characters or fewer |
| InstallDns | Installs and configures DNS on the first domain controller |
| AD-integrated DNS | DNS zones stored in Active Directory and replicated by AD |
| SYSVOL | Replicated share that stores Group Policy templates and scripts |
| NETLOGON | Share used for domain logon scripts and DC discovery support |
| SRV records | DNS service records that clients use to find LDAP, Kerberos, and domain controllers |
| FSMO roles | Special single-master AD roles initially held by the first DC |
| Global Catalog | Forest-wide searchable catalog; first DC is normally a GC |
| Safe stop point | After precheck passes, create snapshot/checkpoint before running install |
| First rule | If DNS, hostname, static IP, and time are not clean, do not create the forest yet |

# Create_New_AD_Forest_And_Root_Domain_Precheck_Skeleton
```powershell
# Run in elevated PowerShell on the prepared future domain controller.
# This performs prerequisite checks only. It does not create the forest.

$PrepPath = "C:\ADPrep"
$DomainFqdn = "corp.local"
$NetbiosName = "CORP"
$DatabasePath = "C:\Windows\NTDS"
$LogPath = "C:\Windows\NTDS"
$SysvolPath = "C:\Windows\SYSVOL"

New-Item -ItemType Directory -Force -Path $PrepPath

# Confirm server state.
hostname
Get-ComputerInfo | Select-Object CsName, CsDomain, CsPartOfDomain
Get-NetIPConfiguration
Get-WindowsFeature AD-Domain-Services

# Load deployment module.
Import-Module ADDSDeployment

# Confirm commands exist.
Get-Command Test-ADDSForestInstallation
Get-Command Install-ADDSForest

# Capture DSRM password securely.
$DSRM = Read-Host "Enter Directory Services Restore Mode password" -AsSecureString

# Run precheck.
$Precheck = Test-ADDSForestInstallation `
  -DomainName $DomainFqdn `
  -DomainNetbiosName $NetbiosName `
  -InstallDns `
  -DatabasePath $DatabasePath `
  -LogPath $LogPath `
  -SysvolPath $SysvolPath `
  -SafeModeAdministratorPassword $DSRM

$Precheck

# Save precheck output.
$Precheck | Out-File "$PrepPath\adds-forest-precheck.txt"

# Stop here and create a VM snapshot/checkpoint before the install step.
```

# Create_New_AD_Forest_And_Root_Domain_Install_Skeleton
```powershell
# Run in elevated PowerShell only after precheck passes and snapshot/checkpoint exists.
# This creates the new AD forest and root domain.
# The server will reboot after promotion.

$PrepPath = "C:\ADPrep"
$DomainFqdn = "corp.local"
$NetbiosName = "CORP"
$DatabasePath = "C:\Windows\NTDS"
$LogPath = "C:\Windows\NTDS"
$SysvolPath = "C:\Windows\SYSVOL"

New-Item -ItemType Directory -Force -Path $PrepPath

Start-Transcript -Path "$PrepPath\adds-forest-install-transcript.txt"

Import-Module ADDSDeployment

# Prompt for DSRM password.
$DSRM = Read-Host "Enter Directory Services Restore Mode password" -AsSecureString

# Final safety check.
Test-ADDSForestInstallation `
  -DomainName $DomainFqdn `
  -DomainNetbiosName $NetbiosName `
  -InstallDns `
  -DatabasePath $DatabasePath `
  -LogPath $LogPath `
  -SysvolPath $SysvolPath `
  -SafeModeAdministratorPassword $DSRM

# Create the forest.
Install-ADDSForest `
  -DomainName $DomainFqdn `
  -DomainNetbiosName $NetbiosName `
  -InstallDns `
  -DatabasePath $DatabasePath `
  -LogPath $LogPath `
  -SysvolPath $SysvolPath `
  -SafeModeAdministratorPassword $DSRM `
  -Force

# The server normally reboots automatically.
# If transcript is interrupted by reboot, that is expected.
```

# Create_New_AD_Forest_And_Root_Domain_DNS_Verification_Skeleton
```powershell
# Run after forest creation.
# AD depends heavily on DNS. Do this before joining clients.

$DomainFqdn = "corp.local"
$DcName = "DC1"
$DcIp = "10.10.10.10"

Import-Module DnsServer

# Verify DNS service.
Get-Service DNS

# Verify zones.
Get-DnsServerZone

# Verify domain zone.
Get-DnsServerZone -Name $DomainFqdn

# Verify DC A record.
Resolve-DnsName "$DcName.$DomainFqdn" -Type A -Server 127.0.0.1

# Verify LDAP SRV records.
Resolve-DnsName "_ldap._tcp.dc._msdcs.$DomainFqdn" -Type SRV -Server 127.0.0.1
Resolve-DnsName "_ldap._tcp.$DomainFqdn" -Type SRV -Server 127.0.0.1

# Verify Kerberos SRV records.
Resolve-DnsName "_kerberos._tcp.$DomainFqdn" -Type SRV -Server 127.0.0.1

# Verify Global Catalog SRV records.
Resolve-DnsName "_gc._tcp.$DomainFqdn" -Type SRV -Server 127.0.0.1

# Verify DC host lookup by configured DNS server.
Resolve-DnsName "$DcName.$DomainFqdn" -Type A -Server $DcIp

# Optional reverse lookup if reverse zone exists.
Resolve-DnsName $DcIp -Server 127.0.0.1 -ErrorAction SilentlyContinue
```

# Create_New_AD_Forest_And_Root_Domain_Failure_Checks
| Symptom | Likely Cause | Check | Fix |
|---|---|---|---|
| `Install-ADDSForest` not found | ADDSDeployment module unavailable or AD DS role/tools not installed | `Get-Command Install-ADDSForest`; `Get-WindowsFeature AD-Domain-Services` | Install AD DS role and management tools |
| Precheck fails on domain name | Invalid FQDN or namespace conflict | Review `Test-ADDSForestInstallation` output | Choose valid AD DNS name |
| Precheck fails on NetBIOS name | NetBIOS name invalid or longer than 15 characters | Check `$NetbiosName.Length` | Use valid short name |
| Precheck fails on DNS | DNS client setting or delegation problem | `Get-DnsClientServerAddress`; review precheck output | Correct DNS client or use correct delegation plan |
| Precheck fails on DSRM password | Password blank or not complex enough | Re-enter secure password | Use strong DSRM password |
| Precheck fails on paths | NTDS, log, or SYSVOL path invalid | `Test-Path <path>` | Create path or use defaults |
| Install fails quickly | Precheck ignored or role not staged | Review transcript and precheck output | Fix precheck blocker before install |
| Install hangs | DNS, network, or system issue | Review console, event logs, and setup logs | Wait if active; otherwise troubleshoot logs |
| Server does not reboot | `-NoRebootOnCompletion` used or install failed | Check install output and event logs | Reboot manually only if install succeeded |
| Cannot log in after reboot | Using local account format instead of domain account | Try `<NETBIOS>\Administrator` | Use domain admin logon |
| `Get-ADDomain` fails after reboot | AD not ready, module issue, or promotion failed | `Get-Service NTDS,ADWS`; event logs | Confirm services and promotion state |
| SYSVOL missing | DFSR/SYSVOL not initialized or failed | `net share`; Directory Service logs | Wait briefly, then troubleshoot DFSR/SYSVOL |
| NETLOGON missing | Netlogon not running or SYSVOL not ready | `Get-Service Netlogon`; `net share` | Start/restart Netlogon after AD health is confirmed |
| DNS zones missing | DNS install/config failed | `Get-DnsServerZone`; DNS Server logs | Fix DNS role and AD-integrated zones |
| DC A record missing | DNS registration failure | `Resolve-DnsName <dc-fqdn>` | Restart Netlogon after DNS is healthy |
| SRV records missing | Netlogon registration failure or DNS zone issue | `Resolve-DnsName _ldap._tcp.dc._msdcs.<domain> -Type SRV` | Fix DNS zone, restart Netlogon |
| External lookup fails | No forwarder or upstream path | `Resolve-DnsName microsoft.com` | Configure forwarder after AD DNS is healthy |
| Client cannot join domain | Client DNS points somewhere else | `Get-DnsClientServerAddress` on client | Point client to new DC DNS |
| Kerberos errors | Time skew after domain creation | `w32tm /query /status`; `Get-Date` | Fix time source |
| FSMO query fails | Netdom missing or AD services not ready | `Get-ADDomain`; `Get-ADForest` | Use AD PowerShell or wait/fix services |
| Forest mode unexpected | Default mode selected by OS/cmdlet | `Get-ADForest | Select ForestMode` | Explicitly set mode on future builds if required |
| Domain mode unexpected | Default mode selected by OS/cmdlet | `Get-ADDomain | Select DomainMode` | Explicitly set mode on future builds if required |
| DNS client not set to loopback | Automatic DNS client config skipped or changed | `Get-DnsClientServerAddress` | Set DNS client to `127.0.0.1` or intended DC DNS |
| Accidentally used wrong domain name | Install command used wrong `$DomainFqdn` | `Get-ADDomain` | Revert snapshot in lab; do not rename forest |
| First DC built without snapshot | Manual rollback is painful | Check hypervisor | Create snapshots in future before promotion |
| Installed on wrong server | Wrong console/session | `hostname`; `Get-ADDomainController` | Revert lab snapshot or plan demotion/rebuild |
