# Create_Baseline_OU_Structure_Mental_Model
| Concept | Operational Meaning |
|---|---|
| OU | Organizational Unit, an AD container used for organization, delegation, and Group Policy targeting |
| Domain DN | Distinguished name of the domain, such as `DC=corp,DC=local` |
| Root OU | Top-level OU under the domain root used to keep managed objects out of default containers |
| Child OU | OU created under another OU, such as `OU=Users,OU=Corp,DC=corp,DC=local` |
| Default containers | Built-in containers like `CN=Users` and `CN=Computers`; they are not OUs and are not ideal for structured management |
| Delegation boundary | OU layer where permissions can be assigned to helpdesk, server admins, workstation admins, or app owners |
| GPO targeting boundary | OU layer where Group Policy can be linked for users, computers, servers, or departments |
| Protected from accidental deletion | OU safety flag that prevents easy accidental deletion until protection is removed |
| Baseline structure | Starting OU layout used before adding production users, computers, servers, and groups |
| Idempotent creation | Script pattern that creates missing OUs but skips OUs that already exist |
| Naming convention | Predictable OU names such as `Users`, `Groups`, `Workstations`, `Servers`, `Admins`, `Service Accounts` |
| First rule | Create OUs around administration and policy boundaries, not just an org chart |

# Run on a domain controller or management workstation with RSAT.
# This creates a simple baseline OU layout.

Import-Module ActiveDirectory

$DomainDN = (Get-ADDomain).DistinguishedName

New-ADOrganizationalUnit `

New-ADOrganizationalUnit `
  -Name "Groups" `
  -Path $RootOU `
  -Description "Security and distribution groups" `
  -ProtectedFromAccidentalDeletion $true

New-ADOrganizationalUnit `
  -Name "Workstations" `
  -Path $RootOU `
  -Description "Domain-joined workstation computer accounts" `
  -ProtectedFromAccidentalDeletion $true

New-ADOrganizationalUnit `
  -Name "Servers" `
  -Path $RootOU `
  -Description "Domain-joined server computer accounts" `
  -ProtectedFromAccidentalDeletion $true

New-ADOrganizationalUnit `
  -Name "Admins" `
  -Path $RootOU `
  -Description "Privileged administrative accounts" `
  -ProtectedFromAccidentalDeletion $true

New-ADOrganizationalUnit `
  -Name "Service Accounts" `
  -Path $RootOU `

# Create_Baseline_OU_Structure_Idempotent_Create_Skeleton
```powershell
# Run on a domain controller or management workstation with RSAT.
# Idempotent pattern: create OUs only if missing.

Import-Module ActiveDirectory

$OutputPath = "C:\ADPrep\ou-structure"
New-Item -ItemType Directory -Force -Path $OutputPath

$DomainDN = (Get-ADDomain).DistinguishedName
$RootOUName = "Corp"
$RootOUDN = "OU=$RootOUName,$DomainDN"

function Ensure-OU {
    param(
        [Parameter(Mandatory)]
        [string]$Name,

        [Parameter(Mandatory)]
        [string]$Path,

        [string]$Description = ""
    )

    $TargetDN = "OU=$Name,$Path"

    $ExistingOU = Get-ADOrganizationalUnit `
      -LDAPFilter "(distinguishedName=$TargetDN)" `
      -Properties ProtectedFromAccidentalDeletion,Description `
      -ErrorAction SilentlyContinue

    if ($ExistingOU) {
        Write-Host "OU already exists: $TargetDN"
    }
    else {
        New-ADOrganizationalUnit `
          -Name $Name `
          -Path $Path `
          -Description $Description `
          -ProtectedFromAccidentalDeletion $true

        Write-Host "Created OU: $TargetDN"
    }
}

# Export before state.
Get-ADOrganizationalUnit -Filter * |
  Export-Csv "$OutputPath\ous-before.csv" -NoTypeInformation

# Create root.
Ensure-OU `
  -Name $RootOUName `
  -Path $DomainDN `
  -Description "Root OU for managed corporate directory objects"

# Create baseline children.
Ensure-OU -Name "Users" -Path $RootOUDN -Description "Standard user accounts"
Ensure-OU -Name "Groups" -Path $RootOUDN -Description "Security and distribution groups"
Ensure-OU -Name "Workstations" -Path $RootOUDN -Description "Domain-joined workstation computer accounts"
Ensure-OU -Name "Servers" -Path $RootOUDN -Description "Domain-joined server computer accounts"
Ensure-OU -Name "Admins" -Path $RootOUDN -Description "Privileged administrative accounts"
Ensure-OU -Name "Service Accounts" -Path $RootOUDN -Description "Application and service identity accounts"
Ensure-OU -Name "Disabled Objects" -Path $RootOUDN -Description "Disabled or retired objects awaiting review"
Ensure-OU -Name "Departments" -Path $RootOUDN -Description "Department-level OU structure"

# Export after state.
Get-ADOrganizationalUnit -Filter * |
  Export-Csv "$OutputPath\ous-after.csv" -NoTypeInformation

# Verify baseline tree.
Get-ADOrganizationalUnit `
  -Filter * `
  -SearchBase $RootOUDN `
  -Properties Description,ProtectedFromAccidentalDeletion |
  Select-Object Name, DistinguishedName, Description, ProtectedFromAccidentalDeletion
```

# Create_Baseline_OU_Structure_Departmental_Layout_Skeleton
```powershell
# Optional departmental OU layer.
# Use only if you actually plan to target delegation or GPOs by department.

Import-Module ActiveDirectory

$DomainDN = (Get-ADDomain).DistinguishedName
$RootOUDN = "OU=Corp,$DomainDN"
$DepartmentsOUDN = "OU=Departments,$RootOUDN"

function Ensure-OU {
    param(
        [string]$Name,
        [string]$Path,
        [string]$Description
    )

    $TargetDN = "OU=$Name,$Path"

    if (-not (Get-ADOrganizationalUnit -LDAPFilter "(distinguishedName=$TargetDN)" -ErrorAction SilentlyContinue)) {
        New-ADOrganizationalUnit `
          -Name $Name `
          -Path $Path `
          -Description $Description `
          -ProtectedFromAccidentalDeletion $true
    }
}

# Ensure Departments parent exists.
Ensure-OU `
  -Name "Departments" `
  -Path $RootOUDN `
  -Description "Department-level OU structure"

# Department examples.
Ensure-OU -Name "IT" -Path $DepartmentsOUDN -Description "IT department users and groups"
Ensure-OU -Name "Finance" -Path $DepartmentsOUDN -Description "Finance department users and groups"
Ensure-OU -Name "HR" -Path $DepartmentsOUDN -Description "Human Resources users and groups"
Ensure-OU -Name "Operations" -Path $DepartmentsOUDN -Description "Operations users and groups"

# Optional child structure inside each department.
$Departments = @("IT", "Finance", "HR", "Operations")

foreach ($Department in $Departments) {
    $DepartmentDN = "OU=$Department,$DepartmentsOUDN"

    Ensure-OU -Name "Users" -Path $DepartmentDN -Description "$Department department users"
    Ensure-OU -Name "Groups" -Path $DepartmentDN -Description "$Department department groups"
}

# Verify.
Get-ADOrganizationalUnit `
  -Filter * `
  -SearchBase $DepartmentsOUDN `
  -Properties Description |
  Select-Object Name, DistinguishedName, Description
```

# Create_Baseline_OU_Structure_Protected_From_Deletion_Skeleton
```powershell
# Run to confirm or enforce accidental deletion protection on the baseline OU tree.

Import-Module ActiveDirectory

$DomainDN = (Get-ADDomain).DistinguishedName
$RootOUDN = "OU=Corp,$DomainDN"

# View protection state.
Get-ADOrganizationalUnit `
  -Filter * `
  -SearchBase $RootOUDN `
  -Properties ProtectedFromAccidentalDeletion |
  Select-Object Name, DistinguishedName, ProtectedFromAccidentalDeletion

# Enforce protection on all OUs under root.
Get-ADOrganizationalUnit `
  -Filter * `
  -SearchBase $RootOUDN |
  Set-ADOrganizationalUnit -ProtectedFromAccidentalDeletion $true

# Also enforce on root OU.
Set-ADOrganizationalUnit `
  -Identity $RootOUDN `
  -ProtectedFromAccidentalDeletion $true

# Verify.
Get-ADOrganizationalUnit `
  -Filter * `
  -SearchBase $RootOUDN `
  -Properties ProtectedFromAccidentalDeletion |
  Select-Object Name, DistinguishedName, ProtectedFromAccidentalDeletion
```

# Create_Baseline_OU_Structure_Failure_Checks
| Symptom | Likely Cause | Check | Fix |
|---|---|---|---|
| `New-ADOrganizationalUnit` not found | ActiveDirectory module missing or RSAT not installed | `Get-Command New-ADOrganizationalUnit` | Install/import ActiveDirectory module |
| Access denied creating OU | Account lacks rights | `whoami`; check delegated permissions | Use Domain Admin or delegated admin account |
| OU created in wrong location | Missing or wrong `-Path` | Check OU distinguished name | Remove/recreate in correct parent path |
| OU created under domain root unexpectedly | `-Path` omitted | `Get-ADOrganizationalUnit -Filter *` | Always use explicit `-Path` |
| Duplicate OU creation fails | OU already exists | `Get-ADOrganizationalUnit -Identity "<ou-dn>"` | Use idempotent creation pattern |
| DN path fails | Parent OU does not exist | `Get-ADOrganizationalUnit -Identity "<parent-dn>"` | Create parent first |
| Space in OU name causes quoting issue | OU name not quoted | Review command syntax | Quote names like `"Service Accounts"` |
| Deleting OU fails | Protected from accidental deletion is enabled | Check `ProtectedFromAccidentalDeletion` | Disable protection first if deletion is intentional |
| Recursive delete removes too much | Unsafe rollback command | Review `-Recursive` target DN | Avoid recursive removal outside disposable lab |
| `Get-ADOrganizationalUnit` returns nothing | Wrong search base or wrong domain context | Check `$DomainDN` and search base | Use exact domain DN from `Get-ADDomain` |
| OUs do not appear in GUI immediately | ADUC refresh or replication delay | Refresh console, query with PowerShell | Validate with PowerShell first |
| OU description missing | Script omitted `-Description` | Query OU properties | Use `Set-ADOrganizationalUnit -Description` |
| Protection not enabled | Script omitted `-ProtectedFromAccidentalDeletion` | Query property | Set property to `$true` |
| Department structure becomes messy | OU design follows org chart too deeply | Review GPO/delegation needs | Keep baseline shallow and operational |
| Users placed in default `CN=Users` | Account creation path omitted later | Check user DN | Use explicit `-Path` in user creation scripts |
| Computers placed in default `CN=Computers` | Domain join default computer container unchanged | Check computer DN | Move computer objects or redirect default computer container in a later task |
| GPO linked to wrong OU | OU layout unclear | Check GPMC links | Document OU purpose before linking GPOs |
| Delegation assigned too high | Root OU used as broad permission boundary | Review ACLs | Delegate at narrower OU level |
| Admin accounts mixed with normal users | No separate privileged identity OU | Check user placement | Move admin accounts to `Admins` OU |
| Service accounts mixed with normal users | No separate service identity OU | Check user placement | Move service accounts to `Service Accounts` OU |
| Disabled objects remain in active OUs | No cleanup OU/process | Check disabled users/computers | Move retired objects to `Disabled Objects` OU |
