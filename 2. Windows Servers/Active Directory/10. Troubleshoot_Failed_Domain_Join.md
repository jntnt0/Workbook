# Troubleshoot_Failed_Domain_Join_Mental_Model
| Concept | Operational Meaning |
|---|---|
| Domain join | Client creates or uses a computer account in AD and establishes trust with the domain |
| Failed domain join | Join attempt fails before reboot, after reboot, or after computer account creation |
| AD DNS dependency | Client must use internal AD DNS, not public DNS, to locate domain controllers |
| DC locator | Client discovery process that uses DNS SRV records to find a domain controller |
| LDAP SRV record | `_ldap._tcp.dc._msdcs.<domain>` record used to find domain controllers |
| Kerberos SRV record | `_kerberos._tcp.<domain>` record used to find authentication service |
| Domain controller FQDN | Current hardened join flow should target a DC by FQDN when using `-Server`, such as `DC1.corp.local` |
| Join credential | Domain account allowed to join the machine to the domain |
| Computer account | AD object created or reused during join |
| Stale computer account | Old AD computer object that may block or confuse join |
| OUPath | Distinguished name where the computer account should be created |
| Secure channel | Trust relationship between client and domain after join |
| Time skew | Kerberos-sensitive clock mismatch between client and DC |
| Reboot requirement | Join may report success but not fully apply until restart |
| First rule | Treat every failed join as DNS first, credential second, object/OU third, time/Kerberos fourth, secure channel fifth |

# Troubleshoot_Failed_Domain_Join_Symptom_Map
| Symptom | Likely Layer | First Check | Do Not Start With |
|---|---|---|---|
| `The domain could not be contacted` | DNS / DC locator | `Resolve-DnsName _ldap._tcp.dc._msdcs.<domain> -Type SRV -Server <dc-ip>` | Deleting computer objects |
| `DNS name does not exist` | DNS / wrong suffix / wrong domain FQDN | `Resolve-DnsName <domain-fqdn> -Server <dc-ip>` | Resetting passwords |
| `Access is denied` | Credential / permission | Validate join account and rights | Rebuilding DNS |
| `The specified domain either does not exist or could not be contacted` | DNS / routing / firewall | DNS client server list and SRV lookup | Rejoining repeatedly |
| `The account already exists` | Stale computer account | `Get-ADComputer <client-name>` | Changing DNS blindly |
| `The network path was not found` | Connectivity / SMB / RPC / DNS | `Test-NetConnection <dc-fqdn> -Port 445` | Resetting secure channel |
| Join succeeds but client cannot log in | Reboot, DNS, DC locator, time, user issue | `Get-ComputerInfo`; `nltest /dsgetdc:<domain>` | Moving FSMO roles |
| Join succeeds but GPO does not apply | OU placement / SYSVOL / DNS / policy | `gpresult /r`; computer object DN | Rejoining client |
| Trust relationship failed | Secure channel / stale computer password | `Test-ComputerSecureChannel -Verbose` | Recreating domain |
| Join works only when using DC IP | DNS name resolution problem | `Resolve-DnsName <dc-fqdn> -Server <dns-server>` | Disabling firewall |
| Join fails with OUPath | Bad OU distinguished name | `Get-ADOrganizationalUnit -Identity "<ou-dn>"` | Changing domain admin password |
| Join takes forever | Dead first DNS server / timeout | `Get-DnsClientServerAddress` | Reinstalling NIC drivers |
| Client joins to wrong OU | OUPath omitted or typo | `Get-ADComputer <client> -Properties DistinguishedName` | Rejoining if move is enough |
| Client cannot find domain over VPN | VPN DNS/suffix/routing issue | `Get-DnsClientServerAddress`; SRV lookup | Editing AD zones |
| Secure channel repair fails | DNS/DC locator still broken or credential issue | `nltest /dsgetdc:<domain>` | Rebuilding computer |

# Troubleshoot_Failed_Domain_Join_Client_DNS_Triage_Skeleton
```powershell
# Run in elevated PowerShell on the failing Windows client.

$DomainFqdn = "corp.local"
$DcName = "DC1"
$DcFqdn = "$DcName.$DomainFqdn"
$DcIp = "10.10.10.10"
$InterfaceAlias = "Ethernet"
$EvidencePath = "C:\ADPrep\domain-join-troubleshooting"

New-Item -ItemType Directory -Force -Path $EvidencePath

# Capture baseline.
hostname | Tee-Object "$EvidencePath\client-hostname.txt"

Get-ComputerInfo |
  Select-Object CsName,CsDomain,CsPartOfDomain |
  Tee-Object "$EvidencePath\client-membership-before.txt"

Get-NetIPConfiguration |
  Tee-Object "$EvidencePath\client-netipconfiguration-before.txt"

Get-DnsClientServerAddress -AddressFamily IPv4 |
  Tee-Object "$EvidencePath\client-dns-before.txt"

# Set client DNS to AD DNS.
Set-DnsClientServerAddress `
  -InterfaceAlias $InterfaceAlias `
  -ServerAddresses $DcIp

Clear-DnsClientCache

# Verify DNS client state.
Get-DnsClientServerAddress `
  -InterfaceAlias $InterfaceAlias `
  -AddressFamily IPv4 |
  Tee-Object "$EvidencePath\client-dns-after.txt"

# Basic reachability.
Test-NetConnection $DcIp |
  Tee-Object "$EvidencePath\test-dc-ip.txt"

Test-NetConnection $DcIp -Port 53 |
  Tee-Object "$EvidencePath\test-dns-tcp-53.txt"

# Required DNS lookups.
Resolve-DnsName $DomainFqdn -Server $DcIp |
  Tee-Object "$EvidencePath\resolve-domain.txt"

Resolve-DnsName $DcFqdn -Server $DcIp |
  Tee-Object "$EvidencePath\resolve-dc-fqdn.txt"

Resolve-DnsName "_ldap._tcp.dc._msdcs.$DomainFqdn" -Type SRV -Server $DcIp |
  Tee-Object "$EvidencePath\resolve-ldap-dc-msdcs-srv.txt"

Resolve-DnsName "_kerberos._tcp.$DomainFqdn" -Type SRV -Server $DcIp |
  Tee-Object "$EvidencePath\resolve-kerberos-srv.txt"
```

# Troubleshoot_Failed_Domain_Join_Join_Command_Triage_Skeleton
```powershell
# Run on the failing client only after DNS and DC locator pass.

$DomainFqdn = "corp.local"
$NetbiosName = "CORP"
$DcFqdn = "DC1.corp.local"
$TargetOU = "OU=Workstations,OU=Corp,DC=corp,DC=local"
$EvidencePath = "C:\ADPrep\domain-join-troubleshooting"

New-Item -ItemType Directory -Force -Path $EvidencePath

$JoinCred = Get-Credential "$NetbiosName\Administrator"

# Attempt a join using explicit domain controller FQDN and target OU.
# Do not use a short DC name here; use FQDN.
try {
    Add-Computer `
      -DomainName $DomainFqdn `
      -Server $DcFqdn `
      -OUPath $TargetOU `
      -Credential $JoinCred `
      -PassThru `
      -Verbose `
      -ErrorAction Stop |
      Tee-Object "$EvidencePath\add-computer-result.txt"

    "Join command succeeded. Restart is required." |
      Tee-Object "$EvidencePath\join-status.txt"
}
catch {
    $_ |
      Format-List * -Force |
      Tee-Object "$EvidencePath\add-computer-error.txt"

    "Join command failed. Review add-computer-error.txt." |
      Tee-Object "$EvidencePath\join-status.txt"
}

# Restart only after success.
# Restart-Computer
```

# Troubleshoot_Failed_Domain_Join_OU_And_Computer_Object_Triage_Skeleton
```powershell
# Run on a domain controller or admin workstation with RSAT.

Import-Module ActiveDirectory

$ClientName = "WIN11-01"
$DomainDN = (Get-ADDomain).DistinguishedName
$TargetOU = "OU=Workstations,OU=Corp,$DomainDN"
$DisabledOU = "OU=Disabled Objects,OU=Corp,$DomainDN"
$EvidencePath = "C:\ADPrep\domain-join-troubleshooting-ad"

New-Item -ItemType Directory -Force -Path $EvidencePath

# Verify target OU exists.
Get-ADOrganizationalUnit -Identity $TargetOU |
  Tee-Object "$EvidencePath\target-ou.txt"

# Check existing computer object.
$Computer = Get-ADComputer `
  -Identity $ClientName `
  -Properties DNSHostName,DistinguishedName,Enabled,LastLogonDate,PasswordLastSet `
  -ErrorAction SilentlyContinue

if ($Computer) {
    $Computer |
      Format-List Name,DNSHostName,DistinguishedName,Enabled,LastLogonDate,PasswordLastSet |
      Tee-Object "$EvidencePath\existing-computer-object.txt"
}
else {
    "No existing computer object found for $ClientName" |
      Tee-Object "$EvidencePath\existing-computer-object.txt"
}

# If object exists in wrong OU, move it instead of rejoining blindly.
if ($Computer -and $Computer.DistinguishedName -notlike "*$TargetOU") {
    Move-ADObject `
      -Identity $Computer.DistinguishedName `
      -TargetPath $TargetOU

    Get-ADComputer `
      -Identity $ClientName `
      -Properties DistinguishedName |
      Tee-Object "$EvidencePath\computer-object-after-move.txt"
}

# If object is stale and lab-only, choose one of these actions:
# Disable-ADAccount -Identity "$ClientName$"
# Move-ADObject -Identity $Computer.DistinguishedName -TargetPath $DisabledOU
# Remove-ADComputer -Identity $ClientName -Confirm:$false
```

# Troubleshoot_Failed_Domain_Join_Credential_And_Permission_Triage_Skeleton
```powershell
# Run from a domain controller or admin workstation with RSAT.
# Purpose: validate the join account and avoid using overly privileged accounts when not needed.

Import-Module ActiveDirectory

$JoinAccount = "JoinAdmin"
$GroupsToCheck = @(
    "Domain Admins",
    "Account Operators",
    "GG_Workstation_Admins",
    "GG_Helpdesk"
)

# Confirm account exists and is enabled.
Get-ADUser `
  -Identity $JoinAccount `
  -Properties Enabled,LockedOut,PasswordExpired,PasswordLastSet,MemberOf |
  Select-Object SamAccountName,Enabled,LockedOut,PasswordExpired,PasswordLastSet,MemberOf

# Check known possible groups.
foreach ($Group in $GroupsToCheck) {
    Get-ADGroupMember $Group -ErrorAction SilentlyContinue |
      Where-Object {$_.SamAccountName -eq $JoinAccount} |
      Select-Object @{Name="Group";Expression={$Group}},Name,SamAccountName,ObjectClass
}

# If using default domain join quota behavior, ordinary users may create a limited number of computer accounts.
# In real operations, prefer delegated join rights to the target OU instead of broad Domain Admin usage.

# On the client, always prompt securely:
# $JoinCred = Get-Credential "CORP\JoinAdmin"
```

# Troubleshoot_Failed_Domain_Join_Event_Log_Triage_Skeleton
```powershell
# Run on the failing client.
# Export useful logs for domain join troubleshooting.

$EvidencePath = "C:\ADPrep\domain-join-troubleshooting"
New-Item -ItemType Directory -Force -Path $EvidencePath

# System log: Netlogon, DNS Client, Group Policy, Kerberos, service errors.
Get-WinEvent -LogName System -MaxEvents 300 |
  Where-Object {
    $_.ProviderName -like "*Netlogon*" -or
    $_.ProviderName -like "*DNS*" -or
    $_.ProviderName -like "*Kerberos*" -or
    $_.ProviderName -like "*GroupPolicy*" -or
    $_.Message -like "*domain*" -or
    $_.Message -like "*domain controller*" -or
    $_.Message -like "*trust relationship*"
  } |
  Select-Object TimeCreated,Id,LevelDisplayName,ProviderName,Message |
  Export-Csv "$EvidencePath\client-system-domainjoin-events.csv" -NoTypeInformation

# DNS Client operational log may exist on modern Windows.
Get-WinEvent -LogName "Microsoft-Windows-DNS-Client/Operational" -MaxEvents 200 -ErrorAction SilentlyContinue |
  Select-Object TimeCreated,Id,LevelDisplayName,ProviderName,Message |
  Export-Csv "$EvidencePath\client-dns-client-operational-events.csv" -NoTypeInformation

# Group Policy operational log.
Get-WinEvent -LogName "Microsoft-Windows-GroupPolicy/Operational" -MaxEvents 200 -ErrorAction SilentlyContinue |
  Select-Object TimeCreated,Id,LevelDisplayName,ProviderName,Message |
  Export-Csv "$EvidencePath\client-grouppolicy-operational-events.csv" -NoTypeInformation

# Classic domain join log file commonly used for join details.
if (Test-Path "C:\Windows\Debug\NetSetup.log") {
    Copy-Item "C:\Windows\Debug\NetSetup.log" "$EvidencePath\NetSetup.log" -Force
}
```

# Troubleshoot_Failed_Domain_Join_Failure_Checks
| Symptom | Likely Cause | Check | Fix |
|---|---|---|---|
| Domain could not be contacted | Client using wrong DNS | `Get-DnsClientServerAddress` | Set DNS to AD DNS |
| Domain name does not exist | Wrong domain FQDN or bad DNS zone | `Resolve-DnsName <domain-fqdn> -Server <dc-ip>` | Correct domain name or DNS |
| DC locator fails | Missing LDAP SRV or client DNS wrong | `Resolve-DnsName _ldap._tcp.dc._msdcs.<domain> -Type SRV` | Fix SRV records or client DNS |
| Kerberos lookup fails | Missing Kerberos SRV or wrong domain | `Resolve-DnsName _kerberos._tcp.<domain> -Type SRV` | Fix AD DNS registration |
| DC FQDN fails | DC A record missing/stale | `Resolve-DnsName <dc-fqdn> -Server <dc-ip>` | Fix DC A record or Netlogon registration |
| Join fails using short DC name | Hardened join expects DC FQDN when targeting server | Check `-Server` value | Use `DC1.corp.local` |
| Access denied | Bad credential or insufficient join rights | Validate credential/account | Use delegated join account or Domain Admin in lab |
| Target OU invalid | OUPath wrong or OU missing | `Get-ADOrganizationalUnit -Identity "<target-ou-dn>"` | Correct OU DN |
| Account already exists | Stale computer object | `Get-ADComputer <client-name>` | Reset, move, disable, or remove stale object |
| Computer object disabled | Existing object disabled | `Get-ADComputer <client> -Properties Enabled` | Enable/reset object if intended |
| Network path not found | DNS, SMB, routing, firewall | `Test-NetConnection <dc-fqdn> -Port 445` | Fix name/path/firewall |
| LDAP path blocked | Firewall/routing/security appliance | `Test-NetConnection <dc-fqdn> -Port 389` | Permit required domain join traffic |
| Kerberos path blocked | Firewall/routing/security appliance | `Test-NetConnection <dc-fqdn> -Port 88` | Permit Kerberos |
| Join takes long time | Dead first DNS server | DNS server order | Remove unreachable DNS server |
| Join succeeds but reboot missing | `-Restart` omitted | Membership state after command | Reboot client |
| Join succeeds but trust fails | Secure channel broken or stale object | `Test-ComputerSecureChannel` | Repair secure channel or rejoin |
| Secure channel repair fails | DNS/DC locator still broken | `nltest /dsgetdc:<domain>` | Fix DNS first |
| User cannot log in | DNS, time, user account, group, or DC locator issue | `whoami`, DC locator, user state | Fix account/DNS/time |
| Kerberos errors | Time skew | `Get-Date`; `w32tm /query /status` | Correct time |
| Client joined wrong OU | OUPath omitted | AD computer DN | Move AD object |
| GPO not applying | Wrong OU or SYSVOL/DC locator issue | `gpresult /r`; `Test-Path \\domain\SYSVOL` | Move object or fix SYSVOL/DNS |
| VPN domain join fails | VPN lacks DNS/routing to DC | DNS server list and SRV lookup over VPN | Fix VPN DNS/routes |
| Public DNS appears after reboot | DHCP overwrote manual DNS | `Get-DnsClientServerAddress` | Fix DHCP scope/router/VPN options |
| Local admin not elevated | Join command not run elevated | `whoami /groups` | Run elevated |
| Remote join fails | WinRM/local credential/firewall issue | `Test-WSMan`; local admin credential | Fix remoting or join locally |
| NetSetup log shows DNS failure | DNS/DC locator root cause | `C:\Windows\Debug\NetSetup.log` | Fix DNS path |
| NetSetup log shows account conflict | Stale computer object | AD computer lookup | Reset/remove/move object |
| NetSetup log shows access denied | Credential/permission issue | Join account check | Use proper delegated account |
