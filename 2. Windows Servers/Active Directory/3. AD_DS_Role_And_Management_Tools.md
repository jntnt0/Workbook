# Install_AD_DS_Role_And_Management_Tools_Mental_Model
| Concept | Operational Meaning |
|---|---|
| AD DS role | Windows Server role named `AD-Domain-Services`; installs the binaries required for Active Directory Domain Services |
| Role installation | Stages the AD DS files and services but does not create a domain or promote the server |
| Domain controller promotion | Separate later action performed with Server Manager or PowerShell such as `Install-ADDSForest` |
| Management tools | Administrative tools such as Active Directory PowerShell module, AD DS snap-ins, and AD DS deployment tooling |
| `-IncludeManagementTools` | Required with `Install-WindowsFeature` when installing management tools through PowerShell |
| ServerManager module | PowerShell module containing `Install-WindowsFeature` and `Get-WindowsFeature` |
| ADDSDeployment module | PowerShell module containing promotion/precheck cmdlets like `Test-ADDSForestInstallation` and `Install-ADDSForest` |
| ActiveDirectory module | PowerShell module used after AD DS tools are installed for commands like `Get-ADDomain`, `New-ADUser`, and `New-ADGroup` |
| Feature source | Optional path used if local Windows feature payload is unavailable |
| Pending reboot | State where feature installation completed but server should reboot before promotion |
| First rule | Installing the AD DS role is not the same as creating the domain |
| Safe stop point | After role/tools install and verification, stop before running `Install-ADDSForest` |

# Install_AD_DS_Role_And_Management_Tools_Local_Install_Skeleton
```powershell
# Run in elevated PowerShell on the future domain controller.
# This installs AD DS role binaries and management tools only.
# It does NOT promote the server to a domain controller.

$PrepPath = "C:\ADPrep"
$InstallLog = "$PrepPath\adds-role-install.log"

New-Item -ItemType Directory -Force -Path $PrepPath

# Baseline current feature state.
Get-WindowsFeature |
  Out-File "$PrepPath\windows-features-before.txt"

# Confirm ServerManager commands are available.
Get-Command Install-WindowsFeature
Get-Command Get-WindowsFeature

# Check current AD DS role state.
Get-WindowsFeature -Name AD-Domain-Services

# Preview install action.
Install-WindowsFeature `
  -Name AD-Domain-Services `
  -IncludeManagementTools `
  -WhatIf

# Install AD DS role and management tools.
$InstallResult = Install-WindowsFeature `
  -Name AD-Domain-Services `
  -IncludeManagementTools `
  -LogPath $InstallLog

# Show result.
$InstallResult

# Save result.
$InstallResult |
  Format-List * |
  Out-File "$PrepPath\adds-role-install-result.txt"

# Verify role and tools.
Get-WindowsFeature -Name AD-Domain-Services
Get-WindowsFeature -Name RSAT-AD-Tools

# Export post-install feature state.
Get-WindowsFeature |
  Out-File "$PrepPath\windows-features-after.txt"

# Stop here.
# Do not run Install-ADDSForest in this task.
```

# Install_AD_DS_Role_And_Management_Tools_Offline_Source_Skeleton
```powershell
# Use this only if the server cannot find required feature payload locally or through Windows Update.
# Example source paths:
#   D:\sources\sxs
#   \\FileServer01\WinSxS\sources\sxs

$PrepPath = "C:\ADPrep"
$SourcePath = "D:\sources\sxs"
$InstallLog = "$PrepPath\adds-role-install-offline-source.log"

New-Item -ItemType Directory -Force -Path $PrepPath

# Confirm source path exists.
Test-Path $SourcePath

# Install AD DS role and management tools using explicit source path.
$InstallResult = Install-WindowsFeature `
  -Name AD-Domain-Services `
  -IncludeManagementTools `
  -Source $SourcePath `
  -LogPath $InstallLog

$InstallResult

$InstallResult |
  Format-List * |
  Out-File "$PrepPath\adds-role-install-offline-source-result.txt"

# Verify role state.
Get-WindowsFeature -Name AD-Domain-Services
Get-WindowsFeature -Name RSAT-AD-Tools
```

```powershell
# Optional remote install pattern.

$TargetServer = "DC1"

  -Name AD-Domain-Services `
  -IncludeManagementTools `
  -ComputerName $TargetServer `

$InstallResult

# Verify remotely.
Get-WindowsFeature `
  -Name AD-Domain-Services `
  -ComputerName $TargetServer

Get-WindowsFeature `
  -Name RSAT-AD-Tools `
  -ComputerName $TargetServer
```

# Install_AD_DS_Role_And_Management_Tools_PrePromotion_Check_Skeleton
```powershell
# This verifies that the staged AD DS role can pass forest prerequisite checks.
# It does NOT promote the server.

$PrepPath = "C:\ADPrep"
$DomainFqdn = "corp.local"
$NetbiosName = "CORP"

# Prompt for DSRM password.
$DSRM = Read-Host "Enter DSRM password for AD DS forest precheck" -AsSecureString

# Run prerequisite check only.
$Precheck = Test-ADDSForestInstallation `
  -DomainName $DomainFqdn `
  -DomainNetbiosName $NetbiosName `
  -InstallDns `
  -SafeModeAdministratorPassword $DSRM

$Precheck

$Precheck |
  Out-File "$PrepPath\adds-forest-precheck.txt"

# Stop here.
# If this passes, the next workbook task is forest creation/promotion.
```

# Install_AD_DS_Role_And_Management_Tools_Failure_Checks
| Symptom | Likely Cause | Check | Fix |
|---|---|---|---|
| `Install-WindowsFeature` not recognized | Not on Windows Server, ServerManager module unavailable, wrong shell | `Get-Module -ListAvailable ServerManager` | Use Windows Server elevated PowerShell |
| Install fails with access denied | PowerShell not elevated | `whoami /groups` | Reopen PowerShell as Administrator |
| AD DS role not found | Wrong feature name or unsupported OS | `Get-WindowsFeature *AD*` | Use `AD-Domain-Services` on Windows Server |
| Management tools missing | `-IncludeManagementTools` omitted | `Get-WindowsFeature RSAT-AD-Tools` | Re-run install with `-IncludeManagementTools` |
| `Import-Module ADDSDeployment` fails | AD DS role/tools not installed correctly | `Get-WindowsFeature AD-Domain-Services` | Reinstall AD DS role and tools |
| `Import-Module ActiveDirectory` fails | RSAT AD tools missing | `Get-WindowsFeature RSAT-AD-Tools` | Install tools with `-IncludeManagementTools` or RSAT feature |
| Install asks for source files | Feature payload missing locally | Review install error and source policy | Use `-Source <path>` with valid Windows source |
| Offline source path fails | Wrong path or server lacks permission | `Test-Path <source-path>` | Correct path and permissions |
| UNC source denied | Computer account lacks read permission | Check share and NTFS permissions | Grant read to target server computer account |
| Install result shows restart needed | Feature install completed but reboot pending | `$InstallResult.RestartNeeded` | Reboot before promotion |
| Precheck fails after role install | DNS/name/password/network issue, not role install itself | Review `Test-ADDSForestInstallation` output | Fix reported blocker before promotion |
| Remote install fails | WinRM/remoting not configured or credentials wrong | `Test-WSMan <server>` | Fix remoting, firewall, and credentials |
| Feature state says installed but tools unavailable | Shell not refreshed or partial install | Open new elevated PowerShell; check modules | Reboot or reinstall management tools |
| Server was accidentally promoted | Wrong command was run after role install | Check `Get-ADDomain` and domain membership | Stop and follow DC demotion/recovery plan, not simple uninstall |
| Uninstall fails | Role in use or server promoted | Check whether server is DC | Demote DC first if promoted; otherwise retry after reboot |
| AD DS role installed on wrong server | Remote target variable or session mistake | `hostname`; check logs | Roll back wrong server and repeat on correct target |
