# Create_Baseline_Groups_And_Test_Users_Mental_Model
| Concept | Operational Meaning |
|---|---|
| User account | AD identity used for interactive logon, authentication, and authorization |
| Test user | Non-production account used to validate logon, group membership, GPO targeting, and permissions |
| Security group | AD group used to assign access rights and permissions |
| Distribution group | AD group used for mail distribution, not security authorization |
| Global group | Common group scope for users from the same domain |
| Domain local group | Common group scope for assigning access to resources in the same domain |
| Universal group | Forest-wide group scope, usually used in multi-domain designs |
| GroupCategory Security | Required when the group will be used for permissions |
| GroupScope Global | Common baseline choice for user role groups in a single-domain lab |
| SamAccountName | Legacy logon name, required for user creation and commonly used in scripts |
| UserPrincipalName | Modern logon format, usually `user@domain-fqdn` |
| Explicit Path | OU placement parameter; prevents users/groups from landing in default `CN=Users` |
| Temporary password | Initial password used for test users, normally forced to change at next logon |
| ChangePasswordAtLogon | Flag that forces user to change password on next sign-in |
| Password policy | Domain policy that can reject weak initial passwords |
| Group membership | Links users to authorization roles or future GPO/security testing |
| First rule | Create groups around access roles, not around random individual users |

# Create_Baseline_Groups_And_Test_Users_Idempotent_Create_Skeleton
```powershell
# Run on a domain controller or admin workstation with RSAT.
# Idempotent pattern: create groups/users only if missing.

Import-Module ActiveDirectory

$OutputPath = "C:\ADPrep\groups-users"
New-Item -ItemType Directory -Force -Path $OutputPath

$Domain = Get-ADDomain
$DomainDN = $Domain.DistinguishedName
$DomainFqdn = $Domain.DNSRoot

$RootOU = "OU=Corp,$DomainDN"
$UsersOU = "OU=Users,$RootOU"
$GroupsOU = "OU=Groups,$RootOU"
$AdminsOU = "OU=Admins,$RootOU"
$ServiceAccountsOU = "OU=Service Accounts,$RootOU"

$TempPassword = Read-Host "Enter temporary password for missing test accounts" -AsSecureString

function Ensure-ADGroup {
    param(
        [Parameter(Mandatory)]
        [string]$Name,

        [Parameter(Mandatory)]
        [string]$Path,

        [string]$Description = ""
    )

    $ExistingGroup = Get-ADGroup -Identity $Name -ErrorAction SilentlyContinue

    if ($ExistingGroup) {
        Write-Host "Group already exists: $Name"
    }
    else {
        New-ADGroup `
          -Name $Name `
          -SamAccountName $Name `
          -GroupScope Global `
          -GroupCategory Security `
          -Path $Path `
          -Description $Description

        Write-Host "Created group: $Name"
    }
}

function Ensure-ADUser {
    param(
        [Parameter(Mandatory)]
        [string]$Name,

        [Parameter(Mandatory)]
        [string]$SamAccountName,

        [Parameter(Mandatory)]
        [string]$UserPrincipalName,

        [Parameter(Mandatory)]
        [string]$Path,

        [Parameter(Mandatory)]
        [securestring]$AccountPassword,

        [string]$Description = "",

        [switch]$PasswordNeverExpires,

        [switch]$CannotChangePassword
    )

    $ExistingUser = Get-ADUser -Identity $SamAccountName -ErrorAction SilentlyContinue

    if ($ExistingUser) {
        Write-Host "User already exists: $SamAccountName"
    }
    else {
        New-ADUser `
          -Name $Name `
          -SamAccountName $SamAccountName `
          -UserPrincipalName $UserPrincipalName `
          -Path $Path `
          -Description $Description `
          -AccountPassword $AccountPassword `
          -Enabled $true `
          -ChangePasswordAtLogon $true `
          -PasswordNeverExpires:$PasswordNeverExpires `
          -CannotChangePassword:$CannotChangePassword

        Write-Host "Created user: $SamAccountName"
    }
}

# Export before state.
Get-ADUser -Filter * |
  Export-Csv "$OutputPath\users-before.csv" -NoTypeInformation

Get-ADGroup -Filter * |
  Export-Csv "$OutputPath\groups-before.csv" -NoTypeInformation

# Ensure groups.
Ensure-ADGroup -Name "GG_IT_Admins" -Path $GroupsOU -Description "Global security group for IT administrators"
Ensure-ADGroup -Name "GG_Workstation_Admins" -Path $GroupsOU -Description "Global security group for workstation administration"
Ensure-ADGroup -Name "GG_Server_Admins" -Path $GroupsOU -Description "Global security group for server administration"
Ensure-ADGroup -Name "GG_Helpdesk" -Path $GroupsOU -Description "Global security group for helpdesk technicians"
Ensure-ADGroup -Name "GG_VPN_Users" -Path $GroupsOU -Description "Global security group for VPN access testing"
Ensure-ADGroup -Name "GG_FileShare_Test_RW" -Path $GroupsOU -Description "Global security group for test read-write access"
Ensure-ADGroup -Name "GG_FileShare_Test_RO" -Path $GroupsOU -Description "Global security group for test read-only access"

# Ensure users.
Ensure-ADUser `
  -Name "Test User 01" `
  -SamAccountName "tuser01" `
  -UserPrincipalName "tuser01@$DomainFqdn" `
  -Path $UsersOU `
  -AccountPassword $TempPassword `
  -Description "Baseline standard test user"

Ensure-ADUser `
  -Name "Test Admin 01" `
  -SamAccountName "tadmin01" `
  -UserPrincipalName "tadmin01@$DomainFqdn" `
  -Path $AdminsOU `
  -AccountPassword $TempPassword `
  -Description "Baseline privileged test user"

Ensure-ADUser `
  -Name "svc_testapp" `
  -SamAccountName "svc_testapp" `
  -UserPrincipalName "svc_testapp@$DomainFqdn" `
  -Path $ServiceAccountsOU `
  -AccountPassword $TempPassword `
  -Description "Baseline test service account" `
  -PasswordNeverExpires `
  -CannotChangePassword

# Memberships.
Add-ADGroupMember -Identity "GG_IT_Admins" -Members "tadmin01" -ErrorAction SilentlyContinue
Add-ADGroupMember -Identity "GG_VPN_Users" -Members "tuser01" -ErrorAction SilentlyContinue
Add-ADGroupMember -Identity "GG_Workstation_Admins" -Members "GG_Helpdesk" -ErrorAction SilentlyContinue

# Export after state.
Get-ADUser -Filter * |
  Export-Csv "$OutputPath\users-after.csv" -NoTypeInformation

Get-ADGroup -Filter * |
  Export-Csv "$OutputPath\groups-after.csv" -NoTypeInformation
```

# Create_Baseline_Groups_And_Test_Users_CSV_Bulk_Create_Skeleton
```powershell
# Optional bulk create pattern.
# Build CSVs first, then import them.

Import-Module ActiveDirectory

$OutputPath = "C:\ADPrep\groups-users"
New-Item -ItemType Directory -Force -Path $OutputPath

$Domain = Get-ADDomain
$DomainDN = $Domain.DistinguishedName
$DomainFqdn = $Domain.DNSRoot

$RootOU = "OU=Corp,$DomainDN"
$UsersOU = "OU=Users,$RootOU"
$GroupsOU = "OU=Groups,$RootOU"
$AdminsOU = "OU=Admins,$RootOU"
$ServiceAccountsOU = "OU=Service Accounts,$RootOU"

$GroupsCsv = "$OutputPath\baseline-groups.csv"
$UsersCsv = "$OutputPath\baseline-users.csv"

# Create group CSV template.
@"
Name,SamAccountName,GroupScope,GroupCategory,Path,Description
GG_IT_Admins,GG_IT_Admins,Global,Security,""$GroupsOU"",""Global security group for IT administrators""
GG_Workstation_Admins,GG_Workstation_Admins,Global,Security,""$GroupsOU"",""Global security group for workstation administration""
GG_Server_Admins,GG_Server_Admins,Global,Security,""$GroupsOU"",""Global security group for server administration""
GG_Helpdesk,GG_Helpdesk,Global,Security,""$GroupsOU"",""Global security group for helpdesk technicians""
GG_VPN_Users,GG_VPN_Users,Global,Security,""$GroupsOU"",""Global security group for VPN access testing""
"@ | Set-Content $GroupsCsv

# Create user CSV template.
@"
Name,GivenName,Surname,SamAccountName,UserPrincipalName,Path,Description
Test User 01,Test,User01,tuser01,tuser01@$DomainFqdn,""$UsersOU"",""Baseline standard test user""
Test Admin 01,Test,Admin01,tadmin01,tadmin01@$DomainFqdn,""$AdminsOU"",""Baseline privileged test user""
svc_testapp,,,svc_testapp,svc_testapp@$DomainFqdn,""$ServiceAccountsOU"",""Baseline test service account""
"@ | Set-Content $UsersCsv

$TempPassword = Read-Host "Enter temporary password for imported test users" -AsSecureString

# Import groups.
Import-Csv $GroupsCsv | ForEach-Object {
    if (-not (Get-ADGroup -Identity $_.SamAccountName -ErrorAction SilentlyContinue)) {
        New-ADGroup `
          -Name $_.Name `
          -SamAccountName $_.SamAccountName `
          -GroupScope $_.GroupScope `
          -GroupCategory $_.GroupCategory `
          -Path $_.Path `
          -Description $_.Description
    }
}

# Import users.
Import-Csv $UsersCsv | ForEach-Object {
    if (-not (Get-ADUser -Identity $_.SamAccountName -ErrorAction SilentlyContinue)) {
        New-ADUser `
          -Name $_.Name `
          -GivenName $_.GivenName `
          -Surname $_.Surname `
          -SamAccountName $_.SamAccountName `
          -UserPrincipalName $_.UserPrincipalName `
          -Path $_.Path `
          -Description $_.Description `
          -AccountPassword $TempPassword `
          -Enabled $true `
          -ChangePasswordAtLogon $true
    }
}

# Verify.
Get-ADGroup -Filter * -SearchBase $GroupsOU
Get-ADUser -Filter * -SearchBase $RootOU
```

# Create_Baseline_Groups_And_Test_Users_Failure_Checks
| Symptom | Likely Cause | Check | Fix |
|---|---|---|---|
| `New-ADUser` not found | ActiveDirectory module missing | `Get-Command New-ADUser` | Install/import ActiveDirectory module |
| `New-ADGroup` not found | ActiveDirectory module missing | `Get-Command New-ADGroup` | Install/import ActiveDirectory module |
| Access denied creating users/groups | Account lacks rights | `whoami`; AD delegation | Use Domain Admin or delegated account |
| User lands in `CN=Users` | `-Path` omitted | Check user DistinguishedName | Use explicit OU `-Path` |
| Group lands in wrong container | `-Path` omitted or wrong | Check group DistinguishedName | Use explicit Groups OU path |
| User creation fails due password | Temporary password violates domain password policy | Review error and domain policy | Use stronger temporary password |
| User is created disabled | Password missing/failed or `-Enabled` not accepted due policy | `Get-ADUser <user> -Properties Enabled` | Set valid password, then enable account |
| SamAccountName already exists | Duplicate user or group name | `Get-ADUser`; `Get-ADGroup` | Use unique name or idempotent script |
| UPN suffix wrong | Domain FQDN variable wrong or custom suffix missing | Check `UserPrincipalName` | Correct UPN or add suffix in separate task |
| Cannot add group member | User/group does not exist or typo | `Get-ADUser`; `Get-ADGroup` | Create missing object or fix name |
| Nested group fails | Wrong group type/scope or typo | `Get-ADGroup <group>` | Confirm target and member group exist |
| CSV import fails | Bad quoting, commas, wrong headers, bad paths | Open CSV and validate headers | Fix CSV formatting and OU paths |
| Service account forced to change password | Script reused `ChangePasswordAtLogon` | Check user properties | Set service account password behavior intentionally |
| PasswordNeverExpires used on normal user | Wrong flag applied | `Get-ADUser <user> -Properties PasswordNeverExpires` | Set property false for normal users |
| Too much privilege assigned | Test group added to privileged built-in group | Check membership of Domain Admins/Administrators | Remove unsafe membership |
| Admin test user mixed with normal users | Wrong `-Path` | Check DistinguishedName | Move or recreate under Admins OU |
| Service account mixed with normal users | Wrong `-Path` | Check DistinguishedName | Move or recreate under Service Accounts OU |
| Group naming inconsistent | Manual naming drift | `Get-ADGroup -Filter *` | Standardize naming prefix and descriptions |
| Description missing | Script omitted Description | `Get-ADGroup -Properties Description` | Use `Set-ADGroup -Description` |
| User cannot log in | Disabled, password expired, wrong UPN, DNS/DC issue | Check user properties and DC locator | Fix account state, password, UPN, or DNS |
| User logon says password must change | Expected when `ChangePasswordAtLogon $true` | Check property | Use normal behavior for test user or disable flag for lab automation |
| Group membership not effective immediately | Logon token not refreshed | `whoami /groups` after new logon | Sign out/in or purge/reissue token |
| Removing group fails | Group has protection/dependencies or insufficient rights | Check permissions and dependencies | Remove members/dependencies first, then delete if lab-only |
| Removing users too destructive | Test accounts reused in other labs | Check group membership and usage notes | Disable first, delete later only if safe |
