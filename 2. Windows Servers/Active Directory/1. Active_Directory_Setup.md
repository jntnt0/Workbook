# Windows_Active_Directory_Setup_Mental_Model
| Concept | Operational Meaning |
|---|---|
| Active Directory Domain Services | Windows Server directory service for users, computers, groups, authentication, authorization, and policy |
| Domain controller | Server that hosts AD DS and services authentication requests |
| Forest | Top-level AD security and replication boundary |
| Domain | Administrative and authentication boundary inside a forest |
| Root domain | First domain created in a new forest |
| DNS domain name | AD domain FQDN, such as `corp.local` or `ad.contoso.com` |
| NetBIOS domain name | Legacy short domain name, such as `CORP`; keep it 15 characters or fewer |
| DSRM password | Directory Services Restore Mode password used for offline AD recovery |
| AD-integrated DNS | DNS zones stored in Active Directory and replicated with AD |
| SYSVOL | Shared folder replicated between domain controllers that stores scripts and Group Policy templates |
| NTDS database | Active Directory database, usually stored under `C:\Windows\NTDS` unless explicitly moved |
| Global Catalog | DC role that stores a partial replica of all objects in the forest for forest-wide searches |
| FSMO roles | Special AD operations roles initially held by the first domain controller in a new forest |
| SRV records | DNS records clients use to find domain controllers, LDAP, Kerberos, and GC services |
| OU | Organizational Unit used to organize objects and target Group Policy |
| Security group | AD object used to assign access rights and permissions |
| Domain join | Process of adding a computer account to the domain |
| First rule | AD setup is DNS setup with authentication layered on top; if DNS is wrong, AD will look broken |

# Windows_Active_Directory_Setup_New_Forest_Skeleton
```powershell
# Run in elevated PowerShell on the future first domain controller.

$ServerName = "DC1"
$InterfaceAlias = "Ethernet"
$DcIp = "10.10.10.10"
$PrefixLength = 24
$Gateway = "10.10.10.1"
$InitialDns = "10.10.10.10"

$DomainFqdn = "corp.local"
$NetbiosName = "CORP"
$DatabasePath = "C:\Windows\NTDS"
$LogPath = "C:\Windows\NTDS"
$SysvolPath = "C:\Windows\SYSVOL"

# Optional rename if server name is not final.
Rename-Computer -NewName $ServerName -Restart

# After reboot, reopen elevated PowerShell.

# Confirm network state.
Get-NetIPConfiguration
Get-NetAdapter

# Configure static IP only if it is not already configured.
New-NetIPAddress `
  -InterfaceAlias $InterfaceAlias `
  -IPAddress $DcIp `
  -PrefixLength $PrefixLength `
  -DefaultGateway $Gateway

Set-DnsClientServerAddress `
  -InterfaceAlias $InterfaceAlias `
  -ServerAddresses $InitialDns

# Install AD DS role and management tools.
Install-WindowsFeature `
  -Name AD-Domain-Services `
  -IncludeManagementTools

# Confirm role install.
Get-WindowsFeature AD-Domain-Services

# Load AD DS deployment module.
Import-Module ADDSDeployment

# Use an interactive DSRM password prompt.
$DSRM = Read-Host "Enter Directory Services Restore Mode password" -AsSecureString

# Run prerequisite check.
Test-ADDSForestInstallation `
  -DomainName $DomainFqdn `
  -DomainNetbiosName $NetbiosName `
  -InstallDns `
  -DatabasePath $DatabasePath `
  -LogPath $LogPath `
  -SysvolPath $SysvolPath `
  -SafeModeAdministratorPassword $DSRM

# Create new forest.
Install-ADDSForest `
  -DomainName $DomainFqdn `
  -DomainNetbiosName $NetbiosName `
  -InstallDns `
  -DatabasePath $DatabasePath `
  -LogPath $LogPath `
  -SysvolPath $SysvolPath `
  -SafeModeAdministratorPassword $DSRM `
  -Force

# Server reboots automatically after promotion.
```

# Windows_Active_Directory_Setup_Post_Install_Baseline_Skeleton
```powershell
# Run after reboot and domain admin sign-in.

$DomainFqdn = "corp.local"
$NetbiosName = "CORP"
$DcName = "DC1"
$InterfaceAlias = "Ethernet"

# Import modules.
Import-Module ActiveDirectory
Import-Module DnsServer

# Confirm domain and forest.
Get-ADDomain
Get-ADForest

# Confirm the local machine is a domain controller.
Get-ADDomainController -Identity $env:COMPUTERNAME

# Confirm DC locator works.
nltest /dsgetdc:$DomainFqdn

# Confirm SYSVOL and NETLOGON.
net share

# Point DC DNS client to itself after promotion.
Set-DnsClientServerAddress `
  -InterfaceAlias $InterfaceAlias `
  -ServerAddresses "127.0.0.1"

# Confirm DNS zones.
Get-DnsServerZone

# Confirm key AD DNS records.
Resolve-DnsName "_ldap._tcp.dc._msdcs.$DomainFqdn" -Type SRV
Resolve-DnsName "_kerberos._tcp.$DomainFqdn" -Type SRV
Resolve-DnsName "$DcName.$DomainFqdn"

# Confirm basic AD containers.
Get-ADDomain | Select-Object DNSRoot, NetBIOSName, DistinguishedName, DomainMode
Get-ADForest | Select-Object Name, RootDomain, ForestMode, GlobalCatalogs
```

# Windows_Active_Directory_Setup_OU_Group_User_Skeleton
```powershell
# Run on a domain controller or management server with RSAT.

Import-Module ActiveDirectory

$Domain = Get-ADDomain
$DomainDN = $Domain.DistinguishedName
$DomainFqdn = $Domain.DNSRoot

$RootOU = "OU=Corp,$DomainDN"
$UsersOU = "OU=Users,$RootOU"
$GroupsOU = "OU=Groups,$RootOU"
$ComputersOU = "OU=Computers,$RootOU"
$ServersOU = "OU=Servers,$RootOU"
$AdminsOU = "OU=Admins,$RootOU"

# Create baseline OU structure.
New-ADOrganizationalUnit -Name "Corp" -Path $DomainDN
New-ADOrganizationalUnit -Name "Users" -Path $RootOU
New-ADOrganizationalUnit -Name "Groups" -Path $RootOU
New-ADOrganizationalUnit -Name "Computers" -Path $RootOU
New-ADOrganizationalUnit -Name "Servers" -Path $RootOU
New-ADOrganizationalUnit -Name "Admins" -Path $RootOU

# Create baseline groups.
New-ADGroup `
  -Name "GG_IT_Admins" `
  -SamAccountName "GG_IT_Admins" `
  -GroupScope Global `
  -GroupCategory Security `
  -Path $GroupsOU `
  -Description "Global group for IT administrators"

New-ADGroup `
  -Name "GG_Workstation_Admins" `
  -SamAccountName "GG_Workstation_Admins" `
  -GroupScope Global `
  -GroupCategory Security `
  -Path $GroupsOU `
  -Description "Global group for delegated workstation administration"

# Create baseline test user.
$TempPassword = Read-Host "Temporary password for test user" -AsSecureString

New-ADUser `
  -Name "Test User" `
  -GivenName "Test" `
  -Surname "User" `
  -SamAccountName "tuser" `
  -UserPrincipalName "tuser@$DomainFqdn" `
  -Path $UsersOU `
  -AccountPassword $TempPassword `
  -Enabled $true `
  -ChangePasswordAtLogon $true

# Add test user to baseline group only if appropriate for lab.
Add-ADGroupMember `
  -Identity "GG_IT_Admins" `
  -Members "tuser"

# Verify.
Get-ADOrganizationalUnit -Filter * -SearchBase $RootOU
Get-ADGroup -Filter * -SearchBase $GroupsOU
Get-ADUser -Identity "tuser" -Properties MemberOf
```

# Windows_Active_Directory_Setup_DNS_Validation_Skeleton
```powershell
# Run on the domain controller.

Import-Module DnsServer

$DomainFqdn = "corp.local"
$DcName = "DC1"
$DcIp = "10.10.10.10"

# List zones.
Get-DnsServerZone

# Confirm domain zone.
Get-DnsServerZone -Name $DomainFqdn

# Confirm _msdcs zone or records.
Get-DnsServerZone | Where-Object {$_.ZoneName -like "*_msdcs*" -or $_.ZoneName -eq $DomainFqdn}

# Confirm DC host record.
Resolve-DnsName "$DcName.$DomainFqdn" -Type A -Server 127.0.0.1

# Confirm LDAP SRV records.
Resolve-DnsName "_ldap._tcp.dc._msdcs.$DomainFqdn" -Type SRV -Server 127.0.0.1

# Confirm Kerberos SRV records.
Resolve-DnsName "_kerberos._tcp.$DomainFqdn" -Type SRV -Server 127.0.0.1

# Confirm Global Catalog SRV records.
Resolve-DnsName "_gc._tcp.$DomainFqdn" -Type SRV -Server 127.0.0.1

# Confirm reverse lookup if reverse zone exists.
Resolve-DnsName $DcIp -Server 127.0.0.1
```

# Windows_Active_Directory_Setup_Domain_Join_Client_Skeleton
```powershell
# Run in elevated PowerShell on a Windows client or member server.

$DomainFqdn = "corp.local"
$DnsServerIp = "10.10.10.10"
$InterfaceAlias = "Ethernet"

# Point client to AD DNS.
Set-DnsClientServerAddress `
  -InterfaceAlias $InterfaceAlias `
  -ServerAddresses $DnsServerIp

Clear-DnsClientCache

# Verify AD DNS discovery before domain join.
Resolve-DnsName $DomainFqdn -Server $DnsServerIp
Resolve-DnsName "_ldap._tcp.dc._msdcs.$DomainFqdn" -Type SRV -Server $DnsServerIp
Resolve-DnsName "_kerberos._tcp.$DomainFqdn" -Type SRV -Server $DnsServerIp

# Join domain.
Add-Computer `
  -DomainName $DomainFqdn `
  -Credential "$DomainFqdn\Administrator" `
  -Restart

# After reboot, verify membership.
Get-ComputerInfo | Select-Object CsName, CsDomain, CsPartOfDomain
whoami
```

# Windows_Active_Directory_Setup_Failure_Checks
| Symptom | Likely Cause | Check | Fix |
|---|---|---|---|
| AD DS role install fails | PowerShell not elevated or missing Windows feature source | Confirm admin shell; `Get-WindowsFeature AD-Domain-Services` | Run elevated PowerShell and retry install |
| Management tools missing | `-IncludeManagementTools` was omitted | Check Server Manager tools or RSAT cmdlets | Re-run install with `-IncludeManagementTools` |
| Forest precheck fails | Bad DNS, bad hostname, weak DSRM password, duplicate name, network issue | Review `Test-ADDSForestInstallation` output | Fix exact blocker before promotion |
| Promotion fails during DNS checks | DNS client setting is wrong or existing DNS unreachable | `Get-DnsClientServerAddress`; `Resolve-DnsName` | Point to valid DNS or use local DNS plan |
| NetBIOS name rejected | Name too long or invalid | Check `<netbios-name>` length | Use 15 characters or fewer |
| Server does not reboot after promotion | Promotion incomplete or `NoRebootOnCompletion` used | Check AD DS logs and pending reboot | Reboot manually if promotion succeeded |
| Cannot log in after reboot | Using local account instead of domain account | Try `<NETBIOS>\Administrator` | Use domain credentials |
| `Get-ADDomain` fails | AD module missing, DC promotion failed, service not ready | Import module, check services | Confirm promotion and AD DS services |
| SYSVOL missing | DFSR/SYSVOL not initialized yet or replication problem | `net share`; event logs | Wait briefly, check DFSR and AD logs |
| NETLOGON missing | Netlogon not ready or DNS/SYSVOL issue | `net share`; `Get-Service Netlogon` | Restart Netlogon only after AD/DNS are healthy |
| SRV records missing | Netlogon registration failed or DNS zone unhealthy | `Resolve-DnsName _ldap._tcp.dc._msdcs.<domain> -Type SRV` | Fix DNS, then restart Netlogon |
| Domain join fails | Client points to public DNS or wrong DNS server | `Get-DnsClientServerAddress` on client | Point client to AD DNS |
| Domain join finds no DC | Missing SRV records or blocked DNS | SRV lookup from client | Fix AD DNS and port 53 path |
| Client joins but logon slow | DNS timeout, unreachable DC, bad time sync | DNS and `w32tm /query /status` | Fix DNS server list and time sync |
| Kerberos errors | Time skew or DNS/DC discovery problem | `w32tm /query /status`; SRV lookups | Fix time source and DNS |
| User creation fails | Running without domain rights or wrong OU path | Check `whoami`; verify OU DN | Use domain admin or delegated account and correct DN |
| OU creation fails | Parent path wrong or OU already exists | `Get-ADOrganizationalUnit -Filter *` | Correct DN or skip existing OU |
| Group membership fails | Group or user does not exist | `Get-ADUser`; `Get-ADGroup` | Create missing object or fix name |
| DNS zone missing after promotion | DNS not installed or zone failed to create | `Get-DnsServerZone`; promotion logs | Install DNS role or recreate AD-integrated zone carefully |
| DC host record wrong | Static IP changed after promotion or stale DNS record | `Resolve-DnsName <dc-fqdn>` | Correct DNS A record and DC IP settings |
| Reverse lookup fails | No reverse zone or PTR record | `Resolve-DnsName <dc-ip>` | Create reverse zone and PTR if required |
| Existing domain name conflicts | Chosen AD FQDN overlaps with public or existing internal namespace | Check current DNS zones and business domain usage | Choose clean internal namespace or plan split DNS |
| First DC uses DHCP | IP can change and break DNS/AD | `Get-NetIPConfiguration` | Configure static IP |
| Time source unreliable | First DC has bad time source | `w32tm /query /status` | Configure reliable external or upstream time source after build |
| Re-running promotion creates confusion | Partial promotion state remains | Check Server Manager, AD DS logs, services | Revert snapshot in lab or clean up carefully |
| Removing only DC breaks domain | Domain depends on the only DC | Count DCs and FSMO holders | Do not demote until replacement DC exists and is verified |
