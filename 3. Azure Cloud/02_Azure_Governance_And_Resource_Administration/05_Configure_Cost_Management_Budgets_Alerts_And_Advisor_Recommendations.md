# Configure_Cost_Management_Budgets_Alerts_And_Advisor_Recommendations_Budget_Bicep_Skeleton

```bicep
// File: budget-subscription.bicep
// Purpose: Create a subscription-scope Cost Management budget with actual and forecasted notifications.
// Deploy at subscription scope.

targetScope = 'subscription'

@description('Cost Management budget name.')
param budgetName string = 'budget-platform-monthly-001'

@description('Budget amount in the billing currency for the selected scope.')
param amount int = 100

@description('Budget time grain. Common values: Monthly, Quarterly, Annually.')
@allowed([
  'Monthly'
  'Quarterly'
  'Annually'
])
param timeGrain string = 'Monthly'

@description('Budget start date. Use YYYY-MM-DD format, usually the first day of the month.')
param startDate string

@description('Budget end date. Use YYYY-MM-DD format.')
param endDate string

@description('Email contacts that receive budget notifications.')
param contactEmails array = [
  'cloudops@contoso.com'
  'financeops@contoso.com'
]

@description('Optional Azure Monitor action group resource IDs.')
param contactGroups array = []

@description('Actual cost warning threshold percentage.')
param actualWarningThreshold int = 50

@description('Actual cost escalation threshold percentage.')
param actualEscalationThreshold int = 80

@description('Forecasted cost threshold percentage.')
param forecastThreshold int = 100

resource budget 'Microsoft.Consumption/budgets@2023-11-01' = {
  name: budgetName
  properties: {
    category: 'Cost'
    amount: amount
    timeGrain: timeGrain
    timePeriod: {
      startDate: startDate
      endDate: endDate
    }
    notifications: {
      Actual_50_Percent: {
        enabled: true
        operator: 'GreaterThan'
        threshold: actualWarningThreshold
        thresholdType: 'Actual'
        contactEmails: contactEmails
        contactGroups: contactGroups
      }
      Actual_80_Percent: {
        enabled: true
        operator: 'GreaterThan'
        threshold: actualEscalationThreshold
        thresholdType: 'Actual'
        contactEmails: contactEmails
        contactGroups: contactGroups
      }
      Forecast_100_Percent: {
        enabled: true
        operator: 'GreaterThan'
        threshold: forecastThreshold
        thresholdType: 'Forecasted'
        contactEmails: contactEmails
        contactGroups: contactGroups
      }
    }
  }
}
```

```bash
# Deploy budget Bicep at subscription scope.

TENANT_ID="<tenant-id>"
SUBSCRIPTION_ID="<subscription-id>"
DEPLOYMENT_NAME="deploy-cost-budget-001"
LOCATION="eastus"
BUDGET_NAME="<budget-name>"

  --name "$DEPLOYMENT_NAME" \
```

```bash
# Purpose: create an Azure Monitor action group for reusable cost alert notifications.
# Budget notification behavior varies by scope and method, but action groups are useful for standardized alert targets.

TENANT_ID="<tenant-id>"
SUBSCRIPTION_ID="<subscription-id>"
ACTION_GROUP_RG="<action-group-resource-group>"
ACTION_GROUP_NAME="<action-group-name>"
ACTION_GROUP_SHORT_NAME="<action-group-short-name>"

EMAIL_RECEIVER_NAME="CloudOpsEmail"
EMAIL_ADDRESS="<alert-email-1>"
EVIDENCE_PATH="$HOME/azure-governance/05-cost-budgets-alerts-advisor"

mkdir -p "$EVIDENCE_PATH"

az login --tenant "$TENANT_ID"
az account set --subscription "$SUBSCRIPTION_ID"

# Create resource group for monitoring artifacts if needed.
az group create \
  --name "$ACTION_GROUP_RG" \
  --location "eastus" \
  -o json | tee "$EVIDENCE_PATH/action-group-rg-create.json"

# Create action group with email receiver.
az monitor action-group create \
  --resource-group "$ACTION_GROUP_RG" \
  --name "$ACTION_GROUP_NAME" \
  --short-name "$ACTION_GROUP_SHORT_NAME" \
  --action email "$EMAIL_RECEIVER_NAME" "$EMAIL_ADDRESS" \
  -o json | tee "$EVIDENCE_PATH/action-group-created.json"

# Capture action group resource ID.
ACTION_GROUP_ID=$(az monitor action-group show \
  --resource-group "$ACTION_GROUP_RG" \
  --name "$ACTION_GROUP_NAME" \
  --query id \
  -o tsv)

echo "$ACTION_GROUP_ID" | tee "$EVIDENCE_PATH/action-group-id.txt"

# Validate action group.

  --resource-group "$ACTION_GROUP_RG" \
  --name "$ACTION_GROUP_NAME" \
```

| Budget alert delayed | Cost data is not real time | Portal | Cost Management > Cost analysis | Wait for cost ingestion and daily budget evaluation |
| Forecast alert not firing | Forecast does not exceed configured threshold | Portal | Budget alert conditions | Lower forecast threshold or review forecast trend |
| Actual alert not firing | Actual cost has not crossed threshold | Portal | Budget details | Validate threshold percentage and actual spend |
| Budget missing expected costs | Filters exclude charges or scope is wrong | Portal | Budget > Filters | Review scope, filters, publisher type, charge type, and resource group filters |
| Multi-subscription budget behaves unexpectedly | Multiple currencies in scope | Portal | Scope billing currency review | Use separate budgets for different currencies |
| Cost analysis does not show new resources yet | Cost data delay | Portal | Cost Management > Cost analysis | Wait for cost and usage data availability |
| New subscription has no Cost Management features | New subscriptions can take time before full Cost Management availability | Portal | Cost Management + Billing | Wait and recheck later |
| Cost alerts page has active old alerts | Alerts were never dismissed | Portal | Cost Management > Cost alerts | Review and dismiss no-longer-relevant alerts |

| Advisor VM recommendation missing | VM lacks required activity history or utilization does not meet rule | Portal | Advisor recommendations | Review VM metrics manually |
| Advisor resize action would cause outage | VM resize can restart running VM | Portal | Recommendation details and VM size operation | Schedule change window and validate workload owner approval |
| Advisor savings estimate seems too high | Potential yearly savings may assume shutdown or full action | Portal | Recommendation details | Validate real savings against workload requirements |
