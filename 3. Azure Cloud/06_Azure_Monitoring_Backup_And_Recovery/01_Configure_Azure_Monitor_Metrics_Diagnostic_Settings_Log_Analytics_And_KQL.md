# Configure_Azure_Monitor_Metrics_Diagnostic_Settings_Log_Analytics_And_KQL_Mental_Model

| Concept | Operational Meaning |
|---|---|
| Azure Monitor | Central Azure service for collecting, analyzing, visualizing, and alerting on platform and workload telemetry |
| Platform metrics | Numeric time-series data collected automatically for many Azure resources |
| Metrics Explorer | Azure portal tool used to graph platform metrics, apply aggregations, filter dimensions, and split charts |
| Activity Log | Subscription-level control plane log for create, update, delete, and other Azure Resource Manager operations |
| Resource logs | Service-specific logs generated by Azure resources, but not collected into a destination until diagnostic settings are configured |
| Diagnostic setting | Routing rule that sends selected logs and metrics from a resource to Log Analytics, Storage, Event Hubs, or partner destinations |
| Log Analytics workspace | Azure Monitor Logs workspace where collected log records are stored and queried |
| KQL | Kusto Query Language used to query Azure Monitor Logs |
| AzureMetrics table | Common table used when platform metrics are routed to Log Analytics |
| AzureActivity table | Table used when Activity Log data is routed to Log Analytics |
| AzureDiagnostics table | Legacy common table used by many diagnostic settings unless resource-specific tables are selected |
| Resource-specific tables | Preferred table model for many services because logs land in service-specific tables instead of the generic AzureDiagnostics table |
| Category group | Diagnostic setting shortcut such as `allLogs` or `audit` that collects a predefined group of resource logs |
| First rule | Metrics may be visible before diagnostic settings exist, but logs in Log Analytics require routing |
| Blunt rule | If no diagnostic setting points to Log Analytics, KQL queries will not magically show resource logs |

  --resource-group "$MONITOR_RG" \
  --workspace-name "$LAW_NAME" \
  --query customerId \

echo "Workspace Customer ID: $LAW_CUSTOMER_ID"

  --resource-group "$MONITOR_RG" \

# Configure_Azure_Monitor_Metrics_Diagnostic_Settings_Log_Analytics_And_KQL_Diagnostic_Settings_Skeleton

```bash
# Run from Azure Cloud Shell or a workstation with Azure CLI.
# Purpose: route target resource logs and metrics to a Log Analytics workspace.

MONITOR_RG="<resource-group>"
LAW_NAME="<workspace-name>"
TARGET_RESOURCE_ID="<target-resource-id>"
DIAG_NAME="<diagnostic-setting-name>"

LAW_ID=$(az monitor log-analytics workspace show \
  --resource-group "$MONITOR_RG" \
  --workspace-name "$LAW_NAME" \
  --query id \
  --output tsv)

# Confirm the target resource exists.
az resource show \
  --ids "$TARGET_RESOURCE_ID" \
  --output table

# Review available diagnostic categories.
az monitor diagnostic-settings categories list \
  --resource "$TARGET_RESOURCE_ID" \
  --output table

# Create diagnostic setting using category group allLogs and AllMetrics.
az monitor diagnostic-settings create \
  --name "$DIAG_NAME" \
  --resource "$TARGET_RESOURCE_ID" \
  --workspace "$LAW_ID" \
  --logs '[{"categoryGroup":"allLogs","enabled":true}]' \
  --metrics '[{"category":"AllMetrics","enabled":true}]' \
  --export-to-resource-specific true

# Confirm diagnostic setting.
az monitor diagnostic-settings show \
  --name "$DIAG_NAME" \
  --resource "$TARGET_RESOURCE_ID" \
  --output jsonc
```

# Configure_Azure_Monitor_Metrics_Diagnostic_Settings_Log_Analytics_And_KQL_PowerShell_Skeleton

```powershell
# Run from PowerShell with Az modules installed.
# Purpose: create a diagnostic setting to send logs and metrics to Log Analytics.

Connect-AzAccount

$SubscriptionId = "<subscription-id>"
$MonitorRg = "<resource-group>"
$WorkspaceName = "<workspace-name>"
$TargetResourceId = "<target-resource-id>"
$DiagName = "<diagnostic-setting-name>"

Set-AzContext -SubscriptionId $SubscriptionId

$Law = Get-AzOperationalInsightsWorkspace `
  -ResourceGroupName $MonitorRg `
  -Name $WorkspaceName

$Metric = New-AzDiagnosticSettingMetricSettingsObject `
  -Enabled $true `
  -Category AllMetrics

$Log = New-AzDiagnosticSettingLogSettingsObject `
  -Enabled $true `
  -CategoryGroup allLogs

New-AzDiagnosticSetting `
  -Name $DiagName `
  -ResourceId $TargetResourceId `
  -WorkspaceId $Law.ResourceId `
  -Log $Log `
  -Metric $Metric `
  -Verbose

Get-AzDiagnosticSetting `
  -ResourceId $TargetResourceId
```

# Configure_Azure_Monitor_Metrics_Diagnostic_Settings_Log_Analytics_And_KQL_KQL_Query_Skeleton

```kusto
// Purpose: verify that Azure Monitor data is landing in Log Analytics.
// Run inside the Log Analytics workspace Logs blade.

//
// 1. Confirm recent activity log records if Activity Log export is enabled.
//
AzureActivity
| where TimeGenerated > ago(24h)
| summarize Count = count() by ActivityStatusValue, OperationNameValue
| order by Count desc

//
// 2. Confirm platform metrics routed to Log Analytics.
//
AzureMetrics
| where TimeGenerated > ago(24h)
| summarize Count = count(), LastRecord = max(TimeGenerated) by ResourceProvider, MetricName, ResourceId
| order by LastRecord desc

//
// 3. Confirm generic diagnostic logs if the resource uses AzureDiagnostics.
//
AzureDiagnostics
| where TimeGenerated > ago(24h)
| summarize Count = count(), LastRecord = max(TimeGenerated) by ResourceProvider, Category, Resource
| order by LastRecord desc

//
// 4. Find recent records across common Azure Monitor tables.
//
union isfuzzy=true
    AzureActivity,
    AzureMetrics,
    AzureDiagnostics
| where TimeGenerated > ago(24h)
| summarize Count = count(), LastRecord = max(TimeGenerated) by Type
| order by Count desc

//
// 5. Check ingestion by resource ID.
//
let TargetResourceId = "<target-resource-id>";
union isfuzzy=true
    AzureActivity,
    AzureMetrics,
    AzureDiagnostics
| where TimeGenerated > ago(24h)
| where _ResourceId =~ TargetResourceId or ResourceId =~ TargetResourceId
| summarize Count = count(), LastRecord = max(TimeGenerated) by Type
| order by LastRecord desc

//
// 6. Basic error hunting in generic diagnostic logs.
//
AzureDiagnostics
| where TimeGenerated > ago(24h)
| where Level in ("Error", "Critical") or ResultType has_any ("Failed", "Error")
| project TimeGenerated, ResourceProvider, Resource, Category, Level, ResultType, OperationName
| order by TimeGenerated desc

//
// 7. Metric trend example.
//
AzureMetrics
| where TimeGenerated > ago(6h)
| where MetricName == "<metric-name>"
| summarize AvgValue = avg(Average), MaxValue = max(Maximum) by bin(TimeGenerated, 15m), ResourceId
| order by TimeGenerated asc

//
// 8. Workspace data volume by table.
//
Usage
| where TimeGenerated > ago(7d)
| summarize GB = sum(Quantity) / 1024 by DataType
| order by GB desc
```

# Configure_Azure_Monitor_Metrics_Diagnostic_Settings_Log_Analytics_And_KQL_Rollback

| Change | Rollback Command / Action | Risk |
|---|---|---|
| Diagnostic setting created on wrong resource | `az monitor diagnostic-settings delete --name "<diagnostic-setting-name>" --resource "<target-resource-id>"` | Stops log and metric routing for that resource |
| Diagnostic setting sent too much data | Update setting to use `audit` or specific categories instead of `allLogs` | May reduce troubleshooting visibility |
| Diagnostic setting used wrong workspace | Delete and recreate setting with correct workspace ID | Temporary telemetry gap |
| Workspace created with wrong name | Create new workspace and repoint diagnostic settings | Historical data stays in old workspace |
| Workspace retention set too high | `az monitor log-analytics workspace update --resource-group "<resource-group>" --workspace-name "<workspace-name>" --retention-time <days>` | Shorter retention can remove old data |
| Metrics sent to Log Analytics unnecessarily | Remove `AllMetrics` from diagnostic setting | Metrics still remain available in Metrics Explorer |
| Resource-specific mode caused table confusion | Recreate diagnostic setting without `--export-to-resource-specific true` if generic table is required | Table names and query patterns change |
| Evidence folder created locally | `rm -rf ./AzureMonitor-Baseline` or delete folder manually | Deletes exported evidence only |
| Log Analytics workspace no longer needed | `az monitor log-analytics workspace delete --resource-group "<resource-group>" --workspace-name "<workspace-name>"` | Deletes workspace and impacts all resources sending logs there |

# Configure_Azure_Monitor_Metrics_Diagnostic_Settings_Log_Analytics_And_KQL_Failure_Checks

| Symptom | Likely Layer | First Check | Do Not Start With |
|---|---|---|---|
| Metrics visible in portal but not Log Analytics | Diagnostic setting not routing metrics | `az monitor diagnostic-settings show ...` | Recreating the resource |
| Logs blade shows no records | No diagnostic setting or ingestion delay | Confirm diagnostic setting and wait a few minutes | Changing KQL randomly |
| `AzureDiagnostics` is empty | Resource-specific tables are enabled or no data ingested | Check diagnostic setting table mode and resource-specific tables | Assuming logging failed |
| `AzureMetrics` is empty | Metrics not routed to Log Analytics | Confirm `AllMetrics` is enabled in diagnostic setting | Recreating workspace |
| `AzureActivity` is empty | Activity Log export not configured | Check subscription Activity Log diagnostic export | Troubleshooting target resource logs |
| `categoryGroup allLogs` fails | Resource does not support category groups | List categories and use individual log categories | Deleting the workspace |
| Diagnostic setting create fails | RBAC or destination issue | Confirm Monitoring Contributor or equivalent and workspace access | Changing provider registration only |
| Storage or Event Hubs destination not accepted | Destination region or firewall restriction | Confirm destination region and trusted Microsoft services setting | Rebuilding the source resource |
| Query returns partial data | Wrong time range or query scope | Check Log Analytics time picker and workspace scope | Editing diagnostic setting first |
| KQL syntax error | Query language issue | Run a smaller `take 10` query first | Assuming ingestion failed |
| Metric query returns empty output | Metric has no recent samples or wrong metric name | Run `az monitor metrics list-definitions` | Recreating diagnostic setting |
| Cost increases after enabling logs | Too many categories or long retention | Review `Usage` table by DataType | Turning off all monitoring |
| Duplicate records | Multiple diagnostic settings or duplicate agent collection | List diagnostic settings and data collection rules | Deleting the workspace |
| Records delayed | Normal ingestion latency or resource inactive | Check latest `TimeGenerated` and generate test activity | Recreating target resource |
| Resource-specific table unknown | Table discovery issue | Use workspace table list, Logs UI schema, or `search` carefully | Forcing everything into AzureDiagnostics |
| PowerShell `New-AzDiagnosticSetting` fails | Az module version or parameter mismatch | Update Az module and check cmdlet help | Switching tools mid-change |
