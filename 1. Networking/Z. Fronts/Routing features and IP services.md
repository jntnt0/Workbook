Recommended_Grouping_Rules

|   |   |
|---|---|
|Rule|Meaning|
|Keep CEF before ECMP and polarization|ECMP and polarization are CEF forwarding behaviors, so the base CEF mental model must come first.|
|Keep route-map before PBR|PBR uses route maps, so route-map logic needs to exist before conditional forwarding.|
|Keep DHCP server before relay|Relay only makes sense once the server-side DORA/pool model is understood.|
|Keep HSRP before VRRP before GLBP|HSRP teaches the base virtual gateway idea, VRRP teaches the standards-based equivalent, GLBP adds load sharing.|
|Keep BFD before IP SLA/PfR3|BFD is fast failure detection. IP SLA is active probing. PfR3 is policy-based performance path control built on measurement concepts.|
|Keep traceroute and performance monitor near the end|These are validation and visibility tools. They help prove behavior after the forwarding and control mechanisms are understood.|

The only duplicate-style item is bfd-final and bidrectional-forwarding-detection-final; I would keep both under one BFD note unless the labs are materially different.
