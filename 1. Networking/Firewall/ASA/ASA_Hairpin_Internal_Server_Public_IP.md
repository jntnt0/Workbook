# ASA_Hairpin_Internal_Server_Public_IP_Mental_Model
| Concept                          | Operational Meaning                                                                                               |
| -------------------------------- | ----------------------------------------------------------------------------------------------------------------- |
| Hairpin NAT                      | Traffic enters and exits the same ASA interface                                                                   |
| Internal server public IP access | Inside client reaches an inside server by using the server's public NAT address                                   |
| Same-interface forwarding        | ASA blocks same-interface forwarding unless `same-security-traffic permit intra-interface` is configured          |
| Destination NAT                  | ASA translates the public server IP back to the real inside server IP                                             |
| Source PAT                       | ASA translates the inside client source to the ASA inside interface so the server returns traffic through the ASA |
| Twice NAT                        | Required because the ASA matches source and destination and may translate both in one rule                        |
| `(inside,inside)` interface pair | Correct NAT direction when inside clients enter the ASA and are sent back out the inside interface                |
| Static server NAT                | Publishes the inside server for outside users                                                                     |
| Hairpin NAT rule                 | Special inside-to-inside rule for inside users using the public IP                                                |
| ACL real-address rule            | ASA 8.3+ ACLs reference the real server IP, not the public NAT IP                                                 |
| DNS doctoring                    | Alternative design where ASA rewrites DNS replies so inside clients use the real server IP                        |
| Split DNS                        | Cleaner design where internal DNS resolves the server name directly to the inside real IP                         |
| Blunt rule                       | Static outside NAT alone does not solve inside users reaching the public IP                                       |

| ---: | ----------------------------------------------------------------------------- | ------ | ------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------ | ----------------------------------------------------------- |
|    1 | Identify the active hairpin NAT rule                                          | ASA    | `show nat detail`                                                                                                                                                                                | Exact `(inside,inside)` NAT rule and objects are identified |

|    4 | Remove all-services hairpin NAT if used                                       | ASA    | `no nat (inside,inside) source dynamic <inside-clients-og> interface destination static <server-public-object> <server-real-object>`                                                             | Broad hairpin NAT is removed                                |
|    5 | Remove inside ACL permit if created only for this lab                         | ASA    | `no access-list <inside-acl> extended permit tcp <inside-client-network> <inside-client-mask> host <server-real-ip> eq <tcp-port>`                                                               | Inside ACL entry is removed                                 |
|    6 | Remove service object if unused                                               | ASA    | `no object service <service-object>`                                                                                                                                                             | Service object is removed                                   |

# ASA_Hairpin_Internal_Server_Public_IP_Failure_Checks
| Symptom                                          | Command                                                                                          | What Usually Broke                                                                            |
| ------------------------------------------------ | ------------------------------------------------------------------------------------------------ | --------------------------------------------------------------------------------------------- |
| Inside client cannot reach public server IP      | `packet-tracer input inside tcp <inside-client-ip> 12345 <server-public-ip> <tcp-port> detailed` | Missing or incorrect `(inside,inside)` hairpin NAT                                            |
| Packet-tracer drops same-interface flow          | `show running-config same-security-traffic`                                                      | Missing `same-security-traffic permit intra-interface`                                        |
| NAT rule has zero hits                           | `show nat detail`                                                                                | Wrong source object, destination object, service object, or interface pair                    |
| Packet-tracer matches Dynamic PAT instead        | `show nat`                                                                                       | Hairpin NAT is ordered below a broader NAT rule                                               |
| Server replies directly to client instead of ASA | `show xlate` and `show conn long`                                                                | Missing source PAT to ASA inside interface                                                    |
| Outside users work but inside users fail         | `packet-tracer input inside tcp <client-ip> 12345 <public-ip> <port> detailed`                   | Outside static NAT exists, but inside hairpin NAT is missing                                  |
| Inside users work but outside users fail         | `packet-tracer input outside tcp <outside-test-ip> 12345 <server-public-ip> <port> detailed`     | Outside static NAT, outside ACL, route, or service listener issue                             |
| ACL blocks traffic                               | `show access-list <inside-acl>`                                                                  | ACL references public IP instead of real server IP on ASA 8.3+                                |
| HTTPS works but HTTP fails                       | `show running-config object service <service-object>`                                            | Service object permits only HTTPS                                                             |
| HTTP works but HTTPS fails                       | `packet-tracer input inside tcp <client-ip> 12345 <public-ip> 443 detailed`                      | Missing HTTPS service NAT, ACL permit, server listener, or host firewall rule                 |
| Ping fails but HTTPS works                       | `packet-tracer input inside icmp <client-ip> 8 0 <public-ip> detailed`                           | Hairpin NAT is service-specific for TCP, not ICMP                                             |
| Browser reaches homepage but login fails         | `show conn`, ASA capture, server logs                                                            | App redirect, alternate port, hostname mismatch, certificate issue, or server-side path issue |
| DNS name resolves to public IP internally        | Client DNS lookup                                                                                | Split DNS missing, DNS doctoring missing, or hairpin NAT required                             |
| DNS doctoring does not rewrite DNS reply         | `show running-config object network <server-real-object>`                                        | Missing `dns` keyword or DNS inspection path mismatch                                         |
| RPF or route drop appears                        | `show asp drop`                                                                                  | Asymmetric return path, missing source PAT, or routing mismatch                               |
| NAT looks correct but flow still fails           | `show conn`, `show xlate`, `show asp drop`                                                       | Stale xlate/conn, host firewall, server service down, or application-layer issue              |
| `show xlate` shows stale mapping                 | `show xlate`                                                                                     | Clear stale translations with `clear xlate` during lab correction                             |
| `show conn` shows old flow                       | `show conn address <inside-client-ip>`                                                           | Clear stale connection state before retesting                                                 |
