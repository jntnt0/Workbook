# ASAv_CLI_Access_And_Privilege_Escalation_Mental_Model

| Concept | Operational Meaning |
|---|---|
| User EXEC mode (`ciscoasa>`) | Default state after SSH login; read-only; most config commands are not recognized |
| Privileged EXEC mode (`ciscoasa#`) | Reached via `enable`; unlocks show commands, copy, reload, and entry into config mode |
| Global configuration mode (`ciscoasa(config)#`) | Reached via `configure terminal` from privileged EXEC; all config changes happen here |
| Sub-config mode (`ciscoasa(config-<object>)#`) | Entered when configuring a specific interface, object, or service; prompt reflects what is being changed |
| `enable` command | The gate between read-only and privileged access; requires the enable password |
| `configure terminal` | Only recognized after `enable`; attempting it from user EXEC returns an unrecognized command error |
| ASDM vs CLI config delta | ASDM applies access-list + access-group in one action; CLI requires both commands separately |
| running-config | Lives in RAM; active immediately; lost on reboot if not saved |
| startup-config | Saved to flash; loaded on boot; written via `write memory` |

# ASAv_CLI_Access_And_Privilege_Escalation_Configuration_Checklist

| Step | Task | Device | Command | Expected Result |
|---:|---|---|---|---|
| 1 | Open PowerShell or CMD | Win_Station | Start Menu or Run | Terminal window opens |
| 2 | SSH into the ASAv management interface | Win_Station | `ssh admin@mgmt.asav.internal` | Prompted with host key trust warning |
| 3 | Accept the host key | Win_Station | Type `yes` and press Enter | Connection proceeds to password prompt |
| 4 | Authenticate as admin | Win_Station | Enter `P@ssw0rd` | Prompt changes to `ciscoasa>` (user EXEC) |
| 5 | Confirm current mode | ASAv | Note the `>` prompt | Confirms user EXEC; config commands are not available here |
| 6 | Attempt configure terminal to observe failure | ASAv | `configure terminal` | Command not recognized; confirms privilege escalation is required |
| 7 | Query available commands | ASAv | `?` | Lists available user EXEC commands; `enable` is visible |
| 8 | Enter privileged EXEC mode | ASAv | `enable` | Prompted for enable password |
| 9 | Provide enable password | ASAv | Enter `P@ssw0rd` | Prompt changes to `ciscoasa#` (privileged EXEC) |
| 10 | Enter global configuration mode | ASAv | `configure terminal` | Prompt changes to `ciscoasa(config)#` |

# ASAv_CLI_Access_And_Privilege_Escalation_Skeleton

! From Win_Station PowerShell or CMD:
ssh admin@mgmt.asav.internal
! Accept host key when prompted: yes
! Password: P@ssw0rd

! Now at ciscoasa>
enable
! Enable password: P@ssw0rd

! Now at ciscoasa#
configure terminal

! Now at ciscoasa(config)#
! Ready to make configuration changes

# ASAv_CLI_Access_And_Privilege_Escalation_Verification_Commands

| Command | Where to Run | What It Confirms |
|---|---|---|
| `?` | `ciscoasa>` | Lists available user EXEC commands; `enable` should appear |
| `show version` | `ciscoasa#` | Confirms privileged mode is active and shows ASA software version |
| `show running-config` | `ciscoasa#` | Confirms access to full running configuration |
| `show privilege` | `ciscoasa#` | Shows current privilege level (should be 15 after `enable`) |

# ASAv_CLI_Access_And_Privilege_Escalation_Rollback

| Step | Task | Device | Command | Expected Result |
|---:|---|---|---|---|
| 1 | Exit global config mode | ASAv | `end` or `exit` | Prompt returns to `ciscoasa#` |
| 2 | Exit privileged EXEC mode | ASAv | `disable` | Prompt returns to `ciscoasa>` |
| 3 | Terminate SSH session | ASAv | `exit` | Session closes; returns to Win_Station terminal |

# ASAv_CLI_Access_And_Privilege_Escalation_Failure_Checks

| Symptom | Likely Cause | Fix |
|---|---|---|
| `configure` is not recognized at `ciscoasa>` | Still in user EXEC mode; `enable` not yet run | Run `enable` and authenticate first |
| SSH connection refused or times out | SSH not enabled on ASAv management interface, or wrong hostname | Verify `mgmt.asav.internal` resolves and SSH is configured on the ASAv |
| Enable password rejected | Wrong password or enable password not set | Use `P@ssw0rd`; if unset, requires console access to recover |
| Prompt shows `ciscoasa(config)#` but changes are lost after reboot | `write memory` was not run before reloading | Run `write memory` before any `reload` to persist running-config to startup-config |
| `show running-config` not available | Still in user EXEC mode (`>`) | Escalate with `enable` first |
