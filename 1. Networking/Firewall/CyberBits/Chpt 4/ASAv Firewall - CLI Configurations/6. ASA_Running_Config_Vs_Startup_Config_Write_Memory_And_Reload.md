# ASA_Running_Config_Vs_Startup_Config_Write_Memory_And_Reload_Mental_Model

| Concept | Operational Meaning |
|---|---|
| running-config | The active configuration loaded in RAM; all changes take effect here immediately; wiped on reboot |
| startup-config | The configuration saved to flash memory; loaded into RAM on every boot; persists across reboots |
| `write memory` | Copies running-config to startup-config; the only way to make changes survive a reboot via CLI |
| Unsaved changes | Any running-config entry that does not exist in startup-config; these are lost if the device reboots without `write memory` |
| `reload` prompt with no unsaved changes | After `write memory`, running-config and startup-config match; `reload` skips the save prompt and goes directly to `Proceed with reload? [confirm]` |
| `reload` prompt with unsaved changes | If `write memory` was not run, `reload` detects the mismatch and prompts `System config has been modified. Save? [Y]es/[N]o` |
| ASDM save process | Four steps: make changes, save in the modification screen, apply changes, initiate save config; all four must be completed to persist |
| CLI save process | Two steps: make changes, run `write memory`; no intermediate apply step required |
| `copy running-config startup-config` | Functionally identical to `write memory`; longer form of the same operation |

# ASA_Running_Config_Vs_Startup_Config_Write_Memory_And_Reload_Configuration_Checklist

| Step | Task | Device | Command | Expected Result |
|---:|---|---|---|---|
| 1 | Confirm you are in privileged EXEC mode | ASAv | Note prompt shows `ciscoasa#` | Ready to run write memory; config mode is not required |
| 2 | Save running-config to startup-config | ASAv | `write memory` | Output shows `Building configuration... [OK]`; configs are now in sync |
| 3 | Initiate a reload | ASAv | `reload` | No save prompt appears; ASA goes directly to `Proceed with reload? [confirm]` |
| 4 | Confirm the reload prompt | ASAv | Note the prompt text | `Proceed with reload? [confirm]` confirms no unsaved changes exist |
| 5 | Cancel or confirm the reload | ASAv | Press Enter to confirm or Ctrl+C to cancel | Reload proceeds or is cancelled depending on lab requirement |

# ASA_Running_Config_Vs_Startup_Config_Write_Memory_And_Reload_Skeleton

ciscoasa# write memory
! Output: Building configuration... [OK]
! running-config is now saved to startup-config

ciscoasa# reload
! Prompt: Proceed with reload? [confirm]
! No unsaved changes exist; save prompt is skipped entirely

! For comparison: if write memory had NOT been run first:
! Prompt: System config has been modified. Save? [Y]es/[N]o

# ASA_Running_Config_Vs_Startup_Config_Write_Memory_And_Reload_Verification_Commands

| Command | Where to Run | What It Confirms |
|---|---|---|
| `show running-config` | `ciscoasa#` | Displays the active config in RAM; what the device is currently running |
| `show startup-config` | `ciscoasa#` | Displays the config saved to flash; what will load on next boot |
| `write memory` | `ciscoasa#` | Output of `[OK]` confirms the save completed successfully |
| `show version` | `ciscoasa#` | Shows uptime and last reload reason; useful after a reboot to confirm it completed |

# ASA_Running_Config_Vs_Startup_Config_Write_Memory_And_Reload_Rollback

| Step | Task | Device | Command | Expected Result |
|---:|---|---|---|---|
| 1 | Revert startup-config to a prior state | ASAv | `copy startup-config running-config` | Loads saved config back into running-config; overwrites unsaved changes |
| 2 | Confirm running-config reflects the reverted state | ASAv | `show running-config` | Config matches what was last saved to startup-config |
| 3 | Save reverted state if it is the intended baseline | ASAv | `write memory` | Reverted config is now persisted to startup-config |

# ASA_Running_Config_Vs_Startup_Config_Write_Memory_And_Reload_Failure_Checks

| Symptom | Likely Cause | Fix |
|---|---|---|
| All changes lost after reboot | `write memory` was not run before `reload` | Always run `write memory` before any reload; verify with `show startup-config` that changes are present before rebooting |
| `reload` prompts to save instead of confirming directly | `write memory` was not run; running-config and startup-config are out of sync | Run `write memory` first; then run `reload` again |
| `write memory` returns an error | Insufficient flash space or file system issue | Check flash with `show flash`; clear old files if space is low |
| Config after reboot does not match expected state | `write memory` was run but a prior incomplete save exists in startup-config | Compare `show running-config` and `show startup-config` line by line to identify the delta |
| ASDM changes not persisted after reboot | ASDM save config step was skipped or only apply was run | All four ASDM steps must complete; apply alone does not write to flash |
| SSH session drops during reload | Expected behavior; the ASA reboots and terminates all active sessions | Reconnect via SSH after the ASA comes back up; allow 2 to 3 minutes for boot |
