# ASDM_Dynamic_PAT_Port_Translation_Mental_Model

| Concept | Operational Meaning |
|---|---|
| PAT (Port Address Translation) | Translates the source or destination port of a packet as it passes through the ASA |
| Dynamic PAT (Hide) | Hides the original source address and port behind a single translated address; many-to-one translation |
| Port translation | In this task the destination port is translated: client sends to port 8888, ASA rewrites it to port 80 before forwarding |
| Why port translation | IIS listens on port 80; the client browses to port 8888; the ASA bridges the gap without reconfiguring the server |
| Source interface | The interface where traffic originates; Workstations in this task |
| Source NAT type | Dynamic PAT (Hide) hides the originating address behind the Servers interface address |
| TCP_8888 object | Pre-existing service object representing TCP port 8888; used as the original service in the NAT rule |
| TCP_80 object | Pre-existing service object representing TCP port 80; used as the translated service in the NAT rule |
| "Always clear the translation" | Forces the ASA to immediately flush existing NAT translations when the rule is applied; ensures stale entries do not interfere |
| Time range dependency | The HTTP access rule had a Saturday-only time range; it must be removed before browsing will succeed on other days |
| Access rule still required | NAT alone does not permit traffic; the existing HTTP access rule on the Workstations interface must remain active |
| Expected result | Chrome browsing to http://IIS_Server:8888 loads the IIS page displaying "Cyberbit" |

# ASDM_Dynamic_PAT_Port_Translation_Configuration_Checklist

| Step | Task | Where | Action | Expected Result |
|---:|---|---|---|---|
| 1 | Delete existing NAT rule | Configuration → Firewall → NAT Rules | Select existing rule → Delete | NAT rule list is empty |
| 2 | Add new NAT rule | NAT Rules → Add | Click Add | New NAT rule dialog opens |
| 3 | Set source interface | Interface field | Select "Workstations" | Source interface set to Workstations |
| 4 | Set original service | Service field (original) | Select "TCP_8888" | Original service shows TCP 8888 |
| 5 | Set NAT type | Source NAT Type dropdown | Select "Dynamic PAT (Hide)" | NAT type set to Dynamic PAT |
| 6 | Set translated address | Source Address field | Select "Servers" interface | Translated address set to Servers interface |
| 7 | Set translated service | Service field (translated) | Select "TCP_80" | Translated service shows TCP 80 |
| 8 | Save NAT rule | Dialog | Click Ok | New PAT rule appears in NAT rule list |
| 9 | Apply with translation clear | Apply button | Click Apply → check "Always clear the translation…" → click Yes | Rule committed; existing translations flushed |
| 10 | Remove time range from HTTP rule | Configuration → Firewall → Access Rules | Double-click time range on HTTP rule → select blank entry → Ok → Apply | HTTP rule active on all days |
| 11 | Browse to port 8888 | Win_Station → Chrome | Navigate to http://IIS_Server:8888 | Page loads displaying "Cyberbit" |

# ASDM_Dynamic_PAT_Port_Translation_Skeleton

```
! Dynamic PAT rule with port translation (CLI equivalent):
nat (Workstations,Servers) source dynamic any interface service TCP_8888 TCP_80

! Pre-existing service objects referenced:
object service TCP_8888
 service tcp destination eq 8888

object service TCP_80
 service tcp destination eq 80
```

# ASDM_Dynamic_PAT_Port_Translation_Verification

| Check | Method | Expected Result |
|---|---|---|
| Old NAT rule removed | Configuration → Firewall → NAT Rules | Previous rule no longer listed |
| New PAT rule present | NAT Rules list | Rule shows Workstations interface, TCP_8888 → TCP_80, Dynamic PAT (Hide) |
| Time range removed from HTTP rule | Access Rules list | HTTP rule time range column is blank |
| Port 8888 browse succeeds | Chrome → http://IIS_Server:8888 | Page loads; "Cyberbit" text displayed |
| Wireshark confirmation (optional) | Capture on Workstations interface | Outbound packet shows destination port 8888; post-NAT packet shows destination port 80 |

# ASDM_Dynamic_PAT_Port_Translation_Rollback

| Step | Action |
|---|---|
| Delete PAT rule | Configuration → Firewall → NAT Rules → select rule → Delete → Apply |
| Restore time range on HTTP rule | Double-click HTTP rule → More Options → Time Range → select "Saturdays" → Ok → Apply |
| Restore original NAT rule | Re-create previous NAT rule with original settings → Apply |

# ASDM_Dynamic_PAT_Port_Translation_Failure_Checks

| Symptom | Likely Cause | Check | Fix |
|---|---|---|---|
| Browse to port 8888 times out | PAT rule not applied | Check NAT Rules list | Add and apply the PAT rule |
| Browse to port 8888 times out | Time range still active on HTTP access rule | Check Access Rules time range column | Remove time range from HTTP rule and apply |
| Browse to port 8888 times out | Stale NAT translation entry | Existing translation cached before rule change | Apply with "Always clear the translation…" checked |
| Browse to port 8888 times out | Wrong service objects in NAT rule | Verify original service is TCP_8888 and translated is TCP_80 | Correct service object assignments |
| Browse to standard port 80 fails | PAT rule intercepts all TCP from Workstations | Check NAT rule scope | Confirm rule only matches TCP_8888, not all TCP |
| Page loads but shows wrong content | Correct port but wrong server response | Confirm IIS is serving on port 80 | Verify IIS binding on IIS_Server |
