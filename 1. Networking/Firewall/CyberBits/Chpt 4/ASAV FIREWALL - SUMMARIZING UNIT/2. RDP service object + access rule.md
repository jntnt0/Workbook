# ASDM_RDP_Service_Object_And_Access_Rule_Mental_Model

| Concept | Operational Meaning |
|---|---|
| Service object | A named object on the ASA that defines a protocol and port; reusable across multiple rules |
| RDP | Remote Desktop Protocol; TCP port 3389; used to remotely access a Windows desktop |
| Service object vs inline service | A service object is defined once and referenced by name; inline services are typed directly into the rule and not reusable |
| TCP service object fields | Requires: protocol (tcp), destination port or range (3389) |
| Access rule dependency | The service object must exist before it can be selected in an access rule |
| Interface binding | The access rule must be bound to the Workstations interface (inbound) to permit traffic originating from that network |
| Four required rule components | Source, destination, service, action — all four must be present |
| MSTSC | The built-in Windows Remote Desktop client; used to initiate the RDP connection |
| RDP credentials in this lab | Username: admin / Password: P@ssw0rd / Host: IIS_Server |

# ASDM_RDP_Service_Object_And_Access_Rule_Configuration_Checklist

| Step | Task | Where | Action | Expected Result |
|---:|---|---|---|---|
| 1 | Open service objects | Configuration → Firewall → Objects → Service Objects/Groups | — | Service object list displayed |
| 2 | Add new service object | Top-left Add button | Click Add → Service Object | New service object dialog opens |
| 3 | Name the object | Name field | Enter "RDP" | Name field populated |
| 4 | Set protocol | Type dropdown | Select "tcp" | Protocol set to TCP |
| 5 | Set destination port | Destination Port/Range field | Enter "3389" | Port field shows 3389 |
| 6 | Save object | Dialog | Click Ok | "RDP" object appears in service object list |
| 7 | Open new access rule | Configuration → Firewall → Access Rules → Add | Click Add | New access rule dialog opens |
| 8 | Set source interface | Interface dropdown | Select "Workstations" | Interface set to Workstations |
| 9 | Set service | Service field | Remove "ip", add "RDP" object | Service field shows RDP (tcp/3389) |
| 10 | Save and apply | Dialog → Apply button | Click Ok, then Apply | Rule appears in access rule list |
| 11 | Launch RDP client | Win_Station desktop | Open MSTSC | Remote Desktop Connection dialog opens |
| 12 | Connect to IIS_Server | MSTSC host field | Enter "IIS_Server", supply credentials | RDP session established to IIS_Server |

# ASDM_RDP_Service_Object_And_Access_Rule_Skeleton

```
! Service object (CLI equivalent):
object service RDP
 service tcp destination eq 3389

! Access rule (CLI equivalent):
access-list Workstations_access_in extended permit tcp any any eq 3389
```

# ASDM_RDP_Service_Object_And_Access_Rule_Verification

| Check | Method | Expected Result |
|---|---|---|
| Service object exists | Configuration → Firewall → Objects → Service Objects/Groups | "RDP" listed with tcp/3389 |
| Access rule exists | Configuration → Firewall → Access Rules | Rule shows Workstations interface, RDP service, permit action |
| RDP connection succeeds | MSTSC → IIS_Server | Remote desktop session opens; IIS_Server desktop displayed |
| Expected desktop shortcuts on IIS_Server | Visual inspection after RDP login | Firefox, Wireshark, EC2 Microsoft Windows Guide |

# ASDM_RDP_Service_Object_And_Access_Rule_Rollback

| Step | Action |
|---|---|
| Close RDP session | Close the RDP window or log off from IIS_Server |
| Delete access rule | Select rule in Access Rules list → Delete → Apply |
| Delete service object | Configuration → Firewall → Objects → Service Objects/Groups → select "RDP" → Delete |

# ASDM_RDP_Service_Object_And_Access_Rule_Failure_Checks

| Symptom | Likely Cause | Check | Fix |
|---|---|---|---|
| RDP times out | Access rule not applied | Verify rule exists on Workstations interface | Add and apply the rule |
| RDP times out | Wrong service in rule | Inspect service column in rule | Confirm service is RDP (tcp/3389), not "ip" |
| RDP times out | Applied to wrong interface | Check interface column in rule | Edit rule, set interface to Workstations |
| Authentication failure | Wrong credentials | Verify username/password | Use admin / P@ssw0rd |
| Connection takes over a minute | Normal lab behavior | Wait up to one minute | No fix needed |
| Service object not available in rule dialog | Object not saved before opening rule | Check Service Objects list | Create and save the object first, then create the rule |
