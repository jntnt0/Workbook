# ASDM_ICMP_Ping_Rules_Workstations_And_Servers_Mental_Model

| Concept | Operational Meaning |
|---|---|
| ICMP is stateless | The ASA does not track ICMP exchanges the way it tracks TCP connections |
| TCP stateful behavior | A single inbound TCP permit rule covers the return traffic automatically via stateful inspection |
| ICMP stateless behavior | Return traffic (echo-reply) is a separate flow; it must be explicitly permitted with its own rule |
| Echo-request | The outbound ICMP packet sent by the pinging host; originates from Workstations |
| Echo-reply | The response ICMP packet sent back by the destination; originates from Servers |
| Two rules required | One rule permits ICMP from the Workstations interface; a second rule permits ICMP from the Servers interface |
| Rule directionality | Each rule is inbound on its respective interface; Workstations rule covers the request, Servers rule covers the reply |
| Implicit deny | Without the Servers-side rule, the echo-reply is dropped and ping fails even though the request was permitted |
| Ping verification | A successful ping confirms both rules are in place and traffic is flowing in both directions |

# ASDM_ICMP_Ping_Rules_Workstations_And_Servers_Configuration_Checklist

| Step | Task | Where | Action | Expected Result |
|---:|---|---|---|---|
| 1 | Open access rules | Configuration → Firewall → Access Rules | Click Add | New access rule dialog opens |
| 2 | Set source interface for rule 1 | Interface dropdown | Select "Workstations" | Interface set to Workstations |
| 3 | Set service for rule 1 | Service field | Remove "ip", add "icmp" | Service field shows icmp |
| 4 | Save rule 1 | Dialog | Click Ok | Rule 1 appears in access rule list on Workstations interface |
| 5 | Add second access rule | Access Rules → Add | Click Add | New access rule dialog opens |
| 6 | Set source interface for rule 2 | Interface dropdown | Select "Servers" | Interface set to Servers |
| 7 | Set service for rule 2 | Service field | Remove "ip", add "icmp" | Service field shows icmp |
| 8 | Save rule 2 | Dialog | Click Ok | Rule 2 appears in access rule list on Servers interface |
| 9 | Apply both rules | Bottom of ASDM | Click Apply | Both rules committed to running config |
| 10 | Verify ping | Win_Station → Command Prompt or PowerShell | ping IIS_Server | Ping replies received |

# ASDM_ICMP_Ping_Rules_Workstations_And_Servers_Skeleton

```
! Rule permitting ICMP from Workstations (echo-request outbound):
access-list Workstations_access_in extended permit icmp any any

! Rule permitting ICMP from Servers (echo-reply return):
access-list Servers_access_in extended permit icmp any any
```

# ASDM_ICMP_Ping_Rules_Workstations_And_Servers_Verification

| Check | Method | Expected Result |
|---|---|---|
| Rule 1 exists | Configuration → Firewall → Access Rules | ICMP permit rule on Workstations interface |
| Rule 2 exists | Configuration → Firewall → Access Rules | ICMP permit rule on Servers interface |
| Ping succeeds | Win_Station → ping IIS_Server | Replies received from IIS_Server |
| Ping fails with only one rule | Remove one rule and retest | Request or reply is dropped; ping times out |

# ASDM_ICMP_Ping_Rules_Workstations_And_Servers_Rollback

| Step | Action |
|---|---|
| Remove Workstations ICMP rule | Select rule on Workstations interface → Delete → Apply |
| Remove Servers ICMP rule | Select rule on Servers interface → Delete → Apply |

# ASDM_ICMP_Ping_Rules_Workstations_And_Servers_Failure_Checks

| Symptom | Likely Cause | Check | Fix |
|---|---|---|---|
| Ping times out completely | Both ICMP rules missing | Check Access Rules for icmp entries on both interfaces | Add both rules and apply |
| Ping sends but no reply | Only the Workstations rule exists; Servers rule missing | Check Servers interface in Access Rules | Add ICMP permit rule on Servers interface |
| Ping reply sends but request never arrives | Only the Servers rule exists; Workstations rule missing | Check Workstations interface in Access Rules | Add ICMP permit rule on Workstations interface |
| Rules exist but ping still fails | Rules applied to wrong interfaces | Verify interface column for each rule | Correct interface assignments and apply |
| Rules exist but ping still fails | Changes not applied | Check for pending changes indicator in ASDM | Click Apply |
