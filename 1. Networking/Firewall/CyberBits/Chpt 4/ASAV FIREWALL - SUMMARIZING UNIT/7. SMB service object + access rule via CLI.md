# ASA_CLI_SMB_Service_Object_And_Access_Rule_Mental_Model

| Concept | Operational Meaning |
|---|---|
| CLI-only administration | This task is performed entirely via SSH CLI; ASDM is not used |
| Privilege escalation path | User exec (>) → enable → privileged exec (#) → configure terminal → global config (config)# |
| Enable password | Required to escalate from > to #; in this lab: P@ssw0rd |
| Service object via CLI | Defined with object service, followed by the service tcp destination eq command |
| Object scope | The object service block must be exited before writing the access-list entry |
| SMB | Server Message Block protocol; TCP port 445; used for Windows file and printer sharing |
| access-list extended permit | The CLI command to add a permit entry to a named ACL on the ASA |
| ACL name convention | Workstations_access_in matches the inbound ACL already bound to the Workstations interface |
| access-group binding | The ACL must already be bound to the interface via access-group for new entries to take effect immediately |
| write memory | Saves running config to startup config; required to persist changes across reboot |
| UNC path | \\IIS_Server\c$ accesses the C: drive share on IIS_Server; requires admin credentials |
| Expected C:\ root folders | temp, EFI, Cyberbit are present at the root of C:\ on IIS_Server in this lab |

# ASA_CLI_SMB_Service_Object_And_Access_Rule_Configuration_Checklist

| Step | Task | Where | Action | Expected Result |
|---:|---|---|---|---|
| 1 | Connect via SSH | Win_Station → SSH client | Connect to mgmt.asav.internal as admin | ASA user exec prompt (>) displayed |
| 2 | Enter enable mode | CLI | Type enable, supply P@ssw0rd | Privileged exec prompt (#) displayed |
| 3 | Enter global config | CLI | Type configure terminal | Global config prompt (config)# displayed |
| 4 | Create SMB service object | CLI | Type object service SMB | Object service config submode entered |
| 5 | Define service | CLI | Type service tcp destination eq 445 | TCP 445 assigned to SMB object |
| 6 | Exit object submode | CLI | Type exit | Returns to (config)# prompt |
| 7 | Create access-list entry | CLI | Type access-list Workstations_access_in extended permit tcp any any eq 445 | SMB permit entry added to Workstations ACL |
| 8 | Save configuration | CLI | Type write memory | Running config saved to startup config |
| 9 | Open file explorer | Win_Station desktop | Open File Explorer or Run dialog | File Explorer or Run dialog open |
| 10 | Navigate to IIS_Server share | File Explorer address bar or Run | Enter \\IIS_Server\c$ | Credential prompt appears |
| 11 | Authenticate | Credential dialog | Enter admin / P@ssw0rd | C:\ root contents displayed |
| 12 | Note desktop folders | C:\ root | Inspect folder list | temp, EFI, and Cyberbit folders present |

# ASA_CLI_SMB_Service_Object_And_Access_Rule_Skeleton

```
! Privilege escalation:
enable
! (supply P@ssw0rd when prompted)
configure terminal

! Create SMB service object:
object service SMB
 service tcp destination eq 445
exit

! Create access-list entry for SMB on Workstations interface:
access-list Workstations_access_in extended permit tcp any any eq 445

! Save to startup config:
write memory
```

# ASA_CLI_SMB_Service_Object_And_Access_Rule_Failure_Checks

| Symptom | Likely Cause | Check | Fix |
|---|---|---|---|
| Cannot escalate to enable | Wrong enable password | Confirm password at prompt | Use P@ssw0rd as enable password |
| Object service command rejected | Not in global config mode | Check prompt; must show (config)# | Run configure terminal first |
| ACL entry rejected | Not in global config mode | Check prompt | Run configure terminal first |
| Share access times out | ACL entry not added or not saved | show running-config access-list | Re-enter access-list command and write memory |
| Share access times out | ACL not bound to interface | show running-config access-group | Confirm Workstations_access_in is bound inbound on Workstations |
| Authentication failure on share | Wrong credentials | Confirm username and password | Use admin / P@ssw0rd |
| Changes lost after reboot | write memory not run | show startup-config | Run write memory after every CLI change |
| Service object created but not referenced | Object exists but ACL entry uses inline port instead | show running-config access-list | Both approaches work; object is reusable but inline eq 445 is functionally equivalent |
