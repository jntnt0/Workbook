ASAv_User_Account_Password_And_Access_Restrictions_Mental_Model

| Concept | Operational Meaning |
|---|---|
| User account password | The password tied to a specific named user account; used for ASDM and SSH authentication; separate from the enable password |
| Enable password | Controls CLI privileged mode access; not affected by changes to user account passwords |
| User Accounts panel | Located at Configuration > Device Management > Users/AAA > User Accounts; lists all local user accounts on the ASAv |
| Change user password checkbox | Must be checked before the password fields become active in the user edit dialog; unchecked means no password change is applied |
| Access Restriction | Controls which management interfaces a user account is permitted to use; set per user in the account properties |
| Full access | User can authenticate via ASDM, SSH, Telnet, and Console; no restrictions on management interface type |
| CLI login prompt | User can authenticate via SSH, Telnet, and Console only; ASDM access is blocked for this account |
| No access | User account exists but cannot authenticate via any management interface; effectively a disabled account |
| ASDM-only access | Not an available option on Cisco ASA; if ASDM access is granted, CLI access via SSH/Telnet/Console is also permitted |
| Principle of least privilege | Best practice to assign users the minimum access level required for their role |

ASAv_User_Account_Password_And_Access_Restrictions_Configuration_Checklist

| Step | Task | Device | Action | Expected Result |
|---:|---|---|---|---|
| 1 | Open ASDM | Win_Station | Launch ASDM from desktop shortcut; connect to `mgmt.asav.internal` as `admin` / `P@ssw0rd` | ASDM loads and displays the device dashboard |
| 2 | Navigate to User Accounts | ASDM | Configuration > Device Management > Users/AAA > User Accounts | User Accounts panel opens; existing accounts are listed |
| 3 | Open the admin user for editing | ASDM | Double-click the `admin` user entry | User account edit dialog opens |
| 4 | Enable the password change fields | ASDM | Check the box next to "Change user password" | Password fields become active |
| 5 | Enter the new user password | ASDM | Type the new password in the Password field | Field is populated |
| 6 | Confirm the new user password | ASDM | Type the same password in the Confirm Password field | Both fields match |
| 7 | Examine access restriction options | ASDM | Scroll to the Access Restriction section below the password fields | Three options are visible: Full access / CLI login prompt / No access |
| 8 | Note available options | ASDM | Review all options in the Access Restriction dropdown or radio buttons | Full access; CLI login prompt for SSH, Telnet and Console; No ASDM/SSH/Telnet/Console access |
| 9 | Select the appropriate access level | ASDM | Choose the access restriction appropriate for the account role | Access restriction is set |
| 10 | Apply the change | ASDM | Click OK, then click Apply | User account password and access restriction are updated on the ASAv |
| 11 | Verify login with new credentials | Win_Station | Close and reopen ASDM; log in with `admin` and the new password | Login succeeds with updated credentials |

    ASDM: Configuration > Device Management > Users/AAA > User Accounts
    Double-click: admin
    Check: "Change user password"
    Password:         <new_password>
    Confirm Password: <new_password>
    Access Restriction: <select appropriate level>
    Click OK > Apply

    CLI equivalent:
    username admin password <new_password>
    username admin privilege <level>

ASAv_User_Account_Password_And_Access_Restrictions_Verification_Commands

    ASDM: Configuration > Device Management > Users/AAA > User Accounts
    Confirm admin account reflects updated access restriction setting

    Login verification:
    Close ASDM
    Reopen ASDM; connect using admin and new password
    Successful login confirms password change applied correctly

    CLI verification (from SSH session):
    ssh admin@mgmt.asav.internal
    Enter new password at prompt
    Successful login confirms SSH access matches assigned access restriction

ASAv_User_Account_Password_And_Access_Restrictions_Rollback

| Step | Task | Device | Action | Expected Result |
|---:|---|---|---|---|
| 1 | Open ASDM | Win_Station | Launch ASDM; connect to `mgmt.asav.internal` with current credentials | ASDM loads |
| 2 | Navigate to User Accounts | ASDM | Configuration > Device Management > Users/AAA > User Accounts | User Accounts panel opens |
| 3 | Open the admin user for editing | ASDM | Double-click the `admin` user | Edit dialog opens |
| 4 | Enable the password change fields | ASDM | Check "Change user password" | Password fields become active |
| 5 | Restore the original password | ASDM | Enter `P@ssw0rd` in both password fields | Fields are populated with original password |
| 6 | Restore the original access restriction | ASDM | Set Access Restriction back to original setting | Access level reverts |
| 7 | Apply | ASDM | Click OK, then Apply | Account reverts to original password and access restriction |
| 8 | Verify rollback | Win_Station | Close and reopen ASDM; log in with `admin` / `P@ssw0rd` | Login succeeds with original credentials |

ASAv_User_Account_Password_And_Access_Restrictions_Failure_Checks

| Symptom | Likely Cause | Device | Command | Fix |
|---|---|---|---|---|
| Password fields remain greyed out | "Change user password" checkbox not checked | ASDM | User account edit dialog | Check the checkbox before entering the new password |
| ASDM login fails after password change | Wrong password entered or change not applied | Win_Station | ASDM login prompt | Verify Apply was clicked; re-enter credentials; if locked out use console access to reset |
| SSH login fails despite Full access setting | SSH not enabled on the management interface | Win_Station | `ssh admin@mgmt.asav.internal` | Confirm SSH is permitted via Startup Wizard or ACL on the management interface |
| User can SSH but cannot access ASDM | Access restriction set to CLI login prompt instead of Full access | ASDM | User Accounts panel | Change access restriction to Full access and apply |
| No access option blocks all logins | Access restriction set to No ASDM/SSH/Telnet/Console | Win_Station | Any login attempt | Use a separate admin account with Full access to correct the restriction |
| Password change lost after reload | Change applied to running config only | ASAv CLI | `show running-config` vs `show startup-config` | Run `write memory` to persist to flash |
| ASDM-only access option not found | Option does not exist on Cisco ASA | ASDM | User account edit dialog | Only three options are available: Full access, CLI login prompt, No access |
