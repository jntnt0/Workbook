ASAv_SSH_Management_Access_Startup_Wizard_Mental_Model

| Concept | Operational Meaning |
|---|---|
| SSH management access | SSH is not permitted by default on Cisco ASA; it must be explicitly enabled per interface before connections are accepted |
| Startup Wizard | ASDM tool at Configuration > Device Setup > Startup Wizard; steps through most core device settings; useful for both initial setup and modifying existing configuration |
| Management access step | Step 8 of the Startup Wizard; defines which protocols are permitted on which interfaces and from which source IP ranges |
| Access Type | The protocol being permitted for management; options include SSH, Telnet, and HTTPS |
| Interface Name | The ASA interface on which the management protocol is being enabled; in this lab the management interface is used |
| IP Address 0.0.0.0 / Subnet Mask 0.0.0.0 | Permits the specified protocol from any source IP address; appropriate for lab environments only |
| Source IP restriction | In production, the IP address and subnet mask should be scoped to the specific management network or host to limit exposure |
| SSH connection refused | Returned when SSH is not permitted on the interface; the device rejects the connection before authentication |
| SSH authentication failure | Returned when SSH is permitted but credentials are wrong; differs from connection refused |
| Startup Wizard vs direct CLI | The wizard modifies the same underlying configuration as CLI; it is a guided interface over standard ASA commands |

ASAv_SSH_Management_Access_Startup_Wizard_Configuration_Checklist

| Step | Task | Device | Action | Expected Result |
|---:|---|---|---|---|
| 1 | Attempt initial SSH connection | Win_Station | Run `ssh admin@mgmt.asav.internal` in CMD/PowerShell | Connection fails; SSH is not permitted on the management interface |
| 2 | Open ASDM | Win_Station | Launch ASDM from desktop shortcut; connect to `mgmt.asav.internal` as `admin` / `P@ssw0rd` | ASDM loads and displays the device dashboard |
| 3 | Navigate to Startup Wizard | ASDM | Configuration > Device Setup > Startup Wizard | Startup Wizard landing page opens |
| 4 | Launch the wizard | ASDM | Click "Launch Startup Wizard" | Wizard opens at Step 1 |
| 5 | Advance to Step 8 | ASDM | Click Next repeatedly until Step 8 (Management Access) is reached | Step 8 management access panel is displayed |
| 6 | Add a new management access entry | ASDM | Click Add | New management access entry dialog opens |
| 7 | Set the access type | ASDM | Set Access Type to `SSH` | SSH is selected as the permitted protocol |
| 8 | Set the interface | ASDM | Set Interface Name to `management` | Management interface is targeted |
| 9 | Set the source IP | ASDM | Set IP Address to `0.0.0.0` | Any source IP will be permitted |
| 10 | Set the subnet mask | ASDM | Set Subnet Mask to `0.0.0.0` | Mask confirms any source IP |
| 11 | Confirm the entry | ASDM | Click Finish | Entry is added to the management access list |
| 12 | Complete the wizard | ASDM | Click Next through remaining steps, then click Finish | Wizard applies the configuration to the ASAv |
| 13 | Retest SSH connection | Win_Station | Run `ssh admin@mgmt.asav.internal` in CMD/PowerShell | SSH connection succeeds; login prompt is presented |
| 14 | Authenticate | Win_Station | Enter `admin` and the current user password at the SSH prompt | SSH session established successfully |

ASAv_SSH_Management_Access_Startup_Wizard_Verification_Commands

    From Win_Station CMD/PowerShell:
    ssh admin@mgmt.asav.internal
    Successful prompt confirms SSH is permitted and credentials are valid

    ASDM verification:
    Configuration > Device Setup > Startup Wizard > Launch
    Navigate to Step 8
    Confirm SSH entry exists for management interface with 0.0.0.0 / 0.0.0.0

    CLI verification (from within SSH session):
    show running-config ssh
    Entry should show: ssh 0.0.0.0 0.0.0.0 management

ASAv_SSH_Management_Access_Startup_Wizard_Failure_Checks

| Symptom | Likely Cause | Device | Command | Fix |
|---|---|---|---|---|
| SSH connection refused after wizard completion | Wizard changes not applied; Finish not clicked on final step | Win_Station | `ssh admin@mgmt.asav.internal` | Relaunch wizard, re-add SSH entry, ensure Finish is clicked on the final step |
| SSH connects but authentication fails | Wrong password entered; user password was changed in a prior step | Win_Station | `ssh admin@mgmt.asav.internal` | Use the current user account password set in the User Accounts panel |
| SSH entry added to wrong interface | Interface Name set to a data interface instead of management | ASDM | Step 8 of Startup Wizard | Delete incorrect entry; re-add with Interface Name set to `management` |
| SSH permitted but only from specific IPs | A prior SSH rule exists with a restricted source IP | ASAv CLI | `show running-config ssh` | Add or replace with `ssh 0.0.0.0 0.0.0.0 management` for unrestricted lab access |
| Startup Wizard Step 8 shows no existing entries | SSH was never previously configured | ASDM | Step 8 panel | Normal state for a new device; add the SSH entry as described |
| SSH configuration lost after reload | Change applied to running config only | ASAv CLI | `show running-config ssh` | Run `write memory` to persist to flash |
