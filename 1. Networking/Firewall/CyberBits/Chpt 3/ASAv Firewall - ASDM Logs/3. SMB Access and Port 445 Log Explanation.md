# ASDM_SMB_Access_Port_445_Log_Explanation_Mental_Model
| Concept | Operational Meaning |
|---|---|
| SMB (Server Message Block) | Windows file sharing protocol; uses TCP port 445 for transport |
| UNC path | Universal Naming Convention path format; `\\hostname\share` is used to access remote Windows file shares |
| `\\IIS_Server\c$` | Administrative share of the C: drive on IIS_Server; requires admin credentials to access |
| TCP port 445 | The destination port the ASA sees when the workstation initiates an SMB connection to the server |
| %ASA-6-302013 | Syslog message generated by the ASA when a TCP connection slot is successfully created between two hosts |
| Severity 6 for 302013 | Successful connections are informational events; the ASA does not treat permitted traffic as a warning |
| Stateful TCP tracking | The ASA tracks each TCP connection in its state table; 302013 fires on establishment, 302014 fires on teardown |
| Hover for explanation | Hovering over a log entry in the ASDM syslog panel displays a tooltip with the human-readable description of that syslog message ID |

# ASDM_SMB_Access_Port_445_Log_Explanation_Configuration_Checklist
| Step | Task | Expected Result |
|---|---|---|
| 1 | Ensure ASDM logging is enabled and the Home screen syslog panel is visible | Syslog panel is active and populated |
| 2 | Open File Explorer on the Windows workstation | File Explorer window opens |
| 3 | Click the address bar and type `\\IIS_Server\c$` then press Enter | Windows credential prompt appears |
| 4 | Enter credentials: Username `admin`, Password `P@ssw0rd` | File Explorer opens the C: drive of IIS_Server |
| 5 | Return to ASDM Home screen and examine the top log entries | New log entries appear for the SMB connection |
| 6 | Identify log entries where the destination port is 445 | One or more 302013 entries are visible with destination port 445 |
| 7 | Hover over a port 445 log entry | Tooltip displays: "A TCP connection slot between two hosts was created." |

# ASDM_SMB_Access_Port_445_Log_Explanation_SMB_Access_Skeleton
Windows File Explorer UNC Access:
  Address bar -> \\IIS_Server\c$
  Credential prompt:
    Username: admin
    Password: P@ssw0rd

SMB Transport:
  Protocol:         TCP
  Destination port: 445
  Source:           Workstations network (Windows workstation)
  Destination:      Servers network (IIS_Server)

Expected ASA behavior:
  ASA permits the TCP connection (if access rules allow)
  ASA creates a connection slot in the state table
  ASA generates syslog %ASA-6-302013

# ASDM_SMB_Access_Port_445_Log_Explanation_Verification_Skeleton
ASDM GUI Path:
  Home screen
  -> Latest ASDM Syslog Messages panel
  -> Locate log entries with destination port 445
  -> Hover over the entry

Expected syslog message:
  ID:          302013
  Severity:    6 (Informational)
  Description: "A TCP connection slot between two hosts was created."

Expected log format example:
  %ASA-6-302013: Built inbound TCP connection <id>
  for Workstations:<workstation-ip>/<src-port>
  to Servers:<iis-server-ip>/445

CLI verification:
  show conn detail
  show logging | include 302013

# ASDM_SMB_Access_Port_445_Log_Explanation_Verification_Commands
| Task | Method | Expected Result |
|---|---|---|
| Confirm SMB connection reached ASA | ASDM syslog panel -> filter for port 445 entries | 302013 entries visible with dst port 445 |
| Confirm log description | Hover over a port 445 log entry | Tooltip reads: "A TCP connection slot between two hosts was created." |
| Confirm syslog ID | Read the Syslog ID column of the port 445 entry | 302013 |
| Confirm severity | Read the first column of the port 445 entry | 6 (Informational) |
| Confirm active connection via CLI | SSH to ASAv -> `show conn detail` | SMB connection appears in the ASA connection table |
| Confirm log via CLI | `show logging \| include 302013` | 302013 entries appear referencing port 445 |

# ASDM_SMB_Access_Port_445_Log_Explanation_Rollback
| Step | Task | Method | Expected Result |
|---|---|---|---|
| 1 | Close the SMB File Explorer window on the workstation | Close the `\\IIS_Server\c$` File Explorer window | SMB session is terminated from the client side |
| 2 | Confirm connection teardown in ASDM | Check syslog panel for %ASA-6-302014 | Teardown entry appears for the port 445 connection confirming the session closed |
| 3 | Confirm connection removed from state table | SSH to ASAv -> `show conn detail` | No active connection entry for port 445 between workstation and IIS_Server |
| 4 | Clear ASDM log panel if required | Right-click panel -> "Clear Content" | Panel is empty |

# ASDM_SMB_Access_Port_445_Log_Explanation_Failure_Checks
| Symptom | What to Check | What Usually Broke |
|---|---|---|
| Credential prompt does not appear | Confirm `\\IIS_Server\c$` was typed correctly in the address bar | UNC path typo or IIS_Server hostname not resolving |
| Access denied after entering credentials | Confirm username is `admin` and password is `P@ssw0rd` | Wrong credentials entered |
| No port 445 entries appear in ASDM syslog | Confirm ASDM logging is enabled and filter severity covers level 6 | Logging disabled or filter set too restrictive to show informational events |
| Log entry shows severity 4 instead of 6 for port 445 | Connection was denied rather than permitted | Access rule is blocking SMB; check ASA access rules for Workstations to Servers on port 445 |
| Hover tooltip does not appear | Hover cursor directly over the log entry text, not the column border | Cursor not positioned over the log row correctly |
| 302013 entry shows wrong destination port | A different connection was logged | Identify the correct row by filtering for dst port 445 specifically |
| File Explorer opens but ASDM shows no new logs | ASDM panel was not refreshed | Scroll up in the syslog panel or wait for the panel to auto-refresh |
