# ASDM_Enable_Logging_And_First_Event_Mental_Model
| Concept | Operational Meaning |
|---|---|
| ASDM logging disabled by default | No syslog messages appear in the Home screen panel until logging is explicitly enabled |
| `logging asdm informational` | The command issued by ASDM when the "Enable ASDM Logging" button is clicked; sets the minimum severity to 6 (Informational) |
| First event logged | The act of enabling logging is itself the first event captured; the ASA records the command execution via syslog ID 111009 |
| Syslog ID 111009 | Severity 7 (Debugging); logs any command executed by a user that does not modify the running configuration |
| Severity scale direction | Higher severity = lower number; Emergency is 0, Debugging is 7 |
| Latest ASDM Syslog Messages panel | Located at the bottom of the ASDM Home screen; displays real-time syslog events filtered by the active logging configuration |
| Command logging vs. config change logging | 111009 fires for read-only or non-modifying commands; a true config change generates a different message ID |

# ASDM_Enable_Logging_And_First_Event_Configuration_Checklist
| Step | Task | Expected Result |
|---|---|---|
| 1 | Open a browser and navigate to `https://mgmt.asav.internal` | ASDM Java launcher or web page loads |
| 2 | Log in with credentials: `admin` / `P@ssw0rd` | ASDM Home screen opens |
| 3 | Scroll to the bottom of the Home screen | "Latest ASDM Syslog Messages" panel is visible with an "Enable ASDM Logging" button |
| 4 | Click "Enable ASDM Logging" | Button disappears; syslog panel begins populating |
| 5 | Examine the first event in the syslog panel | First entry reads: "User 'admin' executed the 'logging asdm informational' command." |

# ASDM_Enable_Logging_And_First_Event_Enable_Logging_Skeleton
ASDM GUI Path:
  Home screen (default after login)
  -> Scroll to bottom
  -> Latest ASDM Syslog Messages panel
  -> Click: "Enable ASDM Logging"

Equivalent CLI command issued by ASDM:
  logging asdm informational

Credentials:
  URL:      https://mgmt.asav.internal
  Username: admin
  Password: P@ssw0rd

# ASDM_Enable_Logging_And_First_Event_Verification_Skeleton
ASDM GUI Path:
  Home screen
  -> Latest ASDM Syslog Messages panel (bottom of screen)
  -> Read the first (bottom-most chronological) entry

Expected first event description:
  "User 'admin' executed the 'logging asdm informational' command."

Expected Syslog ID: 111009
Expected Severity:  7 (Debugging)

Note: Default filter is Informational (severity 1-6); 111009 is severity 7
      and will only appear once the filter is explicitly set to Debugging.
      At default filter level, the first visible event may differ.

CLI verification:
  show logging asdm
  show running-config | include logging asdm

# ASDM_Enable_Logging_And_First_Event_Rollback
| Step | Task | Method | Expected Result |
|---|---|---|---|
| 1 | Identify current logging state | ASDM Home screen or `show logging asdm` | Current state is known |
| 2 | Disable ASDM logging via CLI | `no logging asdm` | ASDM syslog panel stops receiving new messages |
| 3 | Clear existing log panel content | Right-click panel -> "Clear Content" | Panel is empty |
| 4 | Verify logging disabled | `show running-config \| include logging asdm` | No `logging asdm` line present in config |
| 5 | Save config if required | `write memory` | Running config saved to startup config |

# ASDM_Enable_Logging_And_First_Event_Failure_Checks
| Symptom | What to Check | What Usually Broke |
|---|---|---|
| "Enable ASDM Logging" button does not appear | Confirm you are on the Home screen, not a sub-page | Wrong ASDM page is open; navigate back to Home |
| Syslog panel is empty after clicking Enable | Refresh Home screen or wait briefly for events | No traffic has generated logs yet; trigger activity or check logging filter level |
| First event is not the 111009 command log | Check the severity filter setting | Default filter (Informational, severity 1-6) hides severity 7 messages; 111009 only appears when filter is set to Debugging |
| ASDM does not load | Verify browser Java plugin or ASDM launcher is installed | Java not installed or ASDM web launch method mismatch |
| Login fails | Confirm credentials are `admin` / `P@ssw0rd` on `mgmt.asav.internal` | Wrong hostname, wrong credentials, or management interface is unreachable |
| Syslog panel shows "Disable ASDM Logging" instead of "Enable" | Logging was already active from a prior session | Normal; panel should already be populated |
