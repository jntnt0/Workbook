# Lab 8 — Determining the Scope of Infection (Host Count)

## Tab 1

Scoping an infection is one of the most critical steps in any threat hunt. A single compromised host is a contained problem. Multiple hosts mean the attacker has moved laterally and the response effort scales accordingly — every infected machine needs to be isolated, imaged, and remediated. Knowing the true count before containment actions begin prevents incomplete response and reinfection.

The goal of this task is to determine how many hosts were infected by the `GoogleUpdate.exe` malware by examining the `extracted_host` field across all matching log entries.

---

### Step-by-Step Thought Process

**Search for the malicious executable broadly.**  
Unlike previous queries that used the full Image path tied to User067's machine, this search needs to cast a wider net. Searching for `googleupdate.exe` without hard-coding the User067 path allows the query to surface any host on the network where this executable was present in logs.

**Locate the extracted_host field.**  
The `extracted_host` field identifies which machine generated each log entry. By examining the distinct values in this field across all GoogleUpdate.exe results, you can determine exactly how many hosts reported activity tied to this malicious binary.

**Filter by each host to confirm the executable path.**  
Not every result may represent a genuine infection — a log entry could reference the filename in a different context. Filtering by each unique `extracted_host` value and examining the file path of the executable on that host confirms whether it is the malicious instance or a coincidental reference.

**Count confirmed instances.**  
After filtering and path verification, two distinct hosts appear with the malicious `GoogleUpdate.exe` present. The initial infected machine (User067's workstation) and one additional host that the malware propagated to during the attack.

---

### Expected Splunk Query

```
source=/home/splunk/data.csv googleupdate.exe
```
Then locate and inspect the `extracted_host` field in the left panel under Interesting Fields.

---

### Final Query (Clean)

```
source=/home/splunk/data.csv googleupdate.exe
| stats count by extracted_host
```

To verify path per host:
```
source=/home/splunk/data.csv googleupdate.exe
| table extracted_host Image CommandLine
```

---

### Findings

**Host Infection Summary:**

| Host | Executable Path | Status |
|---|---|---|
| Host 1 (User067's workstation) | `C:\Users\User067\Downloads\GoogleUpdate.exe` | Initial infection — payload delivered via phishing and Chrome download |
| Host 2 | `GoogleUpdate.exe` present | Secondary infection — malware propagated from file server share |

**Answer:** 2 hosts were infected. The attacker established the initial foothold on User067's workstation, then the malware propagated to a second host via the file server — confirmed by the two distinct values in the `extracted_host` field across GoogleUpdate.exe log entries.
