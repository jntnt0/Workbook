# Lab 10 — Recognizing Classical Threat-Hunt Patterns

## Tab 1

Threat hunting is the practice of proactively searching through network and endpoint telemetry to detect adversaries that have already bypassed initial defenses. It is not reactive — it does not wait for an alert to fire. A skilled threat hunter assumes compromise is possible and hunts for indicators that security tools missed. The attack investigated across this lab series is a textbook example of why that assumption is correct and why proactive hunting is necessary.

The goal of this task is to evaluate the full attack chain uncovered across Labs 1 through 9 against the defining characteristics of a classical threat hunt scenario, and confirm which patterns are present.

---

### Step-by-Step Thought Process

**Evaluate: Silent attack with user unaware.**  
From the moment the phishing email arrived to the moment the malware copied itself to the file server, the user on User067's workstation had no indication anything was wrong. The payload was disguised as `GoogleUpdate.exe` — a name most users would not question. All reconnaissance commands ran in the background via CMD.exe and PowerShell.exe. No error dialogs, no unusual application behavior, no visible output. This is a textbook silent attack.

**Evaluate: Exfiltration and silent lateral movement.**  
The attacker exfiltrated domain users, system information, and firewall rules — all through native Windows tools that blend into normal administrative telemetry. Lateral movement was achieved by writing to a shared file server, again using a built-in file copy operation. No external scanning tools, no noisy exploit code, no anomalous traffic volume. Every action was designed to stay below the threshold of automated detection.

**Evaluate: Unclear attack flow requiring log analysis.**  
At no point during the attack did a standard security tool fire an alert. There was no AV detection of `GoogleUpdate.exe`. No firewall block on the C&C connection to `199.203.100.79`. No IDS signature match on the `net group "Domain Admins" /domain` query. The attack flow only became visible through systematic Splunk log analysis — correlating ParentImage, CommandLine, DestinationIp, and extracted_host fields across multiple event types. Without that analysis, the compromise would have remained invisible indefinitely.

**Why all three patterns apply simultaneously.**  
These three characteristics are not independent observations — they are interconnected. The attack is unclear precisely because it is silent. It is silent because the attacker used legitimate tools and disguised the payload. The lateral movement went undetected because it mimicked normal file server activity. Every element of this attack was engineered to defeat reactive, signature-based defenses — which is exactly the threat landscape that makes proactive threat hunting necessary.
