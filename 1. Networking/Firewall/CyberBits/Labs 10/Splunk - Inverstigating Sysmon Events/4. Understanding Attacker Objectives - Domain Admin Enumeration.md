# Lab 4 — Understanding Attacker Objectives: Domain Admin Enumeration

## Tab 1

After establishing a foothold and profiling the environment, an attacker's next logical move is privilege escalation. Before they can escalate, they need to know *who* to target. The Domain Admins group holds the highest privilege level in an Active Directory environment — full control over every machine, user, and policy in the domain. Enumerating its membership tells the attacker exactly which accounts are worth stealing or impersonating.

The goal of this task is to understand why the attacker executed `net group "Domain Admins" /domain` and what they intended to do with the results.

---

### Step-by-Step Thought Process

**Understand what the command does.**  
`net group "Domain Admins" /domain` queries Active Directory for the membership list of the Domain Admins group. It returns every username that holds domain administrator rights across the entire network. This is a native Windows command — no external tooling required — which makes it low-noise and unlikely to trigger basic AV.

**Understand why the attacker ran it.**  
The attacker already has a list of domain users from `net users /domain`. That gives them the full population. Domain Admins enumeration narrows that list to the highest-value targets — the accounts whose credentials, if obtained, grant unrestricted access to every host, share, and service in the domain.

**Understand the intended next step.**  
Knowing who the Domain Admins are sets up the next phase of the attack — credential theft, pass-the-hash, or targeted phishing against those specific accounts. With domain admin credentials, the attacker can execute commands on any machine in the domain without restriction.

**Why not the other options.**  
- *Get domain computer list* — that is a separate command (`net group "Domain Computers" /domain` or `Get-ADComputer`)  
- *Get system privileges on the local machine* — local privilege escalation uses different tooling (token impersonation, exploit code, etc.), not domain group enumeration  
- *None of the above* — ruled out; the command's function is well documented

### Findings

| Command | Purpose |
|---|---|
| `cmd.exe /c net group "Domain Admins" /domain` | Retrieves the full membership list of the Domain Admins group from Active Directory |

**Attack Intent Breakdown:**

| Stage | Action |
|---|---|
| Reconnaissance | `net users /domain` — full domain user list |
| Targeting | `net group "Domain Admins" /domain` — narrows to highest-value accounts |
| Intended Next Step | Credential theft or lateral movement targeting Domain Admin accounts |

**Answer:** Retrieve the list of users in the Domain Admins group. This could possibly lead to executing commands using admin rights. The attacker is building a target list of the most privileged accounts in the domain — the accounts that, if compromised, hand over complete control of the network.
