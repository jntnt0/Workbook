# ICMP_Host_Discovery_Payload_Size_Analysis_Mental_Model

| Concept | Operational Meaning |
|---|---|
| ICMP | Internet Control Message Protocol. A Layer 3 protocol used for diagnostics, error reporting, and host discovery |
| Host discovery scan | An attack technique that sends ICMP packets to identify live hosts on a network without completing full connections |
| ICMP payload | The data field carried inside an ICMP packet. Standard ping tools include a data payload by default |
| Payload size of 0 | No data field exists in the packet. Malicious ICMP host discovery packets carry no payload, distinguishing them from legitimate ping traffic |
| dsize | A Snort non-payload option that matches packets based on their payload byte size |
| dsize:0 | Matches packets that carry zero bytes of payload data. Effective for detecting empty ICMP probes |
| Sequence number 0 | Malicious ICMP packets in this capture carry a consistent sequence number of 0, another static detection parameter |
| Legitimate vs malicious ICMP | Legitimate ICMP packets carry a data field and consistent packet IDs. Malicious packets have no data and sequence number 0 |
| Static parameter | A value that does not change between packets, making it a reliable field for Snort rule detection |

# ICMP_Host_Discovery_Payload_Size_Analysis_ICMP_Field_Reference

| ICMP Field | Location in Wireshark | Relevance to This Task |
|---|---|---|
| Type | Internet Control Message Protocol > Type | Identifies the ICMP message type (e.g. Echo Request = 8) |
| Code | Internet Control Message Protocol > Code | Sub-type of the ICMP message |
| Sequence number | Internet Control Message Protocol > Sequence Number | Static at 0 in malicious packets, varies in legitimate traffic |
| Data | Internet Control Message Protocol > Data | Present in legitimate packets, absent in malicious packets |
| Data length | Internet Control Message Protocol > Data length | Reports payload size. Reads 0 in malicious packets |

# ICMP_Host_Discovery_Payload_Size_Analysis_Key_Observations

| Parameter | Malicious Packets #1-#64 | Legitimate Packets #65-#116 | Static / Dynamic | Snort Rule Field |
|---|---|---|---|---|
| Payload size | 0 bytes | Non-zero | Static | dsize:0; |
| Data field present | No | Yes | Static | Basis for dsize:0 detection |
| Sequence number | 0 | Varies | Static in malicious | Can supplement dsize:0 to reduce false positives |
| Packet ID | Varies | Consistent per session | Dynamic in malicious | Not reliable alone for detection |
| Protocol | ICMP | ICMP | Static | Rule header protocol field |

# ICMP_Host_Discovery_Payload_Size_Analysis_Snort_Rule_Skeleton

    alert icmp $EXTERNAL_NET any -> $HOME_NET any
    (msg:"ICMP Host Discovery Scan Detected"; dsize:0;
    sid:1000003; rev:1;)

    Note: dsize:0 matches packets carrying zero bytes of payload data.
    This distinguishes malicious ICMP probes from legitimate ping traffic
    which carries a data payload. Source IP and destination IP will be
    added to the rule header once all static parameters are confirmed.

# ICMP_Host_Discovery_Payload_Size_Analysis_Verification

| Check | What to Look For | Confirms |
|---|---|---|
| Data field in ICMP section | Absent in packets #1-#64 | Payload size is 0 bytes in malicious packets |
| Data field in legitimate packets | Present in packets #65-#116 | Payload size distinguishes malicious from legitimate ICMP |
| Sequence number in malicious range | Consistently 0 across #1-#64 | Sequence number is a secondary static detection parameter |
| Protocol column in Wireshark | All packets show ICMP | Confirms correct protocol field for Snort rule header |
| dsize:0 scope | Only matches packets with zero payload bytes | Legitimate ping traffic with data payload will not trigger the rule |

# ICMP_Host_Discovery_Payload_Size_Analysis_Failure_Checks

| Symptom | Likely Cause | Fix |
|---|---|---|
| Data field appears in malicious packet range | Packet selected is outside #1-#64 | Confirm packet number is within the malicious range |
| Cannot find Data field at all | ICMP section not fully expanded | Expand all rows under Internet Control Message Protocol |
| Data field present but empty | Wireshark displaying field with no content | Check the Data length value. If it reads 0 the payload is confirmed empty |
| Legitimate and malicious packets look identical | Comparing wrong packet ranges | Malicious range is #1-#64. Legitimate range is #65-#116. Do not mix them |
| Confused dsize with packet size | dsize refers to payload only, not total packet length | dsize:0 targets the ICMP data field specifically, not the full packet byte count |
| Sequence number varies in malicious range | Selected packets span both ranges | Re-confirm all selected packets fall within #1-#64 only |
