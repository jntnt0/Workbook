# OSPF_Path_Selection_Intra_Inter_External_Mental_Model
| Concept | Operational Meaning |
|---|---|
| OSPF decision order | OSPF chooses route class before metric: intra-area first, interarea second, external Type 1 third, external Type 2 fourth |
| Intra-area route | Route built from Type 1 and Type 2 LSAs inside the same area. It appears in the RIB as `O` |
| Interarea route | Route built from Type 3 summary LSAs generated by an ABR. It appears in the RIB as `O IA` |
| External Type 1 route | Redistributed route where total metric equals external redistribution metric plus internal cost to reach the ASBR. It appears as `O E1` or `O N1` |
| External Type 2 route | Redistributed route where the external metric stays fixed across the OSPF domain. It appears as `O E2` or `O N2` |
| Default external type | OSPF redistribution defaults to external Type 2 unless configured otherwise |
| Metric tie | If route class and metric tie, OSPF can install multiple equal-cost paths |
| Intra beats lower metric interarea | A worse metric intra-area route can beat a better metric interarea route because route class wins before metric |
| Interarea beats external | An interarea route beats an external route even if the external route has a lower displayed metric |
| E1 beats E2 | External Type 1 is preferred over External Type 2 for the same prefix |
| E2 tie-break | If two E2 routes have the same external metric, the route with lower forwarding cost to the ASBR wins |
| Forward metric | Internal OSPF cost to reach the ASBR or forwarding address for an external route |
| NSSA external route | NSSA routes appear as `O N1` or `O N2` inside or near NSSA logic. On IOS XE, NSSA external can be preferred over matching Type 5 external in typical NSSA behavior |
| Interface cost | `ip ospf cost <1-65535>` manually controls OSPF link cost for path selection |
| Reference bandwidth | `auto-cost reference-bandwidth <mbps>` changes automatic interface cost calculation and should match across routers |
| ECMP | Equal-cost multipathing installs multiple equal OSPF paths when route type and metric tie |
| RIB versus LSDB | The LSDB may contain a prefix even when the route is not installed in the global RIB due to filtering, administrative distance, better route source, or OSPF selection logic |
| Administrative distance | OSPF’s default AD is 110. A non-OSPF route source with a better AD can beat OSPF in the global routing table |
# OSPF_Path_Selection_Intra_Inter_External_Configuration_Checklist
| Step | Task | Device | Command | Expected Result |
|---:|---|---|---|---|
| 1 | Identify the prefix being evaluated | Design step | `<target-prefix>/<length>` | Exact route-selection target is known |
| 2 | Identify all OSPF sources for the same prefix | Design step | `O`, `O IA`, `O E1`, `O E2`, `O N1`, `O N2` candidates | Competing route classes are known before changing cost |
| 3 | Confirm OSPF adjacencies are healthy | All involved routers | `show ip ospf neighbor` | Required neighbors are `FULL` |
| 4 | Confirm OSPF interfaces and areas | All involved routers | `show ip ospf interface brief` | Interfaces are in the expected areas |
| 5 | Confirm current route installed in the RIB | Decision router | `show ip route <target-prefix>` | Current route type, metric, next hop, and outgoing interface are visible |
| 6 | Confirm all OSPF routes in the RIB | Decision router | `show ip route ospf` | OSPF route types and installed paths are visible |
| 7 | Confirm whether the prefix exists in the LSDB | Decision router | `show ip ospf database` | LSDB contains or lacks the prefix source as expected |
| 8 | Check intra-area LSA source for local area routes | Decision router | `show ip ospf database router` | Type 1 LSAs are visible for intra-area topology |
| 9 | Check broadcast network LSA source if relevant | Decision router | `show ip ospf database network` | Type 2 LSAs are visible for DR-based broadcast segments |
| 10 | Check interarea LSA source for `O IA` routes | Decision router | `show ip ospf database summary <target-prefix>` | Type 3 summary LSA is visible if route is interarea |
| 11 | Check ASBR summary source for external routes from other areas | Decision router | `show ip ospf database asbr-summary` | Type 4 ASBR summary LSA is visible when ASBR is in another area |
| 12 | Check external LSA source for `O E1` or `O E2` routes | Decision router | `show ip ospf database external <target-prefix>` | Type 5 LSA shows advertising router, metric type, metric, and forwarding address |
| 13 | Check NSSA external LSA source for `O N1` or `O N2` routes | Decision router | `show ip ospf database nssa-external <target-prefix>` | Type 7 LSA is visible when route originates from NSSA |
| 14 | Confirm reference bandwidth before changing cost | All OSPF routers | `show ip ospf | include Reference bandwidth` | Reference bandwidth is known and consistent or inconsistency is identified |
| 15 | Standardize reference bandwidth if the lab requires modern link-cost behavior | All OSPF routers | `router ospf <process-id>` then `auto-cost reference-bandwidth <mbps>` | Automatic OSPF cost calculation uses the same reference value on all routers |
| 16 | Manually set link cost to influence intra-area or interarea path selection | Selected router interfaces | `interface <interface-id>` then `ip ospf cost <1-65535>` | OSPF metric changes on the selected interface |
| 17 | Verify changed interface cost | Selected routers | `show ip ospf interface <interface-id> | include Cost` | Interface shows the intended OSPF cost |
| 18 | Configure external route as Type 1 when internal cost to ASBR must matter | ASBR | `router ospf <process-id>` then `redistribute <source-protocol> subnets metric-type 1 metric <metric>` | Redistributed routes appear as `O E1` or `O N1` where appropriate |
| 19 | Configure external route as Type 2 when fixed external metric should dominate | ASBR | `router ospf <process-id>` then `redistribute <source-protocol> subnets metric-type 2 metric <metric>` | Redistributed routes appear as `O E2` or `O N2` where appropriate |
| 20 | Set deterministic external metric during redistribution | ASBR | `redistribute <source-protocol> subnets metric <metric> metric-type <1-or-2>` | External LSA has expected metric and metric type |
| 21 | Use route-map redistribution if only selected external prefixes need controlled metric | ASBR | `redistribute <source-protocol> subnets route-map <route-map-name>` | Per-prefix external metric or metric type policy is applied |
| 22 | Configure ECMP path count if equal OSPF paths should install | OSPF routers | `router ospf <process-id>` then `maximum-paths <maximum-paths>` | Router can install up to the configured number of equal-cost OSPF paths |
| 23 | Verify OSPF process settings | All involved routers | `show running-config | section router ospf` | Reference bandwidth, redistribution metric type, and maximum paths are present as intended |
| 24 | Clear or wait for SPF recalculation after lab changes | Lab only | `clear ip ospf process` | OSPF recalculates after adjacency reset. Use only in lab or maintenance window |
| 25 | Recheck selected route after cost or metric change | Decision router | `show ip route <target-prefix>` | Installed route reflects expected OSPF class, metric, and next hop |
| 26 | Verify intra-area preference over interarea | Decision router | `show ip route <target-prefix>` | `O` route wins over `O IA` for same prefix |
| 27 | Verify interarea preference over external | Decision router | `show ip route <target-prefix>` | `O IA` route wins over `O E1` or `O E2` for same prefix |
| 28 | Verify E1 preference over E2 | Decision router | `show ip route <target-prefix>` | `O E1` route wins over `O E2` for same prefix |
| 29 | Verify E2 forwarding metric tie-break | Decision router | `show ip route <target-prefix>` | If E2 external metric ties, lower forward metric path wins |
| 30 | Verify ECMP if expected | Decision router | `show ip route <target-prefix>` | Multiple equal next hops appear for the same OSPF route |
| 31 | Verify forwarding path | Decision router | `traceroute <target-destination>` | Traffic follows expected next hop and path |
| 32 | Verify reachability | Decision router | `ping <target-destination>` | Destination responds through selected OSPF path |
# OSPF_Path_Selection_Intra_Inter_External_Skeleton
! =========================================================
! Standardize OSPF reference bandwidth
! Configure the same value on every OSPF router.
! =========================================================
configure terminal
router ospf <PROCESS_ID>
 auto-cost reference-bandwidth <REFERENCE_BANDWIDTH_MBPS>
end
write memory
! =========================================================
! Manually influence OSPF path selection with interface cost
! Lower cost is preferred inside the same route class.
! =========================================================
configure terminal
interface <INTERFACE_ID>
 ip ospf cost <1-65535>
end
write memory
! =========================================================
! Restore automatic OSPF interface cost
! =========================================================
configure terminal
interface <INTERFACE_ID>
 no ip ospf cost
end
write memory
! =========================================================
! External Type 1 redistribution
! Metric = external metric plus internal cost to ASBR.
! =========================================================
configure terminal
router ospf <PROCESS_ID>
 redistribute <SOURCE_PROTOCOL> subnets metric-type 1 metric <METRIC>
end
write memory
! =========================================================
! External Type 2 redistribution
! Metric = external metric.
! Forward metric to ASBR is used as tie-break when external metric ties.
! =========================================================
configure terminal
router ospf <PROCESS_ID>
 redistribute <SOURCE_PROTOCOL> subnets metric-type 2 metric <METRIC>
end
write memory
! =========================================================
! Redistribution with route-map based metric policy
! =========================================================
configure terminal
ip prefix-list <PREFIX_LIST_NAME> seq 5 permit <PREFIX>/<LENGTH>
route-map <ROUTE_MAP_NAME> permit 10
 match ip address prefix-list <PREFIX_LIST_NAME>
 set metric <METRIC>
 set metric-type type-1
exit
route-map <ROUTE_MAP_NAME> permit 100
exit
router ospf <PROCESS_ID>
 redistribute <SOURCE_PROTOCOL> subnets route-map <ROUTE_MAP_NAME>
end
write memory
! =========================================================
! Equal-cost multipathing
! =========================================================
configure terminal
router ospf <PROCESS_ID>
 maximum-paths <MAXIMUM_PATHS>
end
write memory

# OSPF_Path_Selection_Intra_Inter_External_Failure_Checks
| Symptom | Likely Cause | Check | Corrective Action |
|---|---|---|---|
| Lower metric route does not win | Route class has lower priority | `show ip route <target-prefix>` | Check whether the winning route is `O` over `O IA`, or `O IA` over external |
| Interarea route loses to intra-area route | Normal OSPF route-class behavior | `show ip route <target-prefix>` | No fix needed unless area design is wrong |
| External route loses to interarea route | Normal OSPF route-class behavior | `show ip route <target-prefix>` | No fix needed unless redistribution or area design is wrong |
| E2 route loses to E1 route | E1 is preferred over E2 | `show ip route <target-prefix>` | Change redistribution metric type only if design requires E2 behavior |
| E2 route takes unexpected ASBR | Same external metric, lower forwarding metric wins | `show ip route <target-prefix>`, `show ip ospf database external <target-prefix>` | Adjust internal cost to ASBR or set different external metrics |
| OSPF route is in LSDB but not RIB | Better administrative distance route exists | `show ip route <target-prefix>`, `show ip ospf database` | Check static, EIGRP, BGP, connected, or other route source |
| OSPF route is in LSDB but not RIB | OSPF distribute-list filters local RIB installation | `show ip protocols`, `show running-config | section router ospf` | Remove or correct `distribute-list prefix <name> in` |
| Prefix missing from LSDB entirely | LSA not advertised, filtered, summarized, or blocked by area type | `show ip ospf database`, `show ip ospf`, `show running-config | section router ospf` | Fix advertisement, area filtering, summarization, stub, or NSSA behavior |
| Expected `O IA` route missing | Type 3 LSA is blocked or ABR is not generating it | `show ip ospf database summary <target-prefix>` | Check ABR area placement, Type 3 filtering, summarization, and Area 0 continuity |
| Expected external route missing | Type 5 LSA not originated or blocked by area type | `show ip ospf database external <target-prefix>`, `show ip ospf` | Check ASBR redistribution and confirm area is not stub or totally stub |
| Expected NSSA route missing | Type 7 LSA not originated or not translated | `show ip ospf database nssa-external <target-prefix>` | Check NSSA area type, redistribution, and ABR translation behavior |
| Manual cost does not affect selected path | Competing route is in a different route class | `show ip route <target-prefix>` | Remember cost only decides after route class is selected |
| Manual cost does not appear | Cost configured on wrong interface | `show ip ospf interface <interface-id> | include Cost` | Apply `ip ospf cost <value>` on the outgoing interface that participates in SPF |
| Automatic costs look wrong on fast links | Reference bandwidth is too low | `show ip ospf | include Reference bandwidth` | Configure consistent `auto-cost reference-bandwidth <mbps>` across routers |
| Path selection differs between routers | Reference bandwidth mismatch or different topology view | `show ip ospf`, `show ip ospf database`, `show ip ospf interface brief` | Standardize reference bandwidth and fix LSDB or adjacency inconsistency |
| ECMP does not install | Metrics are not exactly equal | `show ip route <target-prefix>` | Match interface costs, external metrics, and forwarding costs |
| ECMP does not install despite equal metrics | `maximum-paths` too low | `show running-config | section router ospf` | Increase `maximum-paths <value>` under OSPF |
| Route appears as E2 when E1 was expected | Redistribution defaults to Type 2 | `show running-config | section router ospf`, `show ip route <target-prefix>` | Configure `metric-type 1` during redistribution |
| Route appears as E1 when E2 was expected | Redistribution explicitly configured Type 1 | `show running-config | section router ospf` | Configure `metric-type 2` or remove Type 1 policy |
| Forward metric is unexpected | Internal cost to ASBR is unexpected | `show ip route <asbr-router-id>`, `show ip ospf database external <target-prefix>` | Adjust OSPF interface costs toward the ASBR |
| Traffic path differs from route expectation | CEF, recursive next hop, or different route on another router | `show ip route <target-prefix>`, `traceroute <target-destination>` | Verify each hop’s RIB and forwarding path |
| Ping fails but route selection is correct | Return path, ACL, or data-plane issue | `show ip route <source-prefix>` on return-side routers | Verify reverse route and filtering |
