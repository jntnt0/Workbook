# OSPF_ABR_Type_3_LSA_Filtering_Mental_Model
| Concept | Operational Meaning |
|---|---|
| Type 3 LSA | Summary LSA generated by an ABR to advertise prefixes from one area into another area |
| ABR-only feature | Type 3 LSA filtering is configured on the ABR because the ABR is the router generating or regenerating the Type 3 summary LSA |
| Prefix-list required | IOS Type 3 LSA filtering uses an IP prefix list referenced by `area <area-id> filter-list prefix <name> <direction>` |
| Prefix-list logic | `deny` means the matched prefix is filtered; `permit` means the matched prefix is allowed through |
| Permit rest | Always add a broad permit line such as `permit 0.0.0.0/0 le 32` unless the intent is to block everything else |
| In direction | `area <area-id> filter-list prefix <name> in` blocks matching Type 3 LSAs from entering that area |
| Out direction | `area <area-id> filter-list prefix <name> out` blocks matching Type 3 LSAs from being advertised out of that area |
| Not intra-area filtering | This does not filter Type 1 or Type 2 LSAs inside the same area |
| Not external filtering | This does not filter Type 5 external LSAs or Type 7 NSSA LSAs |
| Not local RIB-only filtering | Unlike `distribute-list in`, Type 3 filtering changes what summary LSAs are advertised into the target area LSDB |
| ABR may still know the route | The ABR can still have the route in its own RIB or source-area LSDB even while suppressing the Type 3 LSA into another area |
| Downstream effect | Routers in the filtered area should lose both the `O IA` route and the matching Type 3 summary LSA |
| Backbone impact | Filtering a prefix out of Area 0 can affect all downstream nonbackbone areas that depend on Area 0 for interarea reachability |
| Area design warning | Use Type 3 filtering carefully because it can create intentional black holes if return paths or alternate summaries are not planned |
# OSPF_ABR_Type_3_LSA_Filtering_Configuration_Checklist
| Step | Task | Device | Command | Expected Result |
|---:|---|---|---|---|
| 1 | Identify the prefix that must be filtered | Design step | `<filtered-prefix>/<prefix-length>` | Exact interarea prefix is known before policy is applied |
| 2 | Identify the ABR responsible for the area boundary | Design step | `ABR: <router-name>` | Filtering will be applied on the ABR, not on an internal area router |
| 3 | Identify the source area and destination area | Design step | `Source area: <area-id>, Destination area: <area-id>` | Direction can be chosen correctly |
| 4 | Decide whether to filter into the target area | Design step | `area <target-area> filter-list prefix <name> in` | Matching Type 3 LSAs are blocked from entering the target area |
| 5 | Decide whether to filter out of the source area | Design step | `area <source-area> filter-list prefix <name> out` | Matching Type 3 LSAs are blocked from leaving the source area |
| 6 | Confirm the router is actually an ABR | ABR | `show ip ospf` | Router shows multiple areas and ABR behavior |
| 7 | Confirm interface area placement | ABR | `show ip ospf interface brief` | ABR has at least one interface in Area 0 and at least one interface in another area |
| 8 | Confirm OSPF adjacencies are healthy before filtering | ABR and area routers | `show ip ospf neighbor` | Required neighbors are `FULL` |
| 9 | Confirm the target prefix currently exists as a Type 3 LSA before filtering | ABR and downstream routers | `show ip ospf database summary <prefix>` | Target prefix appears as a Summary Net Link State where expected |
| 10 | Confirm the target prefix currently appears as an interarea route before filtering | Downstream routers | `show ip route <prefix>` | Route appears as `O IA` before filtering |
| 11 | Enter global configuration mode | ABR | `configure terminal` | Router enters global configuration mode |
| 12 | Create prefix-list deny line for the exact Type 3 prefix | ABR | `ip prefix-list <PREFIX_LIST_NAME> seq 5 deny <prefix>/<length>` | Target prefix is matched for suppression |
| 13 | Add permit line for all remaining prefixes | ABR | `ip prefix-list <PREFIX_LIST_NAME> seq 100 permit 0.0.0.0/0 le 32` | Other Type 3 prefixes remain allowed |
| 14 | Enter the OSPF process | ABR | `router ospf <process-id>` | OSPF router configuration mode opens |
| 15 | Apply inbound Type 3 filtering into the destination area if required | ABR | `area <destination-area-id> filter-list prefix <PREFIX_LIST_NAME> in` | Matching Type 3 LSAs are blocked from entering the destination area |
| 16 | Apply outbound Type 3 filtering out of the source area if required | ABR | `area <source-area-id> filter-list prefix <PREFIX_LIST_NAME> out` | Matching Type 3 LSAs are blocked from being advertised out of the source area |
| 17 | Exit configuration mode | ABR | `end` | Router returns to privileged EXEC mode |
| 18 | Save the configuration | ABR | `write memory` | Configuration is saved |
| 19 | Verify prefix-list contents | ABR | `show ip prefix-list <PREFIX_LIST_NAME>` | Deny line for target prefix and permit-rest line are present |
| 20 | Verify OSPF filter-list configuration | ABR | `show running-config | section router ospf` | `area <area-id> filter-list prefix <name> in` or `out` is present |
| 21 | Verify target Type 3 LSA is removed from filtered area | Downstream routers in filtered area | `show ip ospf database summary <prefix>` | Matching summary LSA is absent |
| 22 | Verify target interarea route is removed from filtered area RIB | Downstream routers in filtered area | `show ip route <prefix>` | `% network not in table` or no matching `O IA` route |
| 23 | Verify permitted Type 3 LSAs still exist | Downstream routers in filtered area | `show ip ospf database summary <allowed-prefix>` | Allowed summary LSA remains present |
| 24 | Verify permitted interarea routes still install | Downstream routers in filtered area | `show ip route <allowed-prefix>` | Allowed route still appears as `O IA` |
| 25 | Verify ABR still knows the source prefix where expected | ABR | `show ip route <prefix>` and `show ip ospf database summary <prefix>` | ABR may still know the route in the source area or Area 0 |
| 26 | Verify reachability to filtered prefix fails only where intended | Downstream routers in filtered area | `ping <filtered-prefix-test-ip>` | Ping fails from filtered area if no alternate route exists |
| 27 | Verify reachability to allowed prefixes still works | Downstream routers in filtered area | `ping <allowed-prefix-test-ip>` | Ping succeeds for prefixes not matched by the deny statement |
# OSPF_ABR_Type_3_LSA_Filtering_Skeleton
! =========================================================
! Inbound Type 3 LSA filtering
! Use this when the goal is:
! "Do not let this interarea prefix enter Area <DESTINATION_AREA>."
! =========================================================
configure terminal
ip prefix-list <PREFIX_LIST_NAME> seq 5 deny <FILTERED_PREFIX>/<PREFIX_LENGTH>
ip prefix-list <PREFIX_LIST_NAME> seq 100 permit 0.0.0.0/0 le 32
router ospf <PROCESS_ID>
 area <DESTINATION_AREA_ID> filter-list prefix <PREFIX_LIST_NAME> in
end
write memory
! =========================================================
! Outbound Type 3 LSA filtering
! Use this when the goal is:
! "Do not let this area advertise this prefix out toward other areas."
! =========================================================
configure terminal
ip prefix-list <PREFIX_LIST_NAME> seq 5 deny <FILTERED_PREFIX>/<PREFIX_LENGTH>
ip prefix-list <PREFIX_LIST_NAME> seq 100 permit 0.0.0.0/0 le 32
router ospf <PROCESS_ID>
 area <SOURCE_AREA_ID> filter-list prefix <PREFIX_LIST_NAME> out
end
write memory
! =========================================================
! Example pattern
! Block 11.1.1.0/24 from entering Area 2.
! =========================================================
configure terminal
ip prefix-list T3_FILTER seq 5 deny 11.1.1.0/24
ip prefix-list T3_FILTER seq 100 permit 0.0.0.0/0 le 32
router ospf 1
 area 2 filter-list prefix T3_FILTER in
end
write memory
! =========================================================
! Example pattern
! Block 11.1.1.0/24 from being advertised out of Area 1.
! =========================================================
configure terminal
ip prefix-list T3_FILTER seq 5 deny 11.1.1.0/24
ip prefix-list T3_FILTER seq 100 permit 0.0.0.0/0 le 32
router ospf 1
 area 1 filter-list prefix T3_FILTER out
end
write memory

# OSPF_ABR_Type_3_LSA_Filtering_Rollback
! =========================================================
! Remove inbound Type 3 filter-list from destination area
! =========================================================
configure terminal
router ospf <PROCESS_ID>
 no area <DESTINATION_AREA_ID> filter-list prefix <PREFIX_LIST_NAME> in
end
write memory
! =========================================================
! Remove outbound Type 3 filter-list from source area
! =========================================================
configure terminal
router ospf <PROCESS_ID>
 no area <SOURCE_AREA_ID> filter-list prefix <PREFIX_LIST_NAME> out
end
write memory
! =========================================================
! Remove the prefix-list after detaching it from OSPF
! =========================================================
configure terminal
no ip prefix-list <PREFIX_LIST_NAME>
end
write memory
! =========================================================
! Remove only one sequence from the prefix-list
! =========================================================
configure terminal
no ip prefix-list <PREFIX_LIST_NAME> seq <SEQ_NUMBER>
end
write memory
! =========================================================
! Force OSPF refresh only in a lab or maintenance window
! This resets OSPF adjacencies.
! =========================================================
clear ip ospf process
# OSPF_ABR_Type_3_LSA_Filtering_Failure_Checks
| Symptom | Likely Cause | Check | Corrective Action |
|---|---|---|---|
| Filter has no effect | Filter was configured on a non-ABR | `show ip ospf`, `show ip ospf interface brief` | Move the filter to the ABR between the relevant areas |
| Prefix still appears in filtered area LSDB | Wrong area ID was referenced | `show running-config | section router ospf`, `show ip ospf database summary <prefix>` | Apply the filter-list to the correct source or destination area |
| Prefix still appears in filtered area LSDB | Wrong direction was used | `show running-config | section router ospf` | Use `in` to block entry into an area or `out` to block advertisement out of an area |
| All interarea routes disappeared | Prefix list lacks permit-rest statement | `show ip prefix-list <PREFIX_LIST_NAME>` | Add `ip prefix-list <name> seq 100 permit 0.0.0.0/0 le 32` |
| Filtered router still has the route | You are checking the ABR or source area, not the filtered downstream area | `show ip route <prefix>`, `show ip ospf database summary <prefix>` | Verify from a router inside the destination area |
| Route is gone from RIB but still exists in LSDB | A `distribute-list in` or local RIB filter was used instead of Type 3 filtering | `show running-config | section router ospf`, `show ip ospf database summary <prefix>` | Replace local RIB filtering with `area <area-id> filter-list prefix <name> in/out` on the ABR |
| Prefix is intra-area and does not filter | Type 3 filtering cannot suppress Type 1 or Type 2 LSAs inside the same area | `show ip route <prefix>`, `show ip ospf database router`, `show ip ospf database network` | Use proper area design, summarization, or move the boundary where Type 3 LSAs are generated |
| External route is not filtered | Prefix is Type 5 or Type 7, not Type 3 | `show ip route <prefix>`, `show ip ospf database external`, `show ip ospf database nssa-external` | Use external route filtering, NSSA controls, redistribution policy, or area type controls |
| Prefix-list matches wrong prefix length | Prefix-list line is too broad or too exact | `show ip prefix-list <PREFIX_LIST_NAME>` | Correct the prefix and `le` or `ge` operators |
| Other areas unexpectedly lost the route | Outbound filtering toward Area 0 affected downstream areas | `show ip ospf database summary <prefix>` across areas | Re-scope the filter with `in` on only the intended destination area |
| No Type 3 LSAs are present at all | Area is totally stubby or totally NSSA | `show ip ospf` | Confirm area type before troubleshooting prefix-list behavior |
| ABR is not generating summaries | ABR has no valid Area 0 attachment | `show ip ospf`, `show ip ospf neighbor` | Restore Area 0 adjacency and correct ABR area placement |
| Ping fails for allowed prefixes | Return path or separate filtering issue | `show ip route <allowed-prefix>`, `traceroute <allowed-prefix>` | Verify reverse route and filters on other ABRs |
| Output looks stale after rollback | OSPF has not refreshed yet or lab process is stuck | `show ip ospf database summary <prefix>` | Wait for refresh, flap the relevant interface, or use `clear ip ospf process` only in a lab window |
