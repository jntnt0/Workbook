# LISP_Local_EID_Space_Mapping_Checklist

| Step | Task                                                                             | Device               | Command                                                                                       | Expected Result                                                              |                                                                     |
| ---: | -------------------------------------------------------------------------------- | -------------------- | --------------------------------------------------------------------------------------------- | ---------------------------------------------------------------------------- | ------------------------------------------------------------------- |
|    1 | Identify the local EID subnet                                                    | xTR / ETR            | `show ip route connected`                                                                     | Local endpoint subnet appears as a connected route                           |                                                                     |
|    2 | Verify the EID-facing interface is up                                            | xTR / ETR            | `show ip interface brief`                                                                     | Interface facing local endpoints shows `up/up`                               |                                                                     |
|    3 | Verify the EID prefix is exact                                                   | xTR / ETR            | `show ip route <eid-prefix>`                                                                  | Local EID prefix exists in the routing table                                 |                                                                     |
|    4 | Identify the local RLOC address                                                  | xTR / ETR            | `show ip interface brief`                                                                     | RLOC-facing interface has the underlay locator IP                            |                                                                     |
|    5 | Verify RLOC-side underlay reachability                                           | xTR / ETR            | `ping <map-server-rloc-ip>`                                                                   | xTR can reach the map-server/map-resolver                                    |                                                                     |
|    6 | Verify map-server can reach the xTR RLOC                                         | MSMR                 | `ping <xtr-rloc-ip>`                                                                          | Map-server can reach the xTR locator                                         |                                                                     |
|    7 | Enter LISP configuration mode on the map-server                                  | MSMR                 | `conf t` then `router lisp`                                                                   | Device enters LISP configuration mode                                        |                                                                     |
|    8 | Create or enter the LISP site                                                    | MSMR                 | `site <site-name>`                                                                            | Site configuration mode is entered                                           |                                                                     |
|    9 | Configure the site authentication key                                            | MSMR                 | `authentication-key 0 <key>`                                                                  | Site key is configured                                                       |                                                                     |
|   10 | Allow the local EID prefix for the site                                          | MSMR                 | `eid-prefix <eid-prefix>/<length>`                                                            | Map-server allows this EID space to register                                 |                                                                     |
|   11 | Allow more-specific EIDs if required                                             | MSMR                 | `eid-prefix <summary-eid-prefix>/<length> accept-more-specifics`                              | Map-server accepts more-specific registrations inside the summary            |                                                                     |
|   12 | Exit the site configuration                                                      | MSMR                 | `exit`                                                                                        | Device returns to LISP router configuration mode                             |                                                                     |
|   13 | Enable map-server function                                                       | MSMR                 | `ipv4 map-server`                                                                             | MSMR can accept EID registrations                                            |                                                                     |
|   14 | Enable map-resolver function                                                     | MSMR                 | `ipv4 map-resolver`                                                                           | MSMR can resolve EID-to-RLOC lookups                                         |                                                                     |
|   15 | Save map-server configuration                                                    | MSMR                 | `end` then `write memory`                                                                     | MSMR configuration is saved                                                  |                                                                     |
|   16 | Enter LISP configuration mode on the xTR                                         | xTR / ETR            | `conf t` then `router lisp`                                                                   | xTR enters LISP configuration mode                                           |                                                                     |
|   17 | Map the local EID prefix to the local RLOC                                       | xTR / ETR            | `database-mapping <eid-prefix>/<length> <xtr-rloc-ip> priority <priority> weight <weight>`    | Local EID space is mapped to the local RLOC                                  |                                                                     |
|   18 | Add a second RLOC if the site is multihomed                                      | xTR / ETR            | `database-mapping <eid-prefix>/<length> <second-rloc-ip> priority <priority> weight <weight>` | Same EID prefix has an additional locator                                    |                                                                     |
|   19 | Enable ETR function for registration                                             | xTR / ETR            | `ipv4 etr`                                                                                    | Router can register local EID-to-RLOC mappings                               |                                                                     |
|   20 | Point the ETR to the map-server                                                  | xTR / ETR            | `ipv4 etr map-server <map-server-rloc-ip> key 0 <key>`                                        | xTR can register the local EID prefix with the map-server                    |                                                                     |
|   21 | Enable ITR function if the router must send traffic to remote EIDs               | xTR                  | `ipv4 itr`                                                                                    | Router can encapsulate traffic toward remote EIDs                            |                                                                     |
|   22 | Point the ITR to the map-resolver                                                | xTR                  | `ipv4 itr map-resolver <map-resolver-rloc-ip>`                                                | Router knows where to resolve remote EID mappings                            |                                                                     |
|   23 | Save xTR configuration                                                           | xTR / ETR            | `end` then `write memory`                                                                     | Local EID mapping configuration is saved                                     |                                                                     |
|   24 | Verify local LISP database mapping                                               | xTR / ETR            | `show ip lisp database`                                                                       | Local EID prefix appears with the correct RLOC, priority, and weight         |                                                                     |
|   25 | Verify running LISP configuration                                                | xTR / ETR            | `show running-config                                                                          | section router lisp`                                                         | `database-mapping`, `ipv4 etr`, and map-server commands are present |
|   26 | Verify site registration on the map-server                                       | MSMR                 | `show lisp site`                                                                              | Site shows registered EID prefix and xTR RLOC                                |                                                                     |
|   27 | Verify detailed site registration                                                | MSMR                 | `show lisp site name <site-name>`                                                             | Correct EID prefix, RLOC, priority, weight, and registration state are shown |                                                                     |
|   28 | Verify the EID is not being routed through the underlay as a normal remote route | xTR / ETR            | `show ip route <remote-eid-prefix>`                                                           | Remote EID should normally resolve through LISP, not plain underlay routing  |                                                                     |
|   29 | Force a lookup from the remote xTR                                               | Remote xTR           | `lig <local-eid-ip>`                                                                          | Lookup returns the local EID-to-RLOC mapping                                 |                                                                     |
|   30 | Verify remote map-cache entry                                                    | Remote xTR           | `show ip lisp map-cache <local-eid-prefix>`                                                   | Remote xTR learns the local EID mapping                                      |                                                                     |
|   31 | Test EID-to-EID reachability                                                     | Source host / router | `ping <remote-eid-ip>`                                                                        | Traffic succeeds between EID spaces                                          |                                                                     |
|   32 | Verify forwarding path                                                           | Source host / router | `traceroute <remote-eid-ip>`                                                                  | Traffic follows the expected LISP overlay path                               |                                                                     |
|   33 | Clear stale map-cache after EID mapping changes                                  | xTR                  | `clear ip lisp map-cache *`                                                                   | Old dynamic EID-to-RLOC mappings are cleared                                 |                                                                     |
|   34 | Recheck map-cache after traffic resumes                                          | xTR                  | `show ip lisp map-cache`                                                                      | Updated EID mappings repopulate correctly                                    |                                                                     |
|   35 | Confirm final operational state                                                  | xTR / ETR / MSMR     | `show ip lisp database`, `show ip lisp map-cache`, `show lisp site`                           | Local EID space is mapped, registered, and resolvable                        |                                                                     |

---
```
conf t
router lisp
 database-mapping <eid-prefix>/<length> <xtr-rloc-ip> priority 1 weight 100
 ipv4 etr
 ipv4 etr map-server <map-server-rloc-ip> key 0 <key>
 ipv4 itr
 ipv4 itr map-resolver <map-resolver-rloc-ip>
end
write memory
```

```
conf t
router lisp
 site <site-name>
  authentication-key 0 <key>
  eid-prefix <eid-prefix>/<length>
 exit
 ipv4 map-server
 ipv4 map-resolver
end
write memory
```
