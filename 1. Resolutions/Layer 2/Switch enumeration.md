| Step | Task                            | Device             | Command                              | What You’re Looking For                                                                                                       |
| ---: | ------------------------------- | ------------------ | ------------------------------------ | ----------------------------------------------------------------------------------------------------------------------------- |
|    1 | Confirm all switch ports are up | SW1, SW2, SW3, SW4 | `show ip interface brief`            | Interfaces in the topology should be `up/up`                                                                                  |
|    2 | Confirm switchport modes        | SW1, SW2, SW3, SW4 | `show interfaces switchport`         | Router-facing ports should usually be access ports. Switch-to-switch links should usually be trunks                           |
|    3 | Check VLAN existence            | SW1, SW2, SW3, SW4 | `show vlan brief`                    | The user VLAN must exist on every switch in the path                                                                          |
|    4 | Check trunks                    | SW1, SW2, SW3, SW4 | `show interfaces trunk`              | Trunks should exist between SW1-SW2, SW2-SW3, SW3-SW4 and allow the needed VLAN                                               |
|    5 | Check STP state                 | SW1, SW2, SW3, SW4 | `show spanning-tree vlan <VLAN-ID>`  | One of each redundant pair may be blocking. That is normal unless the only valid path is blocked due to bad VLAN/trunk config |
|    6 | Check MAC learning from R1      | SW1                | `show mac address-table dynamic`     | R1 MAC should appear on the SW1 port facing R1                                                                                |
|    7 | Check MAC movement across path  | SW2, SW3, SW4      | `show mac address-table dynamic`     | R1/R2 MACs should be learned in the expected direction across trunks                                                          |
|    8 | Check router interface status   | R1, R2             | `show ip interface brief`            | Router LAN interfaces should be `up/up` with correct IPs                                                                      |
|    9 | Check router subnet/mask        | R1, R2             | `show running-config interface g0/1` | If same VLAN test, R1/R2 should be in same subnet. If routed test, they need routes                                           |
|   10 | Test local gateway/link         | R1, R2             | `ping <opposite-router-ip>`          | Failure means inspect VLAN/trunk/STP first if same subnet                                                                     |
|   11 | Trace path                      | R1 or R2           | `traceroute <opposite-router-ip>`    | If same VLAN, there should be no routed hop. If routed, hops should match routing design                                      |
|      |                                 |                    | show interface status err-disabled   |                                                                                                                               |
|      |                                 |                    | show etherchannel summary            |                                                                                                                               |
